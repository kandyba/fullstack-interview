Запитання та відповіді: TypeScript

1. Що таке TypeScript?

TypeScript — це надмножина JavaScript, яка додає статичну типізацію та інші корисні можливості для розробки.

let message: string = "Hello, TypeScript!";
console.log(message);
2. Яка різниця між TypeScript і JavaScript?

Основна різниця в тому, що TypeScript підтримує статичну типізацію, тоді як JavaScript — динамічно типізована мова.

TypeScript: let count: number = 5;
JavaScript: let count = 5;
3. Як встановити TypeScript?

TypeScript можна встановити через npm:

npm install -g typescript
4. Що таке файл конфігурації tsconfig.json?

Це файл конфігурації TypeScript, який дозволяє налаштовувати компіляцію проєкту.

{
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true
  }
}
5. Що таке типи даних у TypeScript?

TypeScript підтримує базові типи даних, такі як number, string, boolean, а також складні типи, наприклад, масиви, кортежі та інші.

let age: number = 25;
let name: string = "John";
let isAdmin: boolean = true;
6. Що таке Union типи?

Union типи дозволяють змінній мати кілька типів.

let value: string | number;
value = "Hello";
value = 42;
7. Що таке інтерфейси в TypeScript?

Інтерфейси визначають структуру об'єктів та можуть використовуватися для перевірки типів.

interface User {
  name: string;
  age: number;
}
let user: User = { name: "Alice", age: 30 };
8. Що таке Generic типи?

Generic типи дозволяють створювати компоненти, які можуть працювати з різними типами даних.

function identity(arg: T): T {
  return arg;
}
let output = identity("Hello");
9. Як оголосити кортежі в TypeScript?

Кортежі дозволяють зберігати масив із фіксованою кількістю елементів різних типів.

let tuple: [string, number];
tuple = ["Hello", 42];
10. Що таке Enums у TypeScript?

Enums дозволяють створювати набір іменованих констант.

enum Direction {
  Up = 1,
  Down,
  Left,
  Right
}
let dir: Direction = Direction.Up;
11. Що таке Generics у TypeScript?
Generics дозволяють створювати функції, класи чи інтерфейси, які працюють з будь-яким типом, забезпечуючи типобезпеку. Це зручно для створення повторно використовуваних компонентів. 
function identity(value: T): T {
  return value;
}
const result = identity("Привіт"); // Привіт
12. Що таке Type Guards?
Type Guards (перевірки типів) — це функції чи вирази, які допомагають TypeScript визначити тип змінної під час виконання. 
function isString(value: unknown): value is string {
  return typeof value === "string";
}
if (isString(someValue)) {
  console.log(someValue.toUpperCase());
}
13. Як працює оператор keyof?
Оператор keyof повертає набір ключів переданого об'єкта або інтерфейсу. 
interface User {
  id: number;
  name: string;
}
type UserKeys = keyof User; // "id" | "name"
14. Що таке Conditional Types у TypeScript?
Conditional Types дозволяють створювати типи на основі умов. Вони використовуються для більш складної типізації. 
type IsString = T extends string ? true : false;
type Check = IsString<"текст">; // true
15. Що таке Utility Types?
Utility Types — це вбудовані типи TypeScript, які спрощують маніпуляцію іншими типами, наприклад: Partial, Pick, Omit. 
interface User {
  id: number;
  name: string;
  age: number;
}
type PartialUser = Partial; // Всі властивості необов'язкові
16. Як TypeScript працює з модулем Node.js?
TypeScript використовує систему модулів CommonJS для Node.js. Ви можете експортувати та імпортувати модулі за допомогою синтаксису import і export. 
// math.ts
export function add(a: number, b: number): number {
  return a + b;
}

// main.ts
import { add } from "./math";
console.log(add(2, 3)); // 5
17. Що таке Decorators у TypeScript?
Decorators — це спеціальні функції, які дозволяють додавати метадані до класів, методів чи властивостей. 
function Log(target: any, propertyKey: string) {
  console.log(`${propertyKey} was accessed`);
}
class User {
  @Log
  name: string = "Іван";
}
const user = new User();
console.log(user.name);
18. Як TypeScript працює з інтерфейсами?
Інтерфейси описують структуру об'єктів, а також дозволяють використовувати розширення. 
interface User {
  id: number;
  name: string;
}
const user: User = { id: 1, name: "Іван" };
19. Чим відрізняється тип від інтерфейсу?
Основна відмінність: type може бути використаний для створення об'єднань та примітивів, тоді як interface — для опису об'єктів. Інтерфейси можна розширювати, а типи об'єднувати.
20. Як працює readonly у TypeScript?
Модифікатор readonly робить властивість доступною лише для читання. 
interface User {
  readonly id: number;
  name: string;
}
const user: User = { id: 1, name: "Іван" };
user.name = "Петро"; // Можливо
// user.id = 2; // Помилка
21. Що таке Tuple у TypeScript?
Tuple — це тип масиву з фіксованою кількістю елементів, де кожен елемент має свій визначений тип. 
const user: [number, string] = [1, "Іван"];
console.log(user[0]); // 1
console.log(user[1]); // "Іван"
22. Як працює модифікатор private?
Модифікатор private робить властивість чи метод доступними лише всередині класу. 
class User {
  private id: number;
  constructor(id: number) {
    this.id = id;
  }
}
const user = new User(1);
// console.log(user.id); // Помилка
23. Як працює модифікатор protected?
protected робить властивість чи метод доступними всередині класу і його нащадків. 
class User {
  protected name: string;
  constructor(name: string) {
    this.name = name;
  }
}
class Admin extends User {
  getAdminName() {
    return this.name;
  }
}
24. Що таке Mapped Types?
Mapped Types дозволяють створювати нові типи шляхом перетворення наявних. 
type ReadonlyUser = {
  readonly [Key in keyof User]: User[Key];
};
25. Що таке Union та Intersection Types?
Union — це тип, який може бути одним з декількох типів. Intersection об'єднує декілька типів в один. 
// Union
type Result = string | number;
let value: Result = "текст"; // або число

// Intersection
type Admin = User & { role: string };
26. Як працює Non-Null Assertion Operator (!)?
Non-Null Assertion Operator ! дозволяє TypeScript ігнорувати перевірки на null або undefined. 
const element = document.querySelector(".item")!;
element.textContent = "Привіт!";
27. Що таке Index Signatures?
Index Signatures дозволяють визначати тип властивостей, ключі яких невідомі заздалегідь. 
interface Dictionary {
  [key: string]: string;
}
const dict: Dictionary = { hello: "Привіт", world: "Світ" };
28. Як працює модифікатор static?
static дозволяє визначати властивості або методи класу, які не належать екземпляру, а доступні на рівні класу. 
class MathUtils {
  static add(a: number, b: number): number {
    return a + b;
  }
}
console.log(MathUtils.add(2, 3)); // 5
29. Що таке Type Assertion?
Type Assertion дозволяє явно вказати TypeScript, який тип ви очікуєте. 
const someValue: unknown = "Привіт";
const valueLength: number = (someValue as string).length;
30. Як працюють Never та Unknown типи?
never вказує, що значення ніколи не буде, наприклад у функціях, що завжди кидають помилки. unknown — це тип для значень невідомого типу. 
// never
function error(message: string): never {
  throw new Error(message);
}

// unknown
let value: unknown = "Привіт";
if (typeof value === "string") {
  console.log(value.toUpperCase());
}
31. Що таке Generics у TypeScript?
Generics — це механізм, який дозволяє створювати компоненти, що можуть працювати з різними типами даних. 
function identity(value: T): T {
  return value;
}
console.log(identity("Привіт")); // "Привіт"
console.log(identity(123)); // 123
32. Як використовувати тип Record?
Тип Record створює об'єкт з ключами одного типу та значеннями іншого типу. 
type UserRoles = Record;
const roles: UserRoles = {
  admin: "Адміністратор",
  user: "Користувач",
};
33. Що таке Utility Types у TypeScript?
Utility Types — це вбудовані типи TypeScript, які допомагають змінювати або використовувати інші типи. 
// Partial
interface User {
  name: string;
  age: number;
}
const partialUser: Partial = { name: "Іван" };

// Readonly
const readonlyUser: Readonly = { name: "Іван", age: 25 };
// readonlyUser.age = 30; // Помилка
34. Як працює Omit у TypeScript?
Omit створює новий тип, видаляючи вказані ключі з існуючого типу. 
interface User {
  id: number;
  name: string;
  age: number;
}
type UserWithoutAge = Omit;
const user: UserWithoutAge = { id: 1, name: "Іван" };
35. Як працює Pick у TypeScript?
Pick створює новий тип, вибираючи лише вказані ключі з існуючого типу. 
interface User {
  id: number;
  name: string;
  age: number;
}
type UserNameOnly = Pick;
const user: UserNameOnly = { name: "Іван" };
36. Як працює keyof у TypeScript?
keyof отримує всі ключі об'єкта у вигляді об'єднання строкових літералів. 
interface User {
  id: number;
  name: string;
}
type UserKeys = keyof User; // "id" | "name"
37. Як працює Conditional Types?
Conditional Types дозволяють створювати типи, засновані на умовах. 
type IsString = T extends string ? true : false;
type Check1 = IsString; // true
type Check2 = IsString; // false
38. Як використовувати Overload функції в TypeScript?
Overload дозволяє визначити кілька сигнатур для однієї функції. 
function combine(a: string, b: string): string;
function combine(a: number, b: number): number;
function combine(a: any, b: any): any {
  return a + b;
}
console.log(combine(1, 2)); // 3
console.log(combine("Привіт, ", "світе")); // "Привіт, світе"
39. Що таке Type Guards?
Type Guards використовуються для уточнення типів в умовах. 
function isString(value: unknown): value is string {
  return typeof value === "string";
}
const value: unknown = "Привіт";
if (isString(value)) {
  console.log(value.toUpperCase());
}
40. Як працює Assertion Functions у TypeScript?
Assertion Functions використовуються для підтвердження певного типу значення. 
function assertIsNumber(value: unknown): asserts value is number {
  if (typeof value !== "number") {
    throw new Error("Value is not a number");
  }
}
const value: unknown = 123;
assertIsNumber(value);
console.log(value.toFixed(2)); // 123.00
