Запитання та відповіді для технічної співбесіди — Загальні питання

1. Що таке HTTP?

HTTP (HyperText Transfer Protocol) — це протокол передачі даних між клієнтом (наприклад, браузером) і сервером. Він визначає, як запити (requests) і відповіді (responses) обмінюються у веб-мережі.

HTTP є основою Всесвітньої павутини (WWW) і використовується для завантаження вебсторінок, зображень, відео та API-запитів.

2. З чого складається HTTP-запит?

Кожне HTTP-повідомлення складається з трьох основних частин:

  • Стартовий рядок — визначає тип повідомлення (наприклад, GET /index.html HTTP/1.1).
  • Заголовки (Headers) — описують параметри передачі, формат контенту, авторизацію тощо.
  • Тіло повідомлення (Body) — містить дані (наприклад, JSON або форму).

Тіло завжди відділяється від заголовків пустим рядком.

3. Які методи HTTP-запитів ви знаєте?
  • GET — отримати дані з сервера.
  • HEAD — отримати лише заголовки без тіла відповіді.
  • POST — відправити дані на сервер (наприклад, форма, JSON).
  • PATCH — часткове оновлення ресурсу.
  • PUT — замінити існуючий ресурс повністю.
  • DELETE — видалити вказаний ресурс.
  • CONNECT — створює тунель (наприклад, через HTTPS-проксі).
  • OPTIONS — повертає дозволені методи для ресурсу.
  • TRACE — тестовий запит, який повертає отримане повідомлення.
4. У чому різниця між PUT і PATCH?

PUT повністю замінює існуючий ресурс, навіть якщо передано лише частину полів. PATCH змінює тільки ті поля, які зазначені у тілі запиту.

PUT /user/1
{
  "name": "Serhii",
  "age": 36
}

PATCH /user/1
{
  "age": 37
}
5. Що таке WebSocket?

WebSocket — це двосторонній протокол зв’язку, який дозволяє клієнту і серверу обмінюватися даними в режимі реального часу через постійне з’єднання.

На відміну від HTTP, WebSocket не потребує повторних запитів — сервер може самостійно надсилати повідомлення клієнту.

Використовується для чату, біржових котирувань, онлайн-ігор, push-нотифікацій тощо.

6. Що таке REST API?

REST (Representational State Transfer) — це архітектурний стиль побудови API, який базується на стандартних методах HTTP.

  • Кожен ресурс має власний URL.
  • Використовуються методи GET, POST, PUT, DELETE тощо.
  • Обмін даними зазвичай відбувається у форматі JSON.
  • Сервер не зберігає стан клієнта між запитами (stateless).

REST API — основа більшості сучасних веб-сервісів.

7. Що таке WebRTC?

WebRTC (Web Real-Time Communication) — це технологія для обміну аудіо, відео та іншими даними між браузерами у реальному часі без проміжних серверів.

  • Дозволяє створювати відео- та аудіо-дзвінки.
  • Забезпечує обмін файлами peer-to-peer.
  • Має низьку затримку та шифрування даних.

Використовується у сервісах типу Google Meet, Zoom, Discord тощо.

9. Які патерни проєктування ви найчастіше використовуєте?

Патерни проєктування — це готові рішення типових проблем у розробці. Найбільш поширені у фронтенді:

  • Singleton — лише один екземпляр класу (глобальний Store у Redux, config, logger).
  • Observer — підписники реагують на зміни стану (React state, event listeners, RxJS).
  • Factory — створення об’єктів через функцію-фабрику (createUser, createButton, API client).
  • Strategy — вибір одного з кількох алгоритмів виконання (різні способи валідації, сортування, оплати).
  • Decorator — додавання функціоналу без зміни вихідного коду (HOC у React, middleware, логування). 
// Singleton приклад
class Config {
  constructor() {
    if (Config.instance) return Config.instance;
    this.api = "https://api.example.com";
    Config.instance = this;
  }
}
const c1 = new Config();
const c2 = new Config();
console.log(c1 === c2); // true
10. Для чого потрібен файл package.json?

package.json — це основний файл конфігурації Node.js або фронтенд-проєкту, який зберігає метадані та залежності.

Основні розділи:

  • name — назва пакета.
  • version — поточна версія застосунку.
  • scripts — команди для запуску (npm run start).
  • dependencies — бібліотеки, необхідні для роботи.
  • devDependencies — бібліотеки лише для розробки (Webpack, ESLint).
{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "start": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "react": "^18.2.0"
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
11. Розкажіть про принципи SOLID.

SOLID — п’ять принципів якісного об’єктно-орієнтованого коду:

  • S — Single Responsibility — один клас = одна задача (AuthService — логін, Logger — логування).
  • O — Open/Closed — нова логіка додається без зміни існуючого коду (додаємо нову Strategy / Validator).
  • L — Liskov Substitution — підклас працює там, де очікується базовий клас (будь-який PaymentService → pay()).
  • I — Interface Segregation — краще кілька вузьких інтерфейсів (Readable, Writable замість одного BigInterface).
  • D — Dependency Inversion — залежимо від абстракції (service приймає repository через constructor).
12. Що таке npm і yarn?

npm (Node Package Manager) — менеджер пакетів, який встановлюється разом із Node.js і використовується для встановлення бібліотек.

Yarn — альтернатива npm, створена Facebook, яка забезпечує швидше встановлення, кешування пакетів і кращу роботу з блокуванням залежностей (yarn.lock).

npm install react
yarn add react
13. Що таке Git і які основні команди ви використовуєте?

Git — це система контролю версій, яка дозволяє відслідковувати зміни у коді та працювати в команді.

Основні команди:

  • git init — створити репозиторій.
  • git add . — додати зміни до індексу.
  • git commit -m "msg" — зафіксувати зміни.
  • git push / pull — відправити або отримати зміни з віддаленого репозиторію.
  • git branch — перегляд або створення гілок.
  • git merge — об’єднання гілок.
14. Чим відрізняються JSON і XML?
  • JSON (JavaScript Object Notation) — формат обміну даними, простий і зручний для JavaScript.
  • XML (eXtensible Markup Language) — старіший формат, більш громіздкий.
  • JSON швидший, легший для читання і має менший розмір.
// JSON
{ "name": "Serhii", "age": 36 }

// XML
<user>
  <name>Serhii</name>
  <age>36</age>
</user>
15. Що таке CORS?

CORS (Cross-Origin Resource Sharing) — механізм безпеки браузера, який контролює доступ до ресурсів із різних доменів.

Наприклад, якщо ваш фронтенд на example.com робить запит до api.server.com, сервер повинен надіслати заголовок:

Access-Control-Allow-Origin: *

Без цього запиту браузер заблокує відповідь.

16. HTTPS, DNS, cookies, session, token — коротко поясніть.
  • HTTPS — захищена версія HTTP, шифрує дані за допомогою SSL/TLS.
  • DNS — система доменних імен, перетворює адресу типу google.com у IP-адресу.
  • Cookies — невеликі файли у браузері, зберігають дані користувача (сесії, налаштування).
  • Session — тимчасові дані на сервері, що зберігають стан користувача під час сеансу.
  • Token — спосіб автентифікації без збереження стану (часто у форматі JWT).
Що відбувається після того, як користувач вводить URL у браузері та натискає Enter?
  1. DNS-запит: Браузер визначає IP-адресу домену (наприклад, shop.com) через DNS (Domain Name System).
  2. Створення TCP-з'єднання: Через протокол TCP (і TLS, якщо HTTPS) встановлюється безпечне з'єднання з сервером.
  3. Відправка HTTP-запиту: Браузер надсилає HTTP GET-запит до сервера на отримання ресурсу (наприклад, index.html).
  4. Сервер обробляє запит: Сервер читає файл з диска або генерує його динамічно.
  5. Сервер надсилає відповідь: Відправляє HTML-код разом із HTTP-заголовками назад до браузера.
  6. Браузер рендерить сторінку: Аналізує HTML, завантажує CSS, JS, зображення, запускає скрипти, показує результат.
Основні абревіатури, які варто знати
  • URL — Uniform Resource Locator (уніфікований локатор ресурсу)
  • HTTP — HyperText Transfer Protocol (протокол передачі гіпертексту)
  • HTTPS — HTTP Secure (HTTP + шифрування через TLS)
  • DNS — Domain Name System (система доменних імен)
  • IP — Internet Protocol (унікальна адреса в інтернеті)
  • TCP — Transmission Control Protocol (протокол керування передачею)
  • TLS — Transport Layer Security (протокол шифрування)
  • HTML — HyperText Markup Language (мова розмітки)
  • CSS — Cascading Style Sheets (каскадні таблиці стилів)
  • JS — JavaScript (скриптова мова для вебу)
  • CDN — Content Delivery Network (мережа доставки контенту)
  • API — Application Programming Interface (інтерфейс прикладного програмування)
  • JSON — JavaScript Object Notation (текстовий формат обміну даними)
  • DOM — Document Object Model (об'єктна модель документа)
  • BOM — Browser Object Model (об'єктна модель браузера)
Які етапи завантаження та рендерингу сторінки в браузері?
  1. Отримання HTML: браузер отримує HTML від сервера.
  2. Парсинг HTML: створюється DOM (Document Object Model).
  3. Завантаження ресурсів: CSS, JS, шрифти, зображення — паралельно.
  4. Парсинг CSS: створюється CSSOM (CSS Object Model).
  5. Об’єднання DOM + CSSOM → Render Tree: структура для візуалізації.
  6. Layout (Reflow): обчислення розмірів та позицій елементів.
  7. Painting: пікселі елементів виводяться на екран.
  8. JS виконується: синхронно під час парсингу або після (якщо async/defer).
