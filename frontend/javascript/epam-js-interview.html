Питання по JavaScript на співбесіді EPAM

1. Що таке doctype? Навіщо використовується?

Призначений для вказівки типу поточного документа. Визначає стандарт рендерингу HTML, який має використовувати браузер.

<!DOCTYPE html>
2. Семантика

Семантика — це підхід, що ґрунтується на використанні HTML-тегів відповідно до їхнього призначення та дотриманні логічної ієрархії сторінки. Наприклад: <header>, <main>, <article>, <footer> замість просто <div>.

3. Опишіть базову структуру HTML-сторінки.
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8">
    <title>Назва сторінки</title>
  </head>
  <body>
    <h1>Заголовок</h1>
    <p>Контент сторінки</p>
  </body>
</html>
4. Центрування в CSS

Найпростіший спосіб центрування елемента:

/* Flexbox */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Для тексту */
.text { text-align: center; }

/* Абсолютне позиціонування */
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
5. Селектори типу div p, div > p

div p — вибирає всі елементи <p>, які знаходяться всередині <div> (на будь-якому рівні вкладеності).

div > p — вибирає лише ті <p>, які є прямими нащадками <div>.

6. Типи позиціонування в CSS
  • static — стандартне положення елемента;
  • relative — позиціонується відносно власного місця;
  • absolute — відносно найближчого позиціонованого предка;
  • fixed — відносно вікна браузера;
  • sticky — комбінує relative і fixed (прилипання під час скролу).
7. Типи даних у JavaScript

Примітиви: string, number, boolean, null, undefined, symbol, bigint.

Непримітивні: object (включаючи array, function, date тощо).

9. AJAX

AJAX (Asynchronous JavaScript and XML) — це технологія для обміну даними з сервером без перезавантаження сторінки.

fetch('/api/data')
  .then(res => res.json())
  .then(data => console.log(data));
12. Що означає «this» в JavaScript?

this — це контекст виконання функції. Значення this обчислюється під час виконання і залежить від контексту.

  • У методі об’єкта this — посилається на об’єкт;
  • У звичайній функції без контексту виклику this === undefined у "use strict"
  • У несуворому режимі, якщо звичайну функцію викликати без об’єкта, this буде глобальним об’єктом (window у браузері).
  • У стрілковій функції — бере this із зовнішнього контексту.
13. Різниця між .call(), .apply() та .bind()
  • call() — викликає функцію з указаним контекстом і передає аргументи через кому;
  • apply() — те саме, але аргументи передаються масивом;
  • bind() — не викликає, а повертає нову функцію з фіксованим контекстом.
14. Різниця між синхронними та асинхронними функціями

Синхронні виконуються послідовно, блокуючи потік; асинхронні — не блокують виконання (через callback, promise, async/await).

15. Що таке Проміси (Promises)?

Promise — це об’єкт, що представляє результат асинхронної операції. Має стани: pending → fulfilled → rejected.

const promise = new Promise((res, rej) => res('done'));
promise.then(console.log);
16. Повернути суму всіх значень у масиві (reduce)
const arr = [1,2,3];
const sum = arr.reduce((a,b) => a + b, 0); // 6
17. Написати функцію, яка сортує масив парних чисел
const data = [1,4,2,5,6];
data.sort((a,b) => b % 2 - a % 2);
18. Методи forEach, map, filter, reduce
  • forEach — виконує функцію для кожного елемента (нічого не повертає);
  • map — повертає новий масив з результатів виклику функції;
  • filter — повертає новий масив елементів, що задовольняють умову;
  • reduce — акумулює значення масиву в одне (число, об’єкт, масив тощо).

Junior — Загальні питання

1. У чому різниця між rebase та merge?

rebase — перезаписує історію комітів, роблячи її лінійною. Головна перевага: чиста та зрозуміла історія без додаткових "злиттів".

merge — об'єднує гілки, зберігаючи всю історію змін. Головна перевага: безпечніше, не змінює існуючі коміти.

2. Різниця між Function та Class
  • Конструктор класу не може бути викликаний без new.
  • Класи завжди працюють у режимі use strict.
  • У функціях може бути super, але тільки в контексті класів воно викликає батьківський конструктор або метод.
3. Чим відрізняється функціональний компонент від класового?

Функціональні компоненти — це прості функції, які приймають props і повертають JSX. Вони не мають власного стану (state) та методів життєвого циклу (до появи React Hooks).

4. Що таке Event Loop?

Event Loop — це механізм, що керує виконанням коду в JavaScript. Інтерпретатор виконує код рядок за рядком, асинхронні функції додаються у чергу callback’ів і виконуються, коли стек викликів звільняється.

5. Що таке super у JavaScript?

super — ключове слово, яке використовується для виклику методів або конструктора батьківського класу.

class Parent {
  constructor(name) {
    this.name = name;
  }
}
class Child extends Parent {
  constructor(name, age) {
    super(name); // виклик батьківського конструктора
    this.age = age;
  }
}
6. Які методи HTTP-запитів ви знаєте?
  • GET — отримати дані з сервера;
  • HEAD — як GET, але без тіла відповіді;
  • POST — відправити дані на сервер (створення ресурсу);
  • PUT — замінити ресурс повністю;
  • PATCH — частково оновити ресурс;
  • DELETE — видалити ресурс;
  • OPTIONS — отримати доступні методи для ресурсу;
  • CONNECT — встановити тунель з сервером.
7. Які коди статусів HTTP ви знаєте?
  • 1xx — інформаційні;
  • 2xx — успішні (наприклад, 200 OK, 201 Created);
  • 3xx — перенаправлення (301, 302);
  • 4xx — помилки клієнта (400 Bad Request, 404 Not Found);
  • 5xx — помилки сервера (500 Internal Server Error).

Junior — JavaScript основи

1. Що таке cookie?

Cookie — це невеликі файли даних (до 4 КБ), які зберігаються у браузері користувача для збереження стану між запитами.

Доступ до них можна отримати через document.cookie.

// Створення cookie з терміном дії 1 день
let date = new Date(Date.now() + 86400e3);
date = date.toUTCString();
document.cookie = "user=John; expires=" + date;

// Cookie буде видалено через 1 годину
document.cookie = "user=John; max-age=3600";
  • path=/ — робить cookie доступним на всьому сайті.
  • domain=site.com — вказує домен, для якого доступне cookie.
  • expires / max-age — визначає тривалість зберігання.
2. Що означає директива use strict?

use strict — це спеціальна директива, яка вмикає «строгий режим» виконання коду.

  • Забороняє використання змінних без оголошення.
  • Не дозволяє дублювати імена параметрів.
  • Робить код більш безпечним і передбачуваним.
3. Що таке статична та динамічна типізації?

Статична типізація — типи перевіряються на етапі компіляції (наприклад, у TypeScript, Java, C#).

Динамічна типізація — типи визначаються під час виконання програми (JavaScript, Python).

4. Як клієнт взаємодіє з сервером?

Клієнт взаємодіє з сервером за допомогою HTTP — протоколу передачі гіпертексту. Через нього відправляються запити (GET, POST, PUT, DELETE тощо) і отримуються відповіді у форматі HTML, JSON або XML.

5. Що таке REST?

REST (Representational State Transfer) — це архітектурний стиль взаємодії клієнта та сервера через HTTP.

  • Використовує стандартні методи HTTP: GET, POST, PUT, DELETE.
  • Кожен ресурс має унікальний URL.
  • Не зберігає стан (stateless).
6. Що таке мутабельність і імутабельність?

Mutable — об’єкт можна змінювати після створення. Immutable — після створення об’єкт змінити не можна (як const примітиви: string, number, boolean).

let arr = [1,2];
arr.push(3); // мутабельна операція

const str = 'hi';
str[0] = 'H'; // не змінюється (immutable)
7. Як шукати помилки в коді? Чи використовуєте ви дебаггер?

Для пошуку помилок можна використовувати:

  • console.log() — для простого логування.
  • Debugger — інструмент DevTools у браузері.
  • Вкладку Network — для перевірки запитів і відповідей.
8. Яких відомих людей зі світу JavaScript ви знаєте?

Ден Абрамов — автор Redux і учасник команди React. Відомий своєю роботою над управлінням станом і великою кількістю освітнього контенту для розробників.

1. Як перевірити, що змінна дорівнює NaN?

Для перевірки значення NaN використовуються функції:

  • isNaN(value)
  • Number.isNaN(value)
isNaN('text')        // true, бо 'text' → NaN
Number.isNaN('text') // false, бо 'text' не тип Number
Number.isNaN(NaN)    // true
2. Чим відрізняється поведінка isNaN() і Number.isNaN()?
  • isNaN() — перетворює перевірюване значення на число перед перевіркою.
  • Number.isNaN() — перевіряє без приведення типів, працює лише з числами.
3. Порівняйте ключові слова var, let, const.
  • const — обов’язково ініціалізується під час оголошення, не може бути перевизначена чи оновлена.
  • let — має блочну область видимості, можна оновити, але не перевизначити.
  • var — функціональна область видимості, може бути перевизначена, піддається hoisting із ініціалізацією в undefined.
var a = 1;   // глобальна
let b = 2;   // блочна
const c = 3; // блочна і незмінна
4. Що таке область видимості (scope)?

Область видимості — це частина коду, де можна звертатися до певної змінної чи функції.

  • Глобальна — доступна звідусіль.
  • Функціональна — діє всередині функцій.
  • Блочна — для змінних let і const у фігурних дужках { }.
  • Модульна — ізольована в межах окремого JS-модуля.
6. Для чого призначені методи setTimeout та setInterval?

Це асинхронні методи для виконання функцій із затримкою:

  • setTimeout(func, delay) — виконує функцію один раз через заданий час (мс).
  • setInterval(func, delay) — виконує функцію багаторазово через інтервал часу.
setTimeout(() => console.log('Через 1 сек'), 1000);
let timer = setInterval(() => console.log('Кожну сек'), 1000);
clearInterval(timer); // зупинити

Асинхронність, масиви, колекції

1. Порівняйте підходи до роботи з асинхронним кодом: callbacks vs promises vs async/await.

Callbacks (функції зворотного виклику) — це функції, які передаються в інші функції для виконання після завершення операції.

Приклад:

setTimeout(() => {
  console.log('Це callback');
}, 1000);

Недолік: складно читати вкладений код — виникає «callback hell».

Promises — це об’єкти, що представляють результат асинхронної операції.

Мають 4 стани: pending, fulfilled, rejected, settled.

const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve('OK'), 1000);
});
promise.then(result => console.log(result));

async / await — синтаксичний цукор над промісами, який робить код асинхронних викликів лінійним і зрозумілим.

async function fetchData() {
  const data = await fetch('/api');
  return data.json();
}

Перевага: код читається як синхронний, не блокує виконання програми.

2. Чи можна записувати нові властивості / функції в прототипи стандартних класів (Array, Object тощо)?

Краще цього не робити, бо це може призвести до конфліктів і непередбачуваної поведінки у бібліотеках або вашому коді.

Якщо потрібно розширити функціонал — краще створити власний клас або утиліту:

// Погано
Array.prototype.myMethod = () => {};

// Добре
class MyArray extends Array {
  customMethod() {}
}
3. Назвіть методи масивів, які пам'ятаєте, і скажіть, для чого вони потрібні.

Для додавання / видалення елементів:

  • push(...items) — додає елементи в кінець.
  • pop() — видаляє останній елемент.
  • shift() — видаляє перший елемент.
  • unshift(...items) — додає елементи на початок.
  • splice(pos, deleteCount, ...items) — видаляє або вставляє елементи.
  • slice(start, end) — створює копію частини масиву.
  • concat(...items) — створює новий масив із доданими елементами.

Для пошуку серед елементів:

  • indexOf(), lastIndexOf() — повертають індекс елемента.
  • includes(value) — перевіряє наявність значення.
  • find() / filter() — повертають елементи, що відповідають умові.
  • findIndex() — повертає індекс знайденого елемента.

Для перебору елементів:

  • forEach(func) — викликає функцію для кожного елемента.

Для перетворення масиву:

  • map(func) — створює новий масив із результатів виклику функції.
  • sort(func) — сортує масив «на місці».
  • reverse() — змінює порядок елементів на протилежний.
  • split() / join() — перетворюють рядок у масив і навпаки.
  • reduce(func, initial) — зводить масив до одного значення.

Додатково:

  • Array.isArray(arr) — перевіряє, чи є значення масивом.
4. Опишіть призначення та принципи роботи з колекціями Map та Set.

Map

Зберігає пари ключ → значення, де ключ може бути будь-яким типом (навіть об’єктом).

const map = new Map();
map.set('name', 'John');
map.get('name'); // 'John'
map.has('name'); // true
map.delete('name');

Set

Зберігає унікальні значення без дублікатів.

const set = new Set([1, 2, 2, 3]);
set.add(4);
set.has(2); // true
set.size; // 3

Junior — Об’єкти, колекції, типи даних

1. Чим відрізняються instanceof та typeof?
  • typeof — використовується для перевірки типу примітивних значень (string, number, boolean, undefined, symbol, bigint).
  • instanceof — перевіряє, чи є об’єкт екземпляром певного класу або чи присутній цей клас у його ланцюжку прототипів.
typeof 'text'        // "string"
typeof []            // "object"
[1,2,3] instanceof Array // true
new Date() instanceof Object // true
2. Що означає глибока (deep) і поверхнева (shallow) копія об'єкта? Як зробити кожну з них?

Поверхнева (shallow) копія — копіює лише верхній рівень властивостей, вкладені об’єкти залишаються посиланнями.

const obj = { a: 1, b: { c: 2 } };
const shallow = Object.assign({}, obj);
shallow.b.c = 10; // змінить і obj.b.c

Глибока (deep) копія — копіює всі рівні вкладеності.

const deep = JSON.parse(JSON.stringify(obj));
// або з бібліотекою lodash
// const deep = _.cloneDeep(obj);
3. Методи та властивості колекції Map
  • new Map() — створює колекцію.
  • map.set(key, value) — додає пару ключ-значення.
  • map.get(key) — повертає значення або undefined.
  • map.has(key) — перевіряє наявність ключа.
  • map.delete(key) — видаляє елемент.
  • map.clear() — очищає колекцію.
  • map.size — кількість елементів.
let map = new Map();
map.set("1", "str1");
map.set(1, "num1");
map.set(true, "bool1");

Перебір Map:

  • map.keys() — ітератор по ключах.
  • map.values() — ітератор по значеннях.
  • map.entries() — ітератор по парах [ключ, значення].
4. Методи та властивості колекції Set
  • new Set([iterable]) — створює множину.
  • set.add(value) — додає елемент.
  • set.delete(value) — видаляє елемент.
  • set.has(value) — перевіряє наявність значення.
  • set.clear() — очищає множину.
  • set.size — кількість елементів.
const set = new Set([1, 2, 2, 3]);
set.add(4);
set.has(2); // true
set.size;   // 3
5. Методи Object.keys, Object.values, Object.entries
  • Object.keys(obj) — повертає масив ключів.
  • Object.values(obj) — повертає масив значень.
  • Object.entries(obj) — повертає масив пар [ключ, значення].
const user = {name: 'John', age: 30};
Object.keys(user);   // ['name', 'age']
Object.values(user); // ['John', 30]
Object.entries(user);// [['name','John'], ['age',30]]
6. Різниця між null і undefined
  • undefined — змінна оголошена, але не має значення.
  • null — явно присвоєне «порожнє» значення.
let a;
console.log(a);           // undefined
let b = null;
console.log(b);           // null
typeof null;              // "object"
typeof undefined;         // "undefined"
7. Для чого використовується оператор !!?

Подвійне заперечення !! перетворює будь-яке значення у логічне (true або false).

!!'text'   // true
!!0        // false
!!undefined // false
!![]       // true

Junior — Функції, контекст, DOM, події, сховище, Webpack

1. Розкажіть про стрілочні функції (arrow functions). У чому їх відмінність від звичайних?
  • Не мають власного this — використовують контекст батьківської області.
  • Не мають arguments.
  • Не можуть бути викликані через new.
  • Не мають super.
// звичайна функція
function sum(a, b) { return a + b; }

// стрілочна
const sum = (a, b) => a + b;
2. Як ви розумієте замикання (closure)?

Замикання — це функція, що запам’ятовує змінні із зовнішньої області видимості, навіть після того, як ця область уже завершила виконання.

function counter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}
const add = counter();
add(); // 1
add(); // 2
Функції автоматично запам’ятовують, де вони були створені, використовуючи приховану властивість [[Environment]], а потім їхній код може отримати доступ до зовнішніх змінних.
Коли під час співбесіди запитують “що таке замикання?”, правильною відповіддю буде визначення замикання та пояснення, що всі функції в JavaScript є замиканнями, і, можливо, ще кілька слів про технічні деталі: властивість [[Environment]], і як взагалі працюють лексичні середовища.
3. Що таке втрата контексту, коли вона відбувається і як її запобігти?

Втрата контексту — це ситуація, коли значення this змінюється, і метод не може звернутися до своїх властивостей.

Відбувається при:

  • Передачі методу як колбеку.
  • Виклику методу без об’єкта (const fn = obj.method).

Запобігається:

  • Використанням bind(), call(), apply().
  • Використанням стрілочних функцій.
4. Що таке DOM?

DOM (Document Object Model) — це об’єктна модель HTML-документа, що дозволяє JS отримувати, створювати, змінювати або видаляти елементи сторінки.

document.querySelector('#id');
document.createElement('div');
document.body.appendChild(el);
5. Порівняйте атрибути підключення скрипта async і defer в HTML-документі.
  • async — завантажує скрипт паралельно з HTML, виконує одразу після завантаження (не чекає DOM).
  • defer — завантажує паралельно, але виконує лише після повного розбору HTML.
<script src="file.js" async></script>
<script src="file.js" defer></script>
7. Опишіть процес спливання (bubbling) подій у DOM.

Подія спочатку спрацьовує на елементі-цілі (event.target), а потім піднімається до його батьківських елементів аж до window.

child → parent → grandparent → document → window
8. Як зупинити спливання (bubbling) події?

Метод event.stopPropagation() відключає спливання або занурення події.

9. Як зупинити дефолтну обробку події?

Метод event.preventDefault() забороняє стандартну дію браузера (наприклад, перехід за посиланням).

10. Чому дорівнює this в обробнику подій (event handler)?

У звичайній функції this вказує на елемент, на якому викликана подія.

btn.onclick = function() {
  console.log(this === btn); // true
};

У стрілочній функції this береться із зовнішнього контексту.

11. Що таке LocalStorage і SessionStorage? Який максимальний розмір LocalStorage?
  • LocalStorage — зберігає дані безстроково (до очищення вручну).
  • SessionStorage — зберігає дані лише в межах однієї вкладки браузера.
  • Максимальний розмір LocalStorage ≈ 5 МБ.
localStorage.setItem('user', 'John');
localStorage.getItem('user'); // "John"
sessionStorage.setItem('theme', 'dark');
12. Як отримати висоту блоку? Його положення відносно меж документа?
element.offsetHeight;           // висота з padding та border
element.clientHeight;           // висота без border
element.getBoundingClientRect(); // координати елемента
13. Що таке Webpack?

Webpack — це збирач модулів (module bundler), який об’єднує всі файли (JS, CSS, зображення) в оптимізований пакет для продакшну.

Має конфігураційний файл webpack.config.js, підтримує лоадери, плагіни, девсервер і мінімізацію коду.

Middle — React. Класові компоненти, хуки, контекст, оптимізація

1. Чи працювали ви з класовими компонентами? У чому їх особливість?

Так, класові компоненти — це старіший спосіб створення компонентів у React до появи хуків.

  • Вони використовують класовий синтаксис ES6: class MyComponent extends React.Component.
  • Мають власний стан (state) і методи життєвого циклу.
  • Для доступу до контексту чи рефів застосовується this.
class Counter extends React.Component {
  state = { count: 0 };
  increment = () => this.setState({ count: this.state.count + 1 });

  render() {
    return <button onClick={this.increment}>{this.state.count}</button>;
  }
}
2. Які дані краще зберігати в стані компонента, а які передавати через пропси? Наведіть приклад.
  • У state зберігаються дані, які можуть змінюватися всередині компонента (наприклад, введення користувача, стан завантаження).
  • Через props передаються незмінні дані або callback-функції від батьківського компонента.
function Child({ value, onClick }) {
  return <button onClick={onClick}>{value}</button>;
}

function Parent() {
  const [count, setCount] = useState(0);
  return <Child value={count} onClick={() => setCount(count + 1)} />;
}
3. Чи знайомі ви з хуками? У чому їх переваги? Чи доводилося створювати власні?

Так, хуки (Hooks) дозволяють використовувати стан і життєвий цикл без класів.

Переваги:

  • Менше коду, краща читабельність.
  • Немає необхідності в this.
  • Можна легко ділитися логікою між компонентами через власні хуки.

Приклад користувацького хука:

function useWindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);
  useEffect(() => {
    const handleResize = () => setWidth(window.innerWidth);
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  return width;
}
4. Чи знайомі ви з фрагментами та порталами? Для чого вони потрібні?

Фрагменти (Fragments) дозволяють групувати кілька елементів без створення додаткового DOM-вузла.

<>
  <h1>Заголовок</h1>
  <p>Опис</p>
</>

Портали (Portals) — це спосіб рендерити компонент у DOM-позіції, відмінній від його батьківського вузла. Використовуються, наприклад, для модальних вікон.

ReactDOM.createPortal(
  <Modal />,
  document.getElementById('modal-root')
);
5. Коли і для чого використовуються рефи (refs)?

Refs використовуються для прямого доступу до DOM-елементів або збереження змінних, які не повинні викликати повторний рендер.

function InputFocus() {
  const inputRef = useRef();
  return (
    <>
      <input ref={inputRef} />
      <button onClick={() => inputRef.current.focus()}>Фокус</button>
    </>
  );
}
6. Які ви знаєте методи життєвого циклу компонента?

Основні методи класових компонентів:

  • componentDidMount() — викликається після монтування.
  • componentDidUpdate() — після оновлення.
  • componentWillUnmount() — перед розмонтуванням.
  • shouldComponentUpdate() — контролює, чи потрібно перерендерювати компонент.
7. В якому методі життєвого циклу краще робити запити на сервер і чому?

У componentDidMount(), тому що на момент виклику компонент уже створений і готовий до оновлення стану після отримання даних.

8. В якому методі краще робити підписку та відписку від подій?

Підписку — у componentDidMount(), відписку — у componentWillUnmount(). Це запобігає витокам пам’яті та зайвим викликам подій.

9. Чи є досвід роботи з контекстом? Коли його варто використовувати?

Context API використовується для передачі даних між компонентами без пропс-дрилінгу.

Добре підходить для тем, мови інтерфейсу або авторизаційних даних.

const ThemeContext = React.createContext('light');
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}
10. В чому особливість PureComponent?

PureComponent — це класовий компонент, який автоматично реалізує поверхневе порівняння пропсів і стану у shouldComponentUpdate().

Якщо дані не змінилися — компонент не перерендерюється.

Аналог для функціональних компонентів — React.memo().

11. Чи працювали ви з мемоізованими селекторами (memoized selectors)? Для чого вони використовуються?

Так, у Redux часто використовують бібліотеку reselect для створення мемоізованих селекторів.

Вони зберігають результати обчислень, щоб не виконувати повторні обчислення, якщо дані не змінилися — це оптимізує продуктивність.

12. Чому бібліотека React швидка? Що таке Virtual DOM і Shadow DOM?
  • Virtual DOM — це копія реального DOM у пам’яті. React оновлює лише змінені елементи, а не весь DOM, що підвищує продуктивність.
  • Shadow DOM — це окрема частина DOM, ізольована від решти документа (використовується у Web Components, не у React).
13. Навіщо в списках ключі (keys)?

Ключі потрібні, щоб React міг відслідковувати елементи при оновленні списку. Без унікальних ключів можуть виникати помилки або неправильний рендер.

Можна використовувати індекси як ключі лише якщо список не змінюється (тобто порядок елементів стабільний).

14. В чому основна ідея Redux?

Redux — це централізоване сховище стану, яке дотримується трьох принципів:

  • Єдине джерело істини — один глобальний state.
  • Стан є лише для читання (змінюється через дії — actions).
  • Зміни описуються чистими функціями (reducers).
const initialState = { count: 0 };
function reducer(state = initialState, action) {
  switch (action.type) {
    case 'INC':
      return { count: state.count + 1 };
    default:
      return state;
  }
}

Fullstack — Бази даних, Git, CI/CD

1. Напишіть простий запит для виведення трьох авторів, у яких найбільше книг.
SELECT author, COUNT(*) AS book_count
FROM books
GROUP BY author
ORDER BY book_count DESC
LIMIT 3;
2. Напишіть запит, який вибирає останні три коментарі для конкретного користувача з двох таблиць: коментарі та користувачі.
SELECT c.*
FROM comments c
JOIN users u ON c.user_id = u.id
WHERE u.id = 5
ORDER BY c.created_at DESC
LIMIT 3;
3. Спроектуйте просту схему бази даних для бібліотеки.

Приклад базової структури:

📘 books (id, title, author_id, published_year, genre_id)
👤 authors (id, name, birth_date)
💬 users (id, name, email)
📖 borrows (id, user_id, book_id, borrow_date, return_date)
🏷 genres (id, name)

Зв’язки:

  • Один author має багато books (one-to-many).
  • Один user може взяти кілька books через таблицю borrows.
4. Для чого використовують SQL-оператор HAVING?

HAVING використовується для фільтрації результатів після операції GROUP BY. На відміну від WHERE, він працює з агрегованими значеннями.

SELECT author, COUNT(*) AS books
FROM books
GROUP BY author
HAVING COUNT(*) > 5;
5. Навіщо використовують SQL-оператор LEFT JOIN?

LEFT JOIN повертає всі записи з лівої таблиці, навіть якщо у правій немає відповідних даних.

SELECT users.name, orders.id
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
6. Чим відрізняються embed- та reference-зв'язки в MongoDB?
  • Embed (вбудовані) — дані зберігаються всередині документа. Зручно, коли дані завжди використовуються разом.
  • Reference (посилальні) — документи зберігаються окремо, але пов’язані через ID. Підходить для великих або спільних колекцій.
// Embed
{
  title: "Book A",
  author: { name: "John Smith", country: "UK" }
}

// Reference
{
  title: "Book B",
  author_id: ObjectId("653f...")
}
7. У проекті потрібно внести зміни в структуру таблиць, додати кілька полів та індексів. Як програмісти роблять це на продакшн-середовищі?

Використовують міграції бази даних (database migrations), щоб синхронізувати зміни структури між середовищами.

  • Підготовка скриптів SQL (ALTER TABLE, CREATE INDEX тощо).
  • Тестування на staging середовищі.
  • Застосування на продакшні через CI/CD або вручну з резервним копіюванням.

Інструменти: Sequelize Migrations, Knex.js, Flyway, Liquibase.

8. Кожного разу, коли ви робите pull, чомусь виникає конфлікт в останньому рядку у всіх файлах, які ви редагували. Що відбувається?

Швидше за все, у файлах відрізняється кінець рядка (EOL) — CRLF (Windows) проти LF (Unix).

Щоб уникнути цього:

  • Налаштувати .gitattributes:
* text=auto
*.js text eol=lf
  • У редакторі VSCode встановити End of Line → LF.

    git fetch завантажує останні зміни з віддаленого репозиторію, але не зливає їх з вашою поточною гілкою.

    Після цього можна порівняти зміни та виконати git merge або git rebase вручну.

    10. Які підходи до git hygiene ви знаєте?
    • Один коміт — одна логічна зміна.
    • Регулярне оновлення гілки від main.
    • Очищення історії комітів через rebase -i.
    • Використання зрозумілих назв гілок і комітів.
    • Не пушити у main напряму.
    11. Що таке CI/CD? Для чого це потрібно?

    CI/CD — це підхід до автоматизації процесів інтеграції та розгортання коду.

    • CI (Continuous Integration) — автоматичне тестування та перевірка коду після кожного коміту.
    • CD (Continuous Deployment/Delivery) — автоматичне або напівавтоматичне розгортання на сервері після проходження CI.

    Мета: мінімізувати людські помилки, прискорити доставку продукту, гарантувати стабільність.
