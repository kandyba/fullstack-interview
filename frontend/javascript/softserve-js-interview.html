Технічна співбесіда з JavaScript

01-intro: Основи JavaScript
1. Які є типи даних в JS, основна різниця між ними?

Примітивні типи: string, number, boolean, undefined, null, symbol, bigint

Посилальний тип: object (включає array, function)

Різниця: Примітиви зберігають значення безпосередньо, об'єкти зберігають посилання на місце в пам'яті.

let a = 5; // примітив
let b = {x: 5}; // об'єкт (посилання)
2. Різниця між undefined та null?

undefined: Тип та значення, яке автоматично присвоюється змінній, якщо вона оголошена, але не ініціалізована. Також повертається функцією, якщо немає return.

null: Спеціальне значення, яке програміст присвоює вручну для позначення "порожнього", "відсутнього об'єкта".

Типи:

  • typeof undefined === 'undefined'
  • typeof null === 'object' (історичний баг)

Поведінка при порівнянні:

  • undefined == null → true (нестроге порівняння)
  • undefined === null → false (різні типи)
  • Boolean(undefined) та Boolean(null) → false (обидва falsy)

Приклади:

let x;               // undefined
let y = null;        // null (явно)
typeof x;            // 'undefined'
typeof y;            // 'object'

function foo() {}
console.log(foo());  // undefined (функція без return)

undefined == null    // true
undefined === null   // false

Коли використовувати:

  • undefined: коли змінна ще не має значення, або як дефолт функції.
  • null: щоб вручну обнулити змінну чи сигналізувати "відсутність значення".

Пастки на співбесіді:

  • typeof null === 'object' — класичне підступне запитання
  • return без значення → undefined
  • Обидва значення рівні лише між собою при ==, але ніколи не дорівнюють іншим falsy (0, "", false)
3. Різниця між var, let та const?

var: Функціональна область видимості, hoisting (підняття), можна перевизначати.

let: Блокова область видимості, temporal dead zone, можна перевизначати.

const: Блокова область видимості, незмінне присвоєння (для об'єктів - незмінне посилання).

var a = 1; // function scope
let b = 2; // block scope
const c = 3; // block scope, незмінне
4. Яка різниця між ECMAScript та JavaScript?

ECMAScript: Специфікація, стандарт мови (опис правил).

JavaScript: Конкретна імплементація стандарту ECMAScript в браузерах та середовищах виконання.

5. Що таке динамічна типізація? Як її ще називають?

Тип змінної визначається під час виконання коду і може змінюватися. Також називається слабка типізація.

let x = 5; // number
x = "hello"; // тепер string
6. Які є варіанти додавання JS коду до сторінки?
  • Тег <script> безпосередньо в HTML
  • Зовнішній файл: <script src="file.js"></script>
  • Inline атрибути подій (onclick, onload тощо)
  • Динамічне створення script елемента через JS
7. Які є обмеження для коду написаного на JS, який запускається в браузері?
  • Немає доступу до файлової системи (окрім File API з дозволом)
  • Same Origin Policy (обмеження міжсайтових запитів)
  • Обмежений доступ до камери/мікрофону (потрібен дозвіл)
  • Немає прямого доступу до системних ресурсів
  • Обмеження на роботу з cookies інших доменів
8. Яка різниця між компільованою та інтерпритованою мовою програмування? До якої відноситься JS?

Компільована: Код перетворюється в машинний код ДО виконання (C++, Go).

Інтерпретована: Код виконується рядок за рядком під час виконання (Python, PHP).

JavaScript: Використовує JIT-компіляцію (Just-In-Time) - гібридний підхід, компіляція під час виконання.

Оператори
1. Які є види операторів в JS?
  • Арифметичні (+, -, *, /, %, **)
  • Порівняння (==, ===, !=, !==, <, >, <=, >=)
  • Логічні (&&, ||, !)
  • Присвоєння (=, +=, -=, *=, /=)
  • Бітові (&, |, ^, ~, <<, >>)
  • Тернарний (? :)
  • Інші (typeof, delete, in, instanceof, void)
2. Різниця між оператором == та ===, який краще використовувати?

== (нестроге порівняння): порівнює значення після автоматичного приведення типів (type coercion).

=== (строге порівняння): порівнює і тип, і значення без приведення.

Рекомендовано: завжди використовувати ===, щоб уникнути неочевидних результатів.

Типові приклади:

5 == "5"     // true  (рядок → число)
5 === "5"    // false (різні типи)
null == undefined     // true
null === undefined    // false
false == 0            // true
false === 0           // false
"" == 0               // true
[] == false           // true
[] === false          // false

Підступи:

  • [] == false → true (порожній масив → "" → 0)
  • null == undefined → true, але не рівні нічому іншому
  • 0 == "0" → true
  • "" == false → true

На співбесіді можуть спитати:

  • Чому null == undefined → true?
  • Чому [] == false → true?
  • Чому NaN === NaN → false? (і як це перевіряти?)

Правильна перевірка на NaN:

Number.isNaN(NaN);     // true
Object.is(NaN, NaN);   // true
3. Різниця між префіксним та постфіксним оператором ++, який краще використовувати?

++i (префіксний): Спочатку збільшує, потім повертає нове значення.

i++ (постфіксний): Спочатку повертає старе значення, потім збільшує.

Краще ++i якщо не потрібне старе значення (ефективніше).

let i = 5;
console.log(++i); // 6 (спочатку +1)
console.log(i++); // 6 (потім +1, тепер i=7)
4. Що таке пріорітет операторів, на що він впливає?

Визначає порядок виконання операцій у виразі. Множення/ділення мають вищий пріоритет за додавання/віднімання.

2 + 3 * 4 // 14, бо * має вищий пріоритет
5. Як можна підвищити пріоритет оператора?

Використовувати дужки () для явного визначення порядку.

(2 + 3) * 4 // 20
6. Від чого залежить результат роботи оператора?

Від типів операндів та самого оператора. JS може автоматично приводити типи.

5 + 3    // 8 (числа)
"5" + 3  // "53" (конкатенація рядків)
7. Які значення вважаються false при перетворенні до типу boolean?

Falsy values: false, 0, "" (порожній рядок), null, undefined, NaN

Всі інші значення - truthy.

Boolean(0)         // false
Boolean("")        // false
Boolean("hello")   // true
8. Яка різниця між логічними та арифметичними операторами?

Арифметичні оператори (+, -, *, /, %, **): Виконують математичні операції. Повертають number, або NaN якщо обчислення неможливе.

Логічні оператори (&&, ||, !): Виконують логічні перевірки. Повертають значення одного з операндів (не обов'язково boolean).

Важливі особливості логічних операторів:

  • || повертає перше truthy значення або останнє
  • && повертає перше falsy значення або останнє
  • ! завжди повертає булеве значення (інверсія)
// Арифметичні:
5 + 3       // 8
10 % 3      // 1
2 ** 3      // 8

// Логічні:
true && 5         // 5
false || "hello"  // "hello"
!0                // true
"abc" && 0        // 0
"" || null        // null

Підступні моменти:

  • "5" + 1 → "51" (арифметичний + рядок = конкатенація)
  • Boolean("false") → true (непорожній рядок — truthy)
  • null || undefined → undefined
  • NaN && "hello" → NaN (falsy && щось = falsy)

Для умов завжди приводь результат до boolean: Boolean(x) або подвійне заперечення !!x

9. За яким принципом виконується оператор порівняння рядків 'str' > 'str'?

Посимвольно за кодами Unicode (лексикографічне порівняння). Порівнюються символи зліва направо.

"apple" < "banana" // true (a < b)
"a" < "A"          // false (97 > 65 в Unicode)
10. Яка різниця між isNaN() та Number.isNaN()?

isNaN(): Спочатку приводить аргумент до числа, потім перевіряє.

Number.isNaN(): Перевіряє чи значення строго є NaN без приведення.

isNaN("hello")        // true (приведено до NaN)
Number.isNaN("hello") // false (рядок, не NaN)
Number.isNaN(NaN)     // true
11. Які існують вбудовані способи перетворення рядка до типу number?
  • Number() - повне перетворення
  • parseInt() - парсинг до цілого числа
  • parseFloat() - парсинг до числа з плаваючою точкою
  • Унарний + - швидке перетворення
Number("123")      // 123
parseInt("123px")  // 123
parseFloat("12.5") // 12.5
+"123"             // 123
12. Що таке явне (explicit) та неявне (implicit) перетворення типів?

Явне: Програміст сам викликає функцію перетворення (Number(), String()).

Неявне: JS автоматично приводить типи під час операцій.

// Явне
Number("5")        // 5

// Неявне
"5" * 2            // 10 (рядок → число)
5 + "2"            // "52" (число → рядок)
13. Чому результат даного виразу буде "null"?

Вираз: "" || "null" || false || undefined || 0 || "hello"

Оператор || повертає перше truthy значення. Порожній рядок "" - falsy, а "null" (непорожній рядок) - truthy, тому результат "null".

"" || "null" || false  // "null" (перше truthy)
14. Чому результат даного виразу буде 0?

Вираз: "false" && " " && 0 && undefined

Оператор && повертає перше falsy значення або останнє значення. "false" та " " - truthy, а 0 - перше falsy, тому результат 0.

"false" && " " && 0  // 0 (перше falsy)

Масиви

1. Яка різниця між методами splice() та slice()?

splice(start, deleteCount, ...items): Змінює оригінальний масив — може видаляти, додавати або замінювати елементи. Повертає масив видалених елементів.

slice(start, end): Не змінює масив. Повертає новий масив із копією частини елементів, від start до end (не включно).

Основні відмінності:

  • splice() — мутує масив
  • slice() — створює копію
  • splice() — універсальний: видаляє, вставляє, замінює
  • slice() — тільки «вирізає»
let arr = [1, 2, 3, 4, 5];

// splice: видалити 2 елементи з індексу 1
let removed = arr.splice(1, 2);
// removed = [2, 3], arr = [1, 4, 5]

// splice: вставити елемент
arr.splice(1, 0, 99);
// arr = [1, 99, 4, 5]

// slice: копія від індексу 1 до 3 (не включно)
let copy = arr.slice(1, 3);
// copy = [99, 4], arr не змінюється

Підступи для співбесіди:

  • splice() повертає видалені елементи, не змінений масив
  • slice() не включає end індекс
  • Якщо deleteCount у splice() = 0 → нічого не видалиться
  • Негативні індекси працюють як offset від кінця
let arr = [10, 20, 30, 40, 50];

// slice з негативним індексом
arr.slice(-3);    // [30, 40, 50]

// splice з заміною
arr.splice(2, 1, 99); // [30], arr = [10, 20, 99, 40, 50]

Що питати самому собі: "Чи я змінюю оригінальний масив?" — якщо відповідь "так", це splice().

2. Яким чином можна створити копію всього масиву?

Поверхнева копія (shallow copy): Копіює лише верхній рівень елементів. Якщо масив містить об’єкти — копіюються посилання.

Способи:

  • [...arr] — spread оператор
  • arr.slice() — повертає новий масив
  • Array.from(arr) — створює копію з ітерабельного об'єкта
  • arr.concat() — без аргументів створює копію
  • structuredClone(arr) — глибока копія (тільки у сучасних браузерах)
let arr = [1, 2, 3];

// поверхнева копія
let copy1 = [...arr];
let copy2 = arr.slice();
let copy3 = Array.from(arr);
let copy4 = arr.concat();

// глибока копія (лише якщо всі елементи серіалізуються)
let deepCopy = structuredClone(arr);

Проблема поверхневої копії: вкладені об'єкти/масиви все ще залишаються за посиланням.

let arr = [{x: 1}, {x: 2}];
let shallow = [...arr];

shallow[0].x = 999;
console.log(arr[0].x); // 999 — оригінал теж змінено!

Глибока копія: для копіювання вкладених структур використовуй:

  • structuredClone(arr) — рекомендовано
  • JSON.parse(JSON.stringify(arr)) — обхідний шлях, але не працює з функціями, undefined, Date
  • Рекурсивне копіювання вручну або з бібліотекою (lodash.cloneDeep)

На співбесіді можуть спитати:

  • Чим відрізняється копія з [...arr] і arr.slice()
  • Чому змінився об'єкт у копії і в оригіналі одночасно?
  • Що таке shallow vs deep copy
3. Яка різниця між методами filter() та find()?

filter(): Повертає новий масив з усіма елементами, що відповідають умові.

find(): Повертає перший елемент, що відповідає умові, або undefined.

let arr = [1, 5, 10, 15];
arr.filter(x => x > 5);  // [10, 15]
arr.find(x => x > 5);    // 10
4. Як працює метод reduce()?

Зводить масив до одного значення, виконуючи функцію для кожного елемента з накопиченням результату.

Синтаксис: arr.reduce((accumulator, currentValue) => {...}, initialValue)

let arr = [1, 2, 3, 4];
arr.reduce((sum, num) => sum + num, 0);  // 10

// accumulator - накопичувач (початок: initialValue)
// currentValue - поточний елемент
5. Яким чином можна визначити кількість негативних елементів в масиві?

Використати filter() з перевіркою на від'ємне значення і взяти .length.

let arr = [-1, 5, -3, 10, -2];
let count = arr.filter(x => x < 0).length;  // 3

// Альтернатива з reduce:
arr.reduce((count, x) => x < 0 ? count + 1 : count, 0);

Loops (Цикли)

1. В чому відмінність між циклами while та do-while?

while: Перевіряє умову ДО виконання тіла циклу (може не виконатись жодного разу).

do-while: Перевіряє умову ПІСЛЯ виконання тіла (виконається мінімум 1 раз).

let i = 5;
while (i < 5) { }       // не виконається

do { } while (i < 5);   // виконається 1 раз
2. Який оператор використовується для пропущення ітерації в циклі?

continue - пропускає поточну ітерацію і переходить до наступної.

for (let i = 0; i < 5; i++) {
  if (i === 2) continue;  // пропустити i=2
  console.log(i);         // 0, 1, 3, 4
}
3. Для чого потрібний оператор break?

Повністю зупиняє виконання циклу і виходить з нього.

for (let i = 0; i < 10; i++) {
  if (i === 5) break;  // зупинити на i=5
  console.log(i);      // 0, 1, 2, 3, 4
}
4. В чому різниця між for-in та for-of?

for-in: Ітерується по ключах (властивостях) об'єкта або індексах масиву.

for-of: Ітерується по значеннях ітерабельних об'єктів (масив, рядок, Map, Set).

let arr = ['a', 'b', 'c'];

for (let key in arr) { }      // 0, 1, 2 (індекси)
for (let value of arr) { }    // 'a', 'b', 'c' (значення)

Для масивів краще for-of, для об'єктів - for-in.

5. Який порядок виконання циклу for?
  1. Ініціалізація (let i = 0) - 1 раз на початку
  2. Перевірка умови (i < 5)
  3. Виконання тіла циклу
  4. Інкремент (i++)
  5. Повторення з кроку 2
for (let i = 0; i < 3; i++) {
  console.log(i);
}
// Порядок: i=0 → умова → тіло → i++ → умова → тіло → i++ → умова → вихід

Functions (Функції)

1. Що таке функції в JS?

Блок коду, який виконує певну задачу, може приймати параметри та повертати значення. Функції - це об'єкти першого класу (можна передавати, повертати, зберігати).

2. Які типи функцій існують в JS?
  • Function Declaration: function name() {}
  • Function Expression: const name = function() {}
  • Arrow Function: const name = () => {}
  • Anonymous Function: функція без імені
  • IIFE: (function() {})() - негайно викликана
  • Constructor Function: function User() {} з new
  • Generator Function: function* gen() {}
3. Що таке hoisting? Чи всі функції його мають?

Hoisting (підняття): це механізм, за якого оголошення змінних і функцій "піднімаються" на початок області видимості ще до виконання коду.

Це означає, що можна звертатися до деяких змінних або функцій до їх оголошення в коді — але не до всіх.

Поведінка різних конструкцій:

  • Function Declaration — піднімається повністю: і ім’я, і тіло функції.
  • Function Expression (у вигляді const/let/var) — тільки оголошення змінної, без значення (тобто undefined або ReferenceError).
  • Arrow Function — те саме, що й Function Expression.
  • var — піднімається, але значення залишається undefined (і доступне в усьому scope).
  • let / const — піднімаються, але потрапляють у temporal dead zone (TDZ) до ініціалізації.
// Function Declaration — працює до оголошення:
sayHi(); // Hello
function sayHi() {
  console.log("Hello");
}

// Function Expression — ReferenceError:
sayBye(); // ❌ Error
const sayBye = function() {
  console.log("Bye");
};

// var — undefined:
console.log(a); // undefined
var a = 5;

// let/const — ReferenceError:
console.log(b); // ❌ Error
let b = 10;

TDZ (Temporal Dead Zone): це проміжок часу між початком блоку та ініціалізацією let/const змінної, де доступ до неї заборонений.

На співбесіді можуть спитати:

  • Чому функція через const не працює до оголошення?
  • Чому var x = 10; дозволяє console.log(x) до оголошення, а let x = 10; — ні?
  • Що таке TDZ і як себе проявляє?
4. Що таке аргументи функції? Які типи аргументів існують?

Аргументи: Значення, які передаються функції при виклику.

Типи:

  • Позиційні - передаються по порядку
  • За замовчуванням - function(a = 5) {}
  • Rest parameters - function(...args) {} - збір решти в масив
  • Named (через об'єкт) - function({name, age}) {}
function greet(name, age = 18, ...hobbies) {
  console.log(name, age, hobbies);
}
greet('John', 25, 'coding', 'gaming');
5. Що таке arguments в функціях?

Псевдомасив (array-like object) з усіма переданими аргументами. Доступний тільки в звичайних функціях (не в arrow).

function sum() {
  return Array.from(arguments).reduce((a, b) => a + b, 0);
}
sum(1, 2, 3);  // 6

Краще використовувати rest parameters ...args

6. Яким чином виконати function overriding?

JS не підтримує перевантаження функцій (overloading). При повторному оголошенні - остання версія перезаписує попередню.

Альтернативи:

  • Перевірка кількості/типу аргументів всередині функції
  • Використання rest parameters
  • Різні імена функцій
function process(data) {
  if (typeof data === 'number') { /* ... */ }
  if (Array.isArray(data)) { /* ... */ }
}
7. Що таке callback функція?

Функція, яка передається як аргумент в іншу функцію і викликається всередині неї.

function fetchData(callback) {
  setTimeout(() => {
    callback('data');
  }, 1000);
}

fetchData((result) => {
  console.log(result);  // 'data' через 1 сек
});
8. Що таке scope?

Область видимості - контекст, в якому визначені змінні і до яких з них є доступ.

Типи:

  • Global scope - доступно скрізь
  • Function scope - всередині функції
  • Block scope - всередині блоку {} (let, const)
9. З яким scope працюють функції в JS?

Lexical scope (статичний) - функція має доступ до змінних з місця свого оголошення, а не виклику.

let x = 10;
function outer() {
  let y = 20;
  function inner() {
    console.log(x, y);  // 10, 20 (доступ до зовнішніх scope)
  }
  return inner;
}
10. Що таке closure?

Механізм, коли функція "запам'ятовує" змінні зі свого зовнішнього scope, навіть після завершення зовнішньої функції.

function counter() {
  let count = 0;
  return function() {
    return ++count;
  };
}

const increment = counter();
increment();  // 1
increment();  // 2 (count "запам'ятався")
11. Для чого потрібний оператор return?

Повертає значення з функції і завершує її виконання.

Без return:

  • Функція поверне undefined
  • Використовується для функцій з побічними ефектами (console.log, DOM-маніпуляції)
function add(a, b) {
  return a + b;  // повертає результат
}

function log(msg) {
  console.log(msg);  // нічого не повертає (undefined)
}
03-exceptions (Винятки та обробка помилок)
1. Які є види помилок та які з них найскладніше виявити?

Синтаксичні (Syntax Errors): Помилки в коді, які виявляються при парсингу. Найлегше виявити.

Runtime Errors: Помилки під час виконання (TypeError, ReferenceError, RangeError).

Логічні (Logic Errors): Код працює, але дає неправильний результат. Найскладніше виявити, бо немає exception.

// Syntax Error
let x = ;  // SyntaxError

// Runtime Error
null.toString();  // TypeError

// Logic Error
function sum(a, b) {
  return a - b;  // має бути +, але помилки немає
}
2. Що таке debugging та як його використовувати в JS?

Процес пошуку та виправлення помилок у коді.

Інструменти:

  • console.log() - вивід значень
  • debugger; - зупинка виконання коду
  • DevTools браузера - breakpoints, step-by-step виконання
  • console.trace() - виведення stack trace
  • console.table() - відображення об'єктів у вигляді таблиці
function calculate(x) {
  debugger;  // зупинка для перевірки
  return x * 2;
}

console.log(calculate(5));
3. Що таке exceptions та для чого вони потрібні?

Exception (виняток): Об'єкт помилки, який генерується при виникненні проблеми під час виконання коду.

Для чого потрібні:

  • Обробка помилок без зупинки програми
  • Контроль над поведінкою програми при помилках
  • Передача інформації про помилку
  • Розділення нормальної логіки та обробки помилок
try {
  JSON.parse('invalid json');
} catch (error) {
  console.log('Помилка парсингу:', error.message);
}
4. В якій послідовності викликаються блоки try catch finally?
  1. try: Виконується завжди спочатку
  2. catch: Виконується ТІЛЬКИ якщо виникла помилка в try
  3. finally: Виконується ЗАВЖДИ, незалежно від помилки
try {
  console.log('1. try блок');
  throw new Error('Помилка!');
} catch (error) {
  console.log('2. catch блок');
} finally {
  console.log('3. finally блок (завжди)');
}

// Вивід: 1 → 2 → 3
5. Які є види exceptions?

Вбудовані типи помилок:

  • Error: Базовий тип помилки
  • SyntaxError: Синтаксична помилка
  • ReferenceError: Звернення до неіснуючої змінної
  • TypeError: Неправильний тип даних
  • RangeError: Значення поза допустимим діапазоном
  • URIError: Помилка в encodeURI/decodeURI
  • EvalError: Помилка eval() (рідко використовується)
// ReferenceError
console.log(x);  // x не оголошено

// TypeError
null.toString();

// RangeError
new Array(-1);
6. Як згенерувати exception в коді?

Використовувати оператор throw з об'єктом помилки.

// Простий exception
throw new Error('Щось пішло не так');

// З типом помилки
throw new TypeError('Неправильний тип');

// Власний exception
throw new Error('Користувач не знайдений');

// Можна кидати будь-яке значення
throw 'Помилка';
throw 404;
throw { code: 500, message: 'Server Error' };

// Приклад з перевіркою
function divide(a, b) {
  if (b === 0) {
    throw new Error('Ділення на нуль!');
  }
  return a / b;
}
7. Чи може бути, що блок finally не спрацює?

Майже завжди спрацює, але є виключення:

  • Аварійне завершення процесу (crash)
  • Нескінченний цикл в try/catch
  • process.exit() в Node.js
  • Фатальні системні помилки

В нормальних умовах finally завжди виконується, навіть якщо є return в try/catch.

function test() {
  try {
    return 'try';
  } finally {
    console.log('finally виконається перед return');
  }
}

test();  // виведе "finally виконається перед return", потім поверне 'try'
8. Що спільне та відмінне між операторами throw та return?

Спільне:

  • Обидва переривають виконання функції
  • Можуть передавати значення

Відмінне:

throw return
Генерує помилку (exception) Повертає нормальне значення
Призводить до catch блоку Нормальне завершення функції
Пробрасується вгору по call stack Залишається в поточній функції
Зупиняє виконання при помилці Нормальний потік виконання 
function withReturn() {
  return 'OK';  // нормальне повернення
}

function withThrow() {
  throw new Error('Помилка');  // exception
}

console.log(withReturn());  // 'OK'
console.log(withThrow());   // Uncaught Error (зупиняє виконання)
04-oop-classes
1. Що таке об'єкти в JS та що вони можуть містити в собі

Об'єкти — це колекції пар ключ-значення. Ключі — рядки або Symbols. Значення можуть бути будь-чим: примітиви, функції, інші об'єкти.

Об'єкт може містити:

  • Примітивні значення
  • Інші обʼєкти
  • Методи (функції як властивості)
  • Гетери/сетери
  • Symbol-властивості
2. Що таке ООП, які є принципи ООП та коротко опишіть кожний

ООП (Об'єктно-орієнтоване програмування): парадигма програмування, яка описує систему як набір об'єктів, кожен з яких має стан (властивості) та поведінку (методи).

Основні принципи ООП:

  • Інкапсуляція: приховування внутрішньої реалізації об'єкта, доступ до даних лише через публічні методи.
  • Наслідування: один клас може наслідувати властивості/методи іншого (батьківського) класу.
  • Поліморфізм: один і той самий метод веде себе по-різному в залежності від контексту (перевизначення в дочірньому класі).
  • Абстракція: використання лише суттєвих характеристик без деталізації реалізації.

📌 Приклад ООП на автівці (людською мовою):

  • Ти маєш клас Car — це шаблон: що має машина, як працює.
  • Коли ти створюєш new Car("BMW", "чорна") — ти створюєш обʼєкт.
  • В обʼєкта є властивості: марка, колір, швидкість.
  • І є методи: start(), brake(), honk().
  • Ти не знаєш як працює двигун — ти просто тиснеш "газ". Це — інкапсуляція.
  • Ти можеш створити ElectricCar extends Car — це наслідування.
  • У ElectricCar метод start() працює по-іншому — це поліморфізм.
  • Ти взаємодієш тільки з важливим (кермо, педалі), не знаючи про систему зчеплення — це абстракція.
3. Яким чином можна отримати всі властивості обʼєкта в JS
  • Object.keys(obj) — власні enumerable ключі
  • Object.values(obj) — власні enumerable значення
  • Object.entries(obj) — пари [key, value]
  • for...in — всі enumerable, включно з прототипом
  • Reflect.ownKeys(obj) — всі ключі (включно з символами)
4. Що таке методи та яка відмінність між функціями

Метод: функція, яка є властивістю обʼєкта.

Різниця: метод викликається в контексті обʼєкта, має доступ до this обʼєкта. Звичайна функція — ні.

5. Що таке this та на що він може вказувати

this — контекст виконання функції. Залежить від способу виклику:

  • У методі — посилається на обʼєкт
  • У звичайній функції — undefined (в strict) або window (не strict)
  • У стрілочній функції — бере this з зовнішнього контексту
  • В event listener — DOM-елемент
  • В класі — екземпляр класу
6. Як можна створити копію обʼєкта в JS

Поверхнева копія:

  • {...obj}
  • Object.assign({}, obj)
  • structuredClone(obj) — глибока копія (сучасно)
  • JSON.parse(JSON.stringify(obj)) — обхідний глибокий клон (із втратами)
7. Що таке клас та яка відмінність між обʼєктом

Клас: шаблон (синтаксичний цукор над функцією-конструктором) для створення обʼєктів.

Обʼєкт: конкретний екземпляр, створений з класу або вручну.

Клас не зберігає дані, лише описує структуру. Обʼєкт — має дані і поведінку.

8. Що таке конструктор та яке його призначення

Конструктор: спеціальний метод constructor() всередині класу, який викликається при створенні нового екземпляра через new.

Використовується для ініціалізації початкового стану обʼєкта.

9. Що таке get/set та яке їх призначення

get: метод для читання значення як властивості

set: метод для встановлення значення з додатковою логікою (наприклад, валідація)

Викликаються як звичайні властивості, без ().

10. Що таке валідація даних, де її краще виконувати

Валідація: перевірка коректності даних перед обробкою/збереженням.

Краще виконувати: у set, конструкторі або окремому методі.

11. Що таке статичні властивості класу, яка їх особливість

Властивості, які належать самому класу, а не екземпляру.

Оголошуються через static.

class User {
  static role = "admin";
}
console.log(User.role); // 'admin'
12. Як реалізовується приховування властивостей в класі, для чого це потрібно

Приховування (інкапсуляція): захист внутрішніх даних від прямого доступу/зміни.

Методи:

  • Приватні поля: #name
  • Локальні змінні через замикання (в функціях/модулях)
  • Конвенція підкреслення: _name (неформально)
13. Що таке super, для чого він використовується

super — виклик конструктора або методів батьківського класу з дочірнього.

  • В конструкторі: super(...args)
  • В методі: super.methodName()
14. Як реалізувати наслідування в JS, яке його призначення

За допомогою class Child extends Parent.

Призначення — повторне використання логіки та структуризація коду.

15. Як реалізувати поліморфізм в JS, яке його призначення

Одинаковий інтерфейс — різна реалізація.

Реалізується через перевизначення методів у дочірніх класах.

class Animal {
  speak() { console.log("..."); }
}
class Dog extends Animal {
  speak() { console.log("Woof"); }
}
16. Чи застосовується операція hoisting до класів в JS

Ні. Класові оголошення не піднімаються як функції. Вони потрапляють у TDZ.

new MyClass(); // ❌ ReferenceError
class MyClass {}
05-dom
1. Що таке DOM

DOM (Document Object Model): це обʼєктне представлення HTML-документа у вигляді дерева вузлів. Дає змогу взаємодіяти з елементами сторінки через JS.

2. Для чого потрібно взаємодіяти з DOM через JS

Щоб динамічно змінювати вміст, стилі, структуру або поведінку сторінки на основі дій користувача чи логіки програми.

3. Які є способи пошуку елемента на сторінці
  • document.getElementById()
  • document.getElementsByClassName()
  • document.getElementsByTagName()
  • document.querySelector() — перший, що підходить
  • document.querySelectorAll() — всі, як NodeList
4. В чому різниця між childNodes та children
  • childNodes — всі дочірні вузли (включно з текстовими, пробілами, коментарями)
  • children — тільки елементи (nodeType === 1)
5. Яка різниця між innerHTML та textContent
  • innerHTML — встановлює/отримує HTML-код всередині елемента
  • textContent — працює лише з текстом, без HTML-розмітки

innerHTML небезпечний — схильний до XSS-атак, якщо вставляти неперевірений контент.

6. В чому різниця між prepend та before
  • element.prepend() — додає вміст на початок всередину елемента
  • element.before() — додає зовні, перед самим елементом у DOM
7. Яким чином можна додати новий елемент на сторінку

Потрібно створити елемент і вставити його в DOM:

const el = document.createElement('div');
el.textContent = "Привіт";
document.body.appendChild(el);

Також можна використовувати prepend(), append(), before(), after().

8. Яким чином можна змінити атрибути елемента через JS
  • element.setAttribute('name', 'value')
  • element.getAttribute('name')
  • element.removeAttribute('name')
  • Альтернатива: напряму через властивості: element.id = "main"
9. Яким чином можна змінити CSS класи елемента через JS
  • element.classList.add('className')
  • element.classList.remove('className')
  • element.classList.toggle('className')
  • element.classList.contains('className')
10. Яким чином можна видалити елемент зі сторінки
  • element.remove() — напряму
  • parent.removeChild(child) — старіший спосіб
06-bom-events
1. Що таке BOM

BOM (Browser Object Model): набір об'єктів, через які JS взаємодіє з браузером (не з DOM).

2. Яка різниця між BOM та DOM
  • DOM: доступ до структури HTML-документа
  • BOM: доступ до функціоналу браузера (вікна, вкладки, навігація, гео тощо)
3. Які основні обʼєкти доступні від BOM, коротко опишіть їх призначення
  • window — глобальний обʼєкт, корінь для всіх BOM-обʼєктів
  • navigator — інформація про браузер, платформу, онлайн-статус
  • screen — дані про розмір екрану
  • location — URL сторінки, навігація по історії
  • history — історія відвідувань (навігація вперед/назад)
  • alert, prompt, confirm — взаємодія з користувачем
4. Яким чином можна отримати геолокацію девайсу

Через navigator.geolocation.getCurrentPosition()

navigator.geolocation.getCurrentPosition((pos) => {
  console.log(pos.coords.latitude, pos.coords.longitude);
});
5. Що таке подія, які бувають типи подій

Подія: сигнал, що щось відбулось (клік, навідення, введення, завантаження тощо)

Типи подій:

  • Миші: click, dblclick, mouseover, mouseenter, etc.
  • Клавіатура: keydown, keyup, keypress
  • Форми: submit, change, input
  • Вікно: resize, scroll, load, unload
  • Touch, Drag & Drop, Clipboard, Media, Web API Events
6. Яка різниця між mouseenter та mousehover подіями

mouseenter — спрацьовує один раз при входженні миші в елемент (не спливає).

mouseover — спрацьовує при кожному входженні миші в елемент або його дочірні.

7. Які є варіанти створення обробника на певну подію
  • Через HTML: <button onclick="fn()"> ❌ поганий стиль
  • Через властивість: element.onclick = fn — перезаписує
  • Через addEventListener: підтримує кілька обробників, зняття, опції
8. Яка різниця між onclick та addEventListener("click", callback)
  • onclick — можна лише один обробник, перезаписує попередній
  • addEventListener — можна додати багато обробників на одну подію
  • addEventListener має більше опцій: once, capture, passive
9. Що таке спливання подій та як його зупинити

Спливання (bubbling): подія передається від дочірнього елемента вгору по DOM (до document).

Щоб зупинити: event.stopPropagation()

element.addEventListener("click", function(e) {
  e.stopPropagation();
});
10. Що таке event.target та яка відмінність між event.currentTarget
  • event.target — елемент, на якому фактично сталася подія (найглибший вузол)
  • event.currentTarget — елемент, на якому зараз виконується обробник

Вони відрізняються, якщо обробник призначений на батьківський елемент (делегування).

Regular Expressions

1. Для яких завдань використовуються регулярні вирази

Для пошуку, валідації, заміни тексту за шаблоном (наприклад: email, номер телефону, формат дати).

2. Яка різниця між RegExp.test() та RegExp.exec()
  • test(str) — повертає true/false, чи є збіг
  • exec(str) — повертає масив з результатом або null, зберігає позицію останнього пошуку (з прапором g)
3. Які є квантифікатори в регулярних виразах
  • * — 0 або більше
  • + — 1 або більше
  • ? — 0 або 1
  • {n} — рівно n
  • {n,} — n або більше
  • {n,m} — від n до m
4. Що означають символи ^ та $ в регулярному виразі
  • ^ — початок рядка
  • $ — кінець рядка
  • Використовуються для точного співпадіння в межах рядка
5. Які є прапори в регулярних виразах
  • g — global, всі входження
  • i — ignore case
  • m — multi-line (змінює поведінку ^ і $)
  • u — підтримка юнікоду
  • s — дозволяє . матчити новий рядок
  • y — sticky (збіг тільки з поточної позиції)
6. Що виконує прапор m

Робить ^ та $ чутливими до рядків у багато-рядковому тексті, а не до всього рядка.

7. Які є метасимволи в регулярних виразах
  • \d — цифра
  • \w — літера, цифра або підкреслення
  • \s — пробіл
  • . — будь-який символ (крім переводу рядка)
  • \b — границя слова
  • \ — екранування спецсимволів
8. Що таке групування, для чого воно може бути використане

Групування (...) дозволяє:

  • Застосовувати квантифікатори до групи
  • Отримувати підрядки (capturing groups)
  • Повторно використовувати частини збігу
9. Яким чином можна отримати всі входження в тексті певного регулярного виразу

Через:

  • str.match(/.../g)
  • Або цикл з RegExp.exec() + прапор g
10. Яким чином можна замінити в тексті частину, яка відповідає регулярному виразу

Метод str.replace(regex, replacement)

"abc123".replace(/\d+/g, "#") // "abc#"

  • Копіювання масивів або обʼєктів

  • Map — зберігає пари ключ-значення, де ключі можуть бути будь-якими

09-promises
1. Що таке асинхронність та багатопоточність, що спільного та відмінного
  • Асинхронність — не блокує виконання, дозволяє відкладені операції (наприклад, таймери, запити)
  • Багатопоточність — одночасне виконання кількох потоків (у JS тільки через Web Workers)
  • JS — однопоточний, асинхронний. Використовує event loop для відкладених дій
2. Як реалізувати асинхронність в JS
  • Через setTimeout, setInterval
  • Promise + .then/.catch
  • async/await
  • Event listeners / callbacks
  • fetch, AJAX, Web APIs
3. Що таке Promise, як її можна запустити та завершити

Promise — об'єкт для відстеження завершення асинхронної операції.

Створюється через new Promise((resolve, reject) => {...})

Завершується викликом:

  • resolve(value) — успіх
  • reject(error) — помилка
4. В яких станах може бути Promise
  • pending — очікує
  • fulfilled — успішно завершено
  • rejected — завершено з помилкою
5. Як обробити успішне завершення Promise

За допомогою .then()

fetch(url)
  .then(response => response.json())
  .then(data => console.log(data));
6. Як обробити не успішне завершення Promise
  • .catch() — обробляє помилки
  • .then(null, errorHandler) — теж працює, але менш зручно
  • .finally() — виконується завжди (успіх або помилка)
7. Що таке async та await, яке їх призначення
  • async — функція завжди повертає Promise
  • await — ставить «на паузу» виконання до завершення Promise
  • Дозволяє писати асинхронний код в синхронному стилі
8. Які основні відмінності обробки Promise з використанням async/await
  • then/catch — працює з ланцюжками, але менш читабельно при складній логіці
  • async/await — чистіший код, простіше читати
  • Обробку помилок потрібно робити через try/catch
  • await можна використовувати лише всередині async-функції

module.exports = { add };

const math = require('./math');

  • Серверний: виконується на сервері, працює з базами даних, авторизацією, API

Що таке Long Polling (тривале опитування)?

Long Polling — техніка для отримання даних з сервера у режимі "майже в реальному часі", коли WebSocket ще не завезли або не хочеться мучитись із ними.

  • Клієнт надсилає HTTP-запит до сервера.
  • Сервер не відповідає одразу, а чекає, поки зʼявляться нові дані.
  • Як тільки є дані — сервер відповідає, і клієнт одразу відправляє новий запит.
  • І так по колу, як вечірня розмова з дуже мовчазним співрозмовником.

Використовується коли: потрібне оновлення в реальному часі, але WebSocket — це занадто.

Мінуси: тримає з'єднання, створює навантаження на сервер (бо постійні запити).

Що таке каррінг у JavaScript?

Каррінг (Currying) — це техніка перетворення функції з багатьма аргументами у послідовність функцій з одним аргументом.

// Звичайна функція
function add(a, b) {
  return a + b;
}

// Каррінгована версія
function curriedAdd(a) {
  return function(b) {
    return a + b;
  };
}

curriedAdd(2)(3); // 5

Навіщо це треба:

  • Зручно створювати частково застосовані функції (partial functions)
  • Можна фіксувати перші аргументи, передаючи лише решту
  • Полегшує композицію функцій у функціональному стилі
Що таке WebSocket і для чого він використовується?

WebSocket — це протокол для постійного двостороннього з’єднання між клієнтом і сервером у реальному часі.

Навіщо: миттєва передача даних для чатів, бірж, live-апдейтів, ігор.

  • З’єднання залишається відкритим — дані можна надсилати в обидві сторони без нових запитів.
  • Не має кросдоменних обмежень (на відміну від AJAX).
  • Підтримує як текст, так і двійкові дані.
  • Працює майже в усіх сучасних браузерах.
  • Простий API, але не включає автоматичне перепідключення чи аутентифікацію.

Основні методи:

  • socket.send(data) — надсилає дані
  • socket.close([code], [reason]) — закриває з’єднання

Події:

  • open — з’єднання встановлено
  • message — отримано дані
  • error — сталася помилка
  • close — з’єднання закрито

Типове використання:

const socket = new WebSocket("wss://ws.example.com");

socket.onopen = () => socket.send("Привіт сервере!");
socket.onmessage = e => console.log("Прийшло:", e.data);
socket.onerror = e => console.error("Помилка:", e);
socket.onclose = () => console.log("З’єднання закрито");

Інтеграція: WebSocket-сервер часто запускають окремо від HTTP-сервера (наприклад: wss://ws.site.com), але вони можуть використовувати спільну базу даних.

Що таке XMLHttpRequest і як з ним працювати?

XMLHttpRequest — вбудований об'єкт у браузері для відправки HTTP-запитів без перезавантаження сторінки.

Сьогодні частіше використовують fetch(), але XMLHttpRequest все ще актуальний для підтримки старого коду.

Кроки для створення запиту:

  1. Створити об'єкт: let xhr = new XMLHttpRequest();
  2. Налаштувати запит: xhr.open(method, url, async)
  3. Надіслати: xhr.send([body])

Основні події:

  • load — запит завершився (навіть з помилкою HTTP, як 404)
  • error — запит не відбувся (наприклад, немає з'єднання)
  • progress — йде завантаження (можна отримати обсяг)
// GET-запит
let xhr = new XMLHttpRequest();
xhr.open('GET', '/data');
xhr.send();

xhr.onload = function() {
  if (xhr.status === 200) {
    console.log("Отримано:", xhr.response);
  } else {
    console.log(`Помилка: ${xhr.status}`);
  }
};

Налаштування POST із JSON:

let data = JSON.stringify({ name: "Іван", age: 25 });

let xhr = new XMLHttpRequest();
xhr.open("POST", "/submit");
xhr.setRequestHeader("Content-Type", "application/json");
xhr.send(data);

Додаткові події:

  • loadstart — початок запиту
  • abort — викликано xhr.abort()
  • timeout — запит перевищив час очікування
  • loadend — завжди викликається після load/error/abort/timeout
Що таке CORS і навіщо він потрібен?

CORS (Cross-Origin Resource Sharing) — це механізм безпеки браузера, який блокує запити між різними доменами, якщо сервер не дозволяє це явно.

Навіщо: захищає користувача від крадіжки даних (наприклад, коли один сайт намагається звернутись до API іншого сайту).

Як працює: браузер додає спецзаголовки, а сервер у відповідь має відправити:

  • Access-Control-Allow-Origin — дозволений домен (або * для всіх)
  • Access-Control-Allow-Methods — які методи дозволені (GET, POST, тощо)

Інакше: браузер заблокує відповідь ще до того, як твій код щось отримає.

Що таке URL-об’єкти в JavaScript?

URL — це вбудований об’єкт для зручної роботи з адресами (URL).

Створення: const url = new URL("https://site.com/page?user=Іван#контакти")

Основні властивості:

  • url.protocol → "https:"
  • url.hostname → "site.com"
  • url.pathname → "/page"
  • url.searchParams.get("user") → "Іван"
  • url.hash → "#контакти"

Навіщо: легко читати та змінювати частини URL, не граючись зі строками.

Що таке FormData і для чого він використовується?

FormData — це об'єкт для збирання даних з HTML-форми для відправки на сервер (зазвичай з fetch або XHR).

Автоматично збирає: всі name-поля з форми, включаючи файли.

Формат відправки: multipart/form-data — не треба вказувати заголовки вручну.

Приклад:

let form = document.querySelector("form");
let data = new FormData(form);

fetch("/submit", {
  method: "POST",
  body: data
});

Можна додати поля вручну: formData.append("key", "value")

Яка різниця між WebSocket та Server-Sent Events (SSE)?

WebSocket — двостороннє з'єднання: клієнт і сервер можуть надсилати дані в будь-який момент.

SSE (Server-Sent Events) — тільки сервер → клієнт: клієнт "підписується", сервер надсилає оновлення.

Порівняння:

  • WebSocket: повний дуплекс (двостороння комунікація), підходить для чатів, онлайн-ігор, бірж тощо.
  • SSE: односторонній канал (тільки сервер → клієнт), ідеально для live-оновлень, новин, стрічок подій.
  • SSE працює лише по HTTP, WebSocket — по ws:// або wss://
  • SSE простіше реалізувати, бо це просто HTTP-з'єднання з особливим заголовком.
  • WebSocket потребує більше контролю: підключення, reconnection, ping/pong, безпека.

Коли що використовувати:

  • Потрібна двостороння комунікація? → WebSocket
  • Потрібно лише пушити дані з сервера? → SSE
Що таке Server-Sent Events (SSE)?

SSE — це технологія, де сервер сам надсилає дані клієнту по HTTP, без запитів з боку клієнта.

  • Працює тільки сервер → клієнт
  • Підходить для оновлень у реальному часі (чат, стрічка новин)
  • Працює через EventSource
const source = new EventSource('/events');
source.onmessage = (e) => console.log(e.data);
