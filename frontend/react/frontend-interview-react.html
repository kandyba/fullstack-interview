Запитання та відповіді для технічної співбесіди (React)

1. Що робить метод setState()?

У React state — це об’єкт, який зберігає дані, що можуть змінюватися під час життєвого циклу компонента. Щоб оновити ці дані, необхідно викликати функцію setState().

Метод setState() повідомляє React про зміну стану. Після цього компонент рендериться повторно, і ми бачимо в браузері оновлені дані.

this.setState({ count: this.state.count + 1 });

✅ Важливо: оновлення стану відбувається асинхронно, тому не можна розраховувати на миттєву зміну this.state після виклику setState().

2. Що таке Virtual DOM?

Virtual DOM — це легка копія реального DOM-дерева. React працює не безпосередньо з DOM, а з його віртуальним відображенням у пам’яті.

Коли стан або пропси компонента змінюються, React спочатку оновлює Virtual DOM, порівнює його з попереднім станом (алгоритм diffing), і лише потім застосовує мінімальні зміни до реального DOM.

React.createElement('div', null, 'Привіт, світ!');

✅ Це підхід, який робить React швидким — зміни застосовуються лише там, де потрібно, без повного перерендерингу сторінки.

3. Як відмалювати масив елементів у React?

Для відображення списку елементів у React зазвичай використовують метод масиву map(). React вимагає, щоб кожен елемент у списку мав унікальний ключ key, який допомагає відслідковувати зміни.

const users = ['Anna', 'Mark', 'Olga'];

return (
  <ul>
    {users.map((user, index) => (
      <li key={index}>{user}</li>
    ))}
  </ul>
);

✅ Ключі допомагають React оптимізувати оновлення — він розуміє, які елементи змінилися, додалися або видалилися.

4. У чому різниця між контрольованими та неконтрольованими компонентами?

У HTML елементи форм, такі як <input>, <textarea> чи <select>, мають власний стан. У React змінюване значення зазвичай зберігається у state компонента й оновлюється через setState().

🔹 Контрольований компонент:

React повністю контролює значення поля. Дані зберігаються в state, а зміни обробляються через подію onChange.

class Form extends React.Component {
  state = { name: '' };

  handleChange = (e) => this.setState({ name: e.target.value });

  render() {
    return (
      <input type="text" value={this.state.name} onChange={this.handleChange} />
    );
  }
}

🔹 Неконтрольований компонент:

Значення зчитується напряму з DOM за допомогою ref, без збереження у state.

class Form extends React.Component {
  inputRef = React.createRef();

  handleSubmit = () => {
    alert(this.inputRef.current.value);
  }

  render() {
    return (
      <input type="text" ref={this.inputRef} />
    );
  }
}

✅ Контрольовані компоненти краще для складних форм, а неконтрольовані — простіші, коли потрібно мінімум логіки.

5. Які існують методи життєвого циклу компонента в React?

У React є кілька етапів життєвого циклу компонента. Для класових компонентів вони визначаються спеціальними методами, а у функціональних реалізуються через useEffect().

🔹 Основні етапи:

  • Ініціалізація (Initialization): створення початкового стану state і встановлення початкових пропсів.
  • Монтування (Mounting): додавання компонента в DOM. Основні методи: constructor(), componentDidMount().
  • Оновлення (Updating): повторний рендер після зміни стану або пропсів. Методи: shouldComponentUpdate(), componentDidUpdate().
  • Розмонтування (Unmounting): видалення компонента з DOM. Метод: componentWillUnmount().

✅ У сучасних функціональних компонентах усі ці стадії можна реалізувати за допомогою хука useEffect().

import React, { useEffect } from "react";

function Example() {
  useEffect(() => {
    console.log("Компонент змонтовано");
    return () => console.log("Компонент розмонтовується");
  }, []);

  return <div>Привіт, React!</div>;
}
6. Які React-хуки ви знаєте і використовуєте?

Хук (hook) — це спеціальна функція, яка дозволяє використовувати можливості React (стан, життєвий цикл, контекст тощо) у функціональних компонентах.

🔹 Основні хуки React:

  • useState — для зберігання стану.
  • useEffect — для побічних ефектів (запити, підписки, таймери).
  • useContext — для доступу до контексту без пропсів.
  • useRef — для збереження змінних між рендерами або доступу до DOM-елемента.
  • useMemo — для мемоізації обчислень.
  • useCallback — для мемоізації функцій, щоб уникати зайвих ререндерів.
import { useState, useEffect } from "react";

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => setSeconds(s => s + 1), 1000);
    return () => clearInterval(interval);
  }, []);

  return <p>Минуло: {seconds} секунд</p>;
}

✅ Хуки завжди викликаються лише всередині функціональних компонентів або інших хуків (не в умовах, не в циклах).

7. Які особливості хука useState()?

useState() — це хук, який повертає пару значень: поточний стан і функцію для його оновлення.

const [count, setCount] = useState(0);

<button onClick={() => setCount(count + 1)}>Додати</button>

✅ Особливості:

  • Під час першого рендера useState встановлює початкове значення.
  • Оновлення стану асинхронне — значення не змінюється миттєво.
  • Функцію setState можна викликати з callback’ом, щоб оновити значення на основі попереднього стану:
setCount(prev => prev + 1);

🔹 Після виклику setState React перерендерює компонент з оновленим станом.

8. Які особливості використання хука useEffect()?

useEffect() дозволяє виконувати побічні ефекти у функціональних компонентах. Побічними ефектами вважаються дії, що впливають на зовнішній світ — запити на сервер, робота з DOM, підписки, таймери, логування тощо.

React не дозволяє виконувати такі дії безпосередньо у тілі компонента, тому всі ефекти слід виконувати всередині useEffect().

Функція, передана в useEffect(), виконується після того, як React оновить DOM. Якщо вказати масив залежностей другим аргументом, ефект буде викликатись лише тоді, коли змінюється одна з цих залежностей.

import { useEffect, useState } from "react";

function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => setCount(c => c + 1), 1000);
    return () => clearInterval(interval); // очищення
  }, []);

  return <p>Минуло {count} секунд</p>;
}

✅ Порада: уникай запуску асинхронних ефектів без очищення — це може викликати витоки пам’яті.

9. Як відстежити розмонтування функціонального компонента?

Щоб виконати дію під час видалення компонента з DOM (тобто при “розмонтуванні”), потрібно повернути функцію очищення з useEffect().

Ця функція викликається перед тим, як компонент буде знищено або перед наступним виконанням ефекту.

useEffect(() => {
  const id = setInterval(() => console.log("Працює..."), 1000);

  return () => {
    clearInterval(id);
    console.log("Компонент розмонтовується");
  };
}, []);

✅ Це корисно для відписки від подій, очищення таймерів, скасування запитів — усе, що не повинно існувати після видалення компонента.

10. Що таке state-менеджер і який ви використовуєте?

State-менеджер — це інструмент для централізованого управління станом додатка. Він допомагає відокремити бізнес-логіку від компонентів і полегшує передачу даних між ними.

Без state-менеджера, коли багато компонентів обмінюються станом через пропси, код стає важким для підтримки. Тому часто використовують глобальні сховища, до яких можна звертатися з будь-якого місця.

🔹 Найпоширеніші state-менеджери:

  • Redux — класичний варіант із предбачуваним потоком даних.
  • MobX — реактивний підхід, зручний для великих UI з частими змінами стану.
  • Zustand — легкий і сучасний менеджер стану без надлишкової бюрократії Redux.
  • Recoil — гнучкий менеджер від Meta, добре інтегрується з React 18.

✅ У невеликих проектах достатньо локального стану з useState або useReducer.

11. У яких випадках краще використовувати локальний стан, а у яких — глобальний?

Локальний стан (через useState) зручно використовувати, коли дані стосуються лише одного компонента і не потрібно передавати їх іншим.

Приклади локального стану:

  • Видимість модалки (isOpen).
  • Введене значення інпуту.
  • Активна вкладка або стан кнопки.

Глобальний стан потрібен тоді, коли одні й ті самі дані мають використовуватись у багатьох місцях застосунку. Наприклад — інформація про користувача, тема (light/dark), або товари в кошику.

✅ Рекомендація: якщо стан починає передаватися через кілька рівнів компонентів, варто підняти його вище або винести у глобальний state (через Context API або Redux).

/* локальний state */
const [isOpen, setIsOpen] = useState(false);

/* глобальний state через Context */
const { theme } = useContext(ThemeContext);
12. Що таке reducer у Redux і які параметри він приймає?

Reducer — це чиста функція, яка приймає два параметри: state і action, та повертає новий стан на основі отриманої дії.

const initialState = { page: 1 };

function reducer(state = initialState, action) {
  switch (action.type) {
    case "SET_PAGE":
      return { ...state, page: action.payload };
    default:
      return state;
  }
}

✅ Reducer не змінює існуючий об’єкт стану напряму, а створює новий.

13. Що таке action у Redux і як він змінює стан?

Action — це звичайний JavaScript-об’єкт, який описує, що має відбутись. Обов’язково містить поле type, яке визначає тип дії.

const setPage = {
  type: "SET_PAGE",
  payload: 3
};

Щоб змінити стан, викликається функція dispatch(action), яка передає action у reducer.

14. Що таке JSX?

JSX — це синтаксичне розширення JavaScript, яке дозволяє писати HTML-подібний код усередині JS. Під капотом JSX компілюється у виклики React.createElement().

const element = <h1 className="greeting">Привіт, React!</h1>;

✅ JSX робить код більш наочним і спрощує роботу з компонентами.

15. Що таке props?

Props — це властивості, які передаються з батьківського компонента в дочірній. Вони доступні лише для читання та не можуть бути змінені в середині дочірнього компонента.

function Welcome(props) {
  return <h2>Привіт, {props.name}!</h2>;
}

✅ Для передачі даних «зверху вниз» props — основний механізм у React.

16. У чому різниця між прямим оновленням стану та колбеком у setState/useState?

React може об’єднувати кілька викликів setState, тому при використанні колбека гарантується робота з актуальним станом.

// ❌ Потенційно неправильне оновлення
setCount(count + 1);

// ✅ Безпечно з колбеком
setCount(prev => prev + 1);

✅ Використовуй колбек, коли нове значення залежить від попереднього.

17. Для чого потрібен useMemo і коли його використовують?

useMemo() використовується для мемоізації результатів складних обчислень. Це оптимізація, яка запобігає повторним обчисленням при кожному рендері.

const expensiveValue = useMemo(() => compute(a, b), [a, b]);

✅ Використовуй useMemo для оптимізації продуктивності при великих обчисленнях або при передачі мемоізованих значень у дочірні компоненти.

18. Для чого потрібен useCallback і коли його використовують?

useCallback() повертає мемоізовану версію функції, яка змінюється лише при зміні залежностей. Це запобігає створенню нової функції при кожному рендері.

const handleClick = useCallback(() => {
  console.log('Клік!');
}, []);

✅ Корисно при передачі колбеків у дочірні компоненти, обгорнуті у React.memo().

19. Для чого потрібен useContext?

useContext() дозволяє отримувати значення з контексту без необхідності передавати props через усі рівні компонентів.

const ThemeContext = createContext();

function Button() {
  const theme = useContext(ThemeContext);
  return <button style={{ color: theme.color }}>Натисни</button>;
}

✅ Контекст зручний для тем, локалізації, авторизації тощо.

20. Для чого потрібен useRef?

useRef() створює змінний об’єкт, який зберігає значення між рендерами, не викликаючи повторного рендеру.

const inputRef = useRef(null);

<input ref={inputRef} />
<button onClick={() => inputRef.current.focus()}>Фокус</button>

✅ Найчастіше використовується для доступу до DOM-елементів або збереження будь-якого мутованого значення.

21. Для чого потрібен React.memo?

React.memo() — це компонент вищого порядку, який запам’ятовує результат рендера. Якщо props не змінились, React не буде повторно рендерити компонент.

const User = React.memo(({ name }) => <p>{name}</p>);

✅ Використовується для оптимізації продуктивності при незмінних props.

22. Що таке React Fiber?

React Fiber — це новий механізм рендерингу (з версії React 16), який робить процес оновлення DOM інкрементальним.

Його мета — підвищити продуктивність React при виконанні анімацій, складних переходів та великої кількості оновлень. Fiber дозволяє розділяти роботу на дрібні частини і виконувати її поступово, не блокуючи інтерфейс.

23. Що таке React Fragment?

Fragment дозволяє групувати кілька елементів без створення додаткового DOM-вузла.

return (
  <>
    <Header />
    <Content />
  </>
);

✅ Зручно використовувати, щоб уникнути зайвих контейнерів <div> у верстці.

24. Що таке React Reconciliation?

Reconciliation — це алгоритм React, який порівнює старе й нове DOM-дерево, щоб визначити мінімальні зміни для оновлення інтерфейсу.

React ефективно оновлює лише ті частини DOM, які дійсно змінились.

25. Навіщо потрібні ключі (key) у списках?

Ключі допомагають React ідентифікувати елементи, що змінились, додались або були видалені.

{items.map(item => <li key={item.id}>{item.text}</li>)}

✅ Використовуй унікальні id — це допомагає React уникати зайвих ререндерів.

26. Як виконувати асинхронні дії (actions) у Redux?

Для асинхронних дій у Redux використовують middleware redux-thunk. Він дозволяє action creator’ам повертати функцію замість об’єкта.

export const fetchTodos = () => async dispatch => {
  dispatch({ type: "TODOS_START" });
  try {
    const res = await fetch("/api/todos");
    const data = await res.json();
    dispatch({ type: "TODOS_SUCCESS", payload: data });
  } catch (e) {
    dispatch({ type: "TODOS_ERROR", error: e });
  }
};
27. Як відмалювати елемент за умовою?

Можна використовувати логічні оператори (&&), тернарні вирази або return усередині умов.

{isLoggedIn ? <LogoutButton /> : <LoginButton />}
{count > 0 && <p>Повідомлень: {count}</p>}
28. Як відстежити зміну поля об’єкта у функціональному компоненті?

Для цього використовують useEffect() з масивом залежностей, у якому вказують конкретне поле.

useEffect(() => {
  console.log(user.name);
}, [user.name]);
29. Як отримати доступ до DOM-елемента в React?

Можна використати useRef() або React.createRef().

const inputRef = useRef(null);

<input ref={inputRef} />
<button onClick={() => inputRef.current.focus()}>Фокус</button>

🟦 Middle рівень

30. Що таке чиста функція і чому React вимагає чистоти компонентів?

Чиста функція — це така, яка при однакових аргументах завжди повертає однаковий результат і не має побічних ефектів.

  • Не змінює зовнішні змінні або об’єкти.
  • Не взаємодіє з DOM чи API безпосередньо.
  • Повертає значення лише через return.
function add(a, b) { return a + b } // ✅ чиста
let total = 0;
function addToTotal(a) { total += a } // ❌ не чиста

React-компоненти мають бути чистими, щоб React міг передбачити результат рендеру.

31. У чому різниця між контрольованими та неконтрольованими компонентами?

Контрольовані компоненти — це ті, де значення елементів форми зберігаються у state, і зміни керуються React.

Неконтрольовані компоненти зберігають значення напряму у DOM (через ref).

// Контрольований
<input value={name} onChange={e => setName(e.target.value)} />

// Неконтрольований
const inputRef = useRef(null);
<input ref={inputRef} />
32. У чому різниця між mounting і rendering у React?
  • Mounting — створення компонента і вставлення його у DOM вперше.
  • Rendering — виклик функції компонента для створення його віртуального представлення (JSX).

Кожен mounting включає rendering, але не кожен rendering означає нове монтування.

33. Чому не можна викликати хуки в умовах чи циклах?

React покладається на порядок виклику хуків, щоб правильно співставляти стани між рендерами. Якщо викликати хук умовно — порядок зламається, і React втратить зв’язок між станами.

✅ Викликай хуки тільки:

  • на верхньому рівні компонента
  • або в custom hook’ах, теж без умов
34. Що таке dependency array у useEffect / useMemo / useCallback?

Це список залежностей, при зміні яких React повторно виконує ефект або обчислення.

useEffect(() => {
  console.log("Змінено name або age");
}, [name, age]);

Якщо передати порожній масив — ефект виконається лише один раз (аналог componentDidMount).

35. Як можна оптимізувати продуктивність React-додатка?
  • Мемоізація компонентів (React.memo).
  • Мемоізація значень і колбеків (useMemo, useCallback).
  • Розділення коду (React.lazy + Suspense).
  • Уникання зайвих перерендерів через правильні key.
  • Профілювання за допомогою React Developer Tools.
36. Що таке HOC (Higher-Order Component)?

Це функція, яка приймає компонент і повертає новий компонент із додатковою логікою або пропсами.

function withLogger(Wrapped) {
  return (props) => {
    console.log('Рендер', Wrapped.name);
    return <Wrapped {...props} />;
  }
}
37. Що таке Render Props?

Це патерн, коли компонент приймає функцію як prop, і ця функція визначає, що потрібно відрендерити.

<Mouse render={pos => <p>X: {pos.x}, Y: {pos.y}</p>} />
38. Що таке React.lazy та Suspense?

React.lazy() дозволяє завантажувати компонент лише тоді, коли він потрібен. Suspense показує запасний контент під час завантаження.

const About = React.lazy(() => import('./About'));

<Suspense fallback={<p>Завантаження...</p>}>
  <About />
</Suspense>
39. Що таке Error Boundary?

Це компонент, який перехоплює помилки у дочірніх компонентах і показує fallback UI.

Може бути лише класовим компонентом із методом componentDidCatch.

40. Що таке StrictMode і навіщо він потрібен?

StrictMode — це інструмент для виявлення потенційних проблем у коді під час розробки. Він активує додаткові перевірки, наприклад, подвійний рендер компонентів для виявлення побічних ефектів.

🟩 Middle+ рівень

41. Як можна поєднати Context і Reducer без Redux?

Комбінація useReducer + useContext дозволяє створити легку альтернативу Redux.

const AppContext = createContext();

function AppProvider({ children }) {
  const [state, dispatch] = useReducer(reducer, initialState);
  return <AppContext.Provider value={{ state, dispatch }}>
    {children}
  </AppContext.Provider>
}
42. Що таке автоматичний batching у React 18?

React 18 об’єднує кілька оновлень стану в один рендер, щоб зменшити кількість перерендерів і підвищити ефективність.

43. Що таке hydration у SSR?

Hydration — це процес, коли React «прив’язує» свій JavaScript до вже згенерованої HTML-сторінки після SSR. Це дозволяє сторінці стати інтерактивною без повторного рендерингу з нуля.

44. Що таке forwardRef і навіщо він потрібен?

forwardRef дозволяє передавати ref із батьківського компонента у дочірній.

const Input = forwardRef((props, ref) => <input ref={ref} {...props} />);
45. Що таке SyntheticEvent у React?

SyntheticEvent — це обгортка над нативними подіями браузера, яка забезпечує кросбраузерну сумісність і оптимізацію React.

46. Що таке React Portal і навіщо його використовують?

Portals дозволяють рендерити дочірні елементи у DOM-вузол поза межами батьківського компонента.

ReactDOM.createPortal(children, document.getElementById('modal-root'))

✅ Використовується для модалок, тултипів і контекстних меню.

47. Що таке Concurrent Rendering?

Це механізм React 18, який дозволяє виконувати оновлення з різними пріоритетами, щоб інтерфейс залишався плавним навіть під час важких обчислень.

48. Що таке props.children?

props.children — це все, що передано між відкривальним і закривальним тегом компонента.

<Card>
  <p>Контент усередині</p>
</Card>
49. Що таке Code Splitting і як його реалізувати?

Це розбиття JavaScript на частини, щоб завантажувати код поступово. Реалізується за допомогою React.lazy і Suspense.

50. У чому різниця між SSR та CSR?
  • CSR (Client-Side Rendering) — HTML генерується у браузері після завантаження JS.
  • SSR (Server-Side Rendering) — HTML створюється на сервері перед відправкою клієнту.

SSR покращує SEO та швидкість початкового відображення.
