Запитання та відповіді для технічної співбесіди (CSS, SCSS)

1. Що таке doctype? Навіщо використовується?

DOCTYPE — це інструкція, що вказує браузеру, який тип документа обробляти. Вона допомагає правильно інтерпретувати HTML та CSS. Найбільш поширений DOCTYPE для HTML5 виглядає так:

<!DOCTYPE html>
2. Що таке семантика в HTML?

Семантика в HTML — це підхід до використання тегів відповідно до їхнього значення. Семантичні елементи, такі як <article>, <section> або <header>, покращують доступність сторінки та полегшують пошукову оптимізацію.

3. Опишіть базову структуру HTML сторінки.

Базова структура HTML сторінки включає такі елементи:

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Назва сторінки</title>
  </head>
  <body>
    <h1>Заголовок</h1>
    <p>Текст контенту</p>
  </body>
</html>
4. Як зробити центрування елементів у CSS?

Центрування елементів у CSS можна здійснити кількома способами в залежності від типу елемента та його контексту:

1. За допомогою Flexbox

Flexbox дозволяє легко центрувати елементи по горизонталі і вертикалі:

.container {
  display: flex;
  justify-content: center; /* по горизонталі */
  align-items: center; /* по вертикалі */
  height: 100vh;
}

2. За допомогою Grid

CSS Grid дає ще простіший спосіб центрування, використовуючи властивості place-items:

.container {
  display: grid;
  place-items: center;
  height: 100vh;
}

3. Використання margin: auto

Для блочних елементів, що мають фіксовану ширину, можна використовувати margin: auto для горизонтального центрування:

.container {
  width: 300px;
  margin: 0 auto;
}

4. Позиціонування (absolute) та transform

Якщо елемент має позиціонування absolute, можна використати transform для центрування:

.container {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

5. Табличне вирівнювання

Цей метод старіший, але все ще іноді використовується. Для цього елемент має бути в таблиці або в елементі з властивістю display: table-cell:

.container {
  display: table-cell;
  text-align: center;
  vertical-align: middle;
  height: 100vh;
}

6. Для вирівнювання по вертикалі з флексом без використання align-items

Інший спосіб для вирівнювання елементів по вертикалі — використання лише margin:

.container {
  height: 100vh;
}
.center {
  margin-top: auto;
  margin-bottom: auto;
}
5. Селектори типу div p, div > p.

Селектор div p вибирає всі елементи <p>, які знаходяться всередині елемента <div>, на будь-якому рівні вкладеності. Селектор div > p вибирає тільки ті елементи <p>, які є безпосередньо нащадками елемента <div>.

6. Типи позиціонування в CSS?

У CSS є 4 основних типи позиціонування:

  • static — за замовчуванням, елементи розташовуються у потоці документа.
  • relative — елемент позиціонується відносно його звичайного місця в потоці.
  • absolute — елемент позиціонується відносно найближчого батька з position: relative.
  • fixed — елемент позиціонується відносно вікна браузера і залишається на місці при прокручуванні сторінки.
7. Що таке Webpack?

Webpack — це збирач модулів для JavaScript. Він дозволяє збирати ресурси, такі як JavaScript, CSS, зображення та шрифти, у єдиний бандл або кілька бандлів. Webpack використовується для оптимізації й автоматизації процесів зборки проектів.

8. Що таке селектори в CSS?

Селектори CSS дозволяють вибирати HTML-елементи для їхнього стилізування.

h1 { color: blue; }
.className { background-color: yellow; }
#idName { border: 1px solid black; }
8.1. Що таке комбіновані селектори?

Це селектори, які використовуються разом для уточнення вибору елементів.

div p { color: green; }  /* Всі 
 у межах 
 */
div > p { color: red; }  /* Тільки прямі нащадки 
 */
div + p { color: blue; } /* Сусідній 
, що йде одразу після 
 */
8.2. Що таке псевдокласи в CSS?

Псевдокласи застосовуються для вибору елементів у специфічному стані.

a:hover { color: red; } /* Змінює колір тексту посилання при наведенні */
input:focus { border: 2px solid blue; } /* Виділяє активне поле вводу */
8.3. Що таке псевдоелементи в CSS?

Псевдоелементи дозволяють стилізувати певну частину елемента.

p::first-line { font-weight: bold; } /* Перший рядок тексту */
div::before { content: 'Note: '; color: gray; } /* Додає текст перед елементом */
8.4. Як працює універсальний селектор (*)?

Універсальний селектор (*) вибирає всі елементи на сторінці.

* { margin: 0; padding: 0; } /* Скидає всі відступи */
8.5. Що таке атрибутні селектори в CSS?

Атрибутні селектори дозволяють вибирати елементи за значенням їх атрибутів.

input[type="text"] { border: 1px solid black; } /* Вибір полів введення тексту */
a[target="_blank"] { color: blue; } /* Вибір посилань із певним target */
8.6. Як працює селектор :nth-child()?

:nth-child() вибирає елементи за їхньою позицією серед нащадків.

li:nth-child(2) { color: red; } /* Другий 
  •  у списку */
li:nth-child(odd) { background: gray; } /* Непарні елементи */
    • 8.7. Що таке селектори групи?

    Селектори групи дозволяють застосовувати один і той же стиль до кількох елементів.

    h1, h2, h3 { font-family: Arial, sans-serif; } /* Застосовує стиль до кількох тегів */
    8.8. Як вибрати елемент без певного класу?

    Селектор :not() використовується для вибору елементів, які не мають заданого класу.

    div:not(.active) { color: gray; } /* Вибирає DIV, які не мають класу 'active' */
    9. У чому різниця між Flexbox і CSS Grid?

    Обидва інструменти — це потужні сучасні способи створення адаптивних макетів, але мають різні призначення:

    🔹 Flexbox — для одновимірного розташування

    • Працює з **одним напрямком** — рядком або стовпцем.
    • Зручний для вирівнювання елементів усередині блоку (наприклад, кнопок, навігації).
    • Основна вісь задається через flex-direction (row / column).
    .container {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

    🔹 CSS Grid — для двовимірного розташування

    • Працює по **двох осях** — рядках і стовпцях.
    • Зручно створювати складні макети сторінок, сітки галерей тощо.
    • Підтримує іменовані лінії, області, авто-розподіл.
    .container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  grid-gap: 20px;
}

    10. Що таке Box Model у CSS?

    Box Model описує, як браузер обчислює розміри та простір навколо елемента. Кожен елемент складається з чотирьох частин:

    • Content — вміст елемента (текст, зображення тощо).
    • Padding — внутрішні відступи між контентом і рамкою.
    • Border — рамка навколо елемента.
    • Margin — зовнішній відступ між елементами.
    div {
  width: 200px;
  padding: 10px;
  border: 5px solid black;
  margin: 20px;
}

    Повна ширина елемента: width + padding + border + margin.

    Щоб розмір включав padding і border, використовують box-sizing: border-box;.

    11. Як працює position: sticky?

    position: sticky поєднує поведінку relative і fixed. Елемент залишається на місці під час скролу, доки не досягне межі контейнера.

    header {
  position: sticky;
  top: 0;
  background: #fff;
  border-bottom: 1px solid #ccc;
}

    Важливо: батьківський елемент повинен мати висоту, більшу за сам елемент, і не бути overflow: hidden.

    12. Які одиниці вимірювання використовуються для адаптивного дизайну?
    • px — пікселі, фіксована одиниця.
    • % — відсотки від батьківського елемента.
    • em — відносно шрифту батьківського елемента.
    • rem — відносно шрифту html.
    • vw, vh — відсотки від ширини або висоти вікна браузера.

    Адаптивна верстка часто поєднує rem і vw для шрифтів і розмірів блоків.

    13. У чому різниця між inline, block і inline-block?
    • inline — займає ширину контенту, не переноситься на новий рядок. Не можна задавати width/height.
    • block — займає всю ширину батьківського елемента, переноситься на новий рядок.
    • inline-block — поєднує властивості обох: можна задавати width/height, але не переноситься автоматично.
    span { display: inline; }
div { display: block; }
button { display: inline-block; }
    14. Що таке специфічність (specificity) у CSS?

    Специфічність визначає, який стиль переможе при конфлікті. Рахується за формулою: inline > id > class > tag.

    • inline стилі (написані прямо в елементі) — найвищий пріоритет.
    • #id має більшу вагу, ніж .class.
    • .class має більшу вагу, ніж тег.
    • !important перевизначає все (але краще не зловживати).
    p { color: blue; }
.text { color: red; }
#main { color: green; } /* переможе */
    15. Як працює наслідування (inheritance) у CSS?

    Деякі властивості автоматично передаються дочірнім елементам (наприклад, шрифти, колір тексту).

    • Успадковуються: color, font-family, line-height.
    • Не успадковуються: margin, padding, border.

    Щоб примусово наслідувати, використовують inherit.

    div { color: blue; }
p { color: inherit; }
    16. Як працює z-index і що таке stacking context?

    z-index визначає порядок накладання елементів по осі Z (поверх/під іншими).

    • Працює лише для елементів із position відмінним від static.
    • Новий stacking context створюється елементами з position + z-index, opacity < 1, transform, filter тощо.
    .box1 {
  position: relative;
  z-index: 2;
}
.box2 {
  position: relative;
  z-index: 1;
}
    17. Яка різниця між display: none і visibility: hidden?
    • display: none — елемент повністю зникає з DOM-потоку.
    • visibility: hidden — елемент залишається на місці, але стає невидимим.
    .hidden1 { display: none; }   /* не займає місце */
.hidden2 { visibility: hidden; } /* займає місце */
    18. Що таке overflow і які його типи?

    Властивість overflow керує тим, як відображається вміст, що виходить за межі контейнера.

    • visible — контент видно за межами блоку.
    • hidden — контент обрізається.
    • scroll — завжди показує смуги прокручування.
    • auto — смуги з’являються за потреби.
    .box {
  width: 200px;
  height: 100px;
  overflow: auto;
}
    19. Що таке float і чому його зараз майже не використовують?

    float використовується для обтікання текстом елементів (як зображення в статті).

    Раніше його застосовували для верстки, але зараз його замінили Flexbox і Grid, бо вони більш гнучкі.

    img {
  float: left;
  margin-right: 20px;
}
    20. Як центрувати елемент по вертикалі та горизонталі (підсумок)?

    Основні способи:

    • Flexbox: display: flex; justify-content: center; align-items: center;
    • Grid: display: grid; place-items: center;
    • Absolute + transform: top: 50%; left: 50%; transform: translate(-50%, -50%);
    • margin: auto — для блочних елементів із фіксованими розмірами.

    🧩 2. Flexbox & Grid

    21. Як працює Flexbox і його основні властивості?

    Flexbox — це одновимірна модель для розміщення елементів у ряд або стовпець. Контейнер із display: flex керує своїми дітьми за допомогою властивостей:

    • justify-content — вирівнювання по горизонталі (main axis)
    • align-items — вирівнювання по вертикалі (cross axis)
    • flex-wrap — дозволяє переносити елементи на новий рядок
    • align-content — вирівнює кілька рядків одночасно
    .container {
  display: flex;
  justify-content: space-between;
  align-items: center;
  flex-wrap: wrap;
}
    22. Як працюють flex-grow, flex-shrink і flex-basis?
    • flex-grow — наскільки елемент може розширюватися, щоб заповнити простір.
    • flex-shrink — наскільки елемент може стискатись.
    • flex-basis — початковий розмір елемента (до розподілу простору).
    .item {
  flex-grow: 1;
  flex-shrink: 0;
  flex-basis: 200px;
}
/* короткий запис */
.item { flex: 1 0 200px; }
    23. Як створити адаптивну сітку лише через Flexbox?

    Використай flex-wrap та відсоткову ширину елементів:

    .container {
  display: flex;
  flex-wrap: wrap;
  gap: 10px;
}
.item {
  flex: 1 1 30%; /* адаптивно */
  min-width: 250px;
}

    Цей підхід дозволяє елементам адаптуватися під ширину екрана без media queries.

    24. Як працює CSS Grid?

    Grid — це двовимірна система розташування елементів (рядки + колонки). Контейнер із display: grid має властивості:

    • grid-template-columns / grid-template-rows — задають структуру.
    • gap — відстань між клітинками.
    • grid-column / grid-row — позиціонування елемента в сітці.
    .container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}
.item {
  grid-column: span 2; /* розтягується на 2 колонки */
}
    25. Як працюють auto-fit і auto-fill у Grid?

    Обидві директиви допомагають автоматично створювати адаптивні колонки.

    • auto-fill — створює максимальну кількість колонок, навіть якщо вони порожні.
    • auto-fit — заповнює лише реальні елементи і розтягує їх.
    .container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 10px;
}
    26. Як створити sticky footer із Flex або Grid?

    Варіант з Flexbox:

    body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
main { flex: 1; }
footer { background: #eee; }

    Варіант з Grid:

    body {
  display: grid;
  grid-template-rows: 1fr auto;
  min-height: 100vh;
}
    27. Як змінити порядок елементів у Flexbox?

    Використай властивість order — за замовчуванням усі елементи мають значення 0.

    .item:nth-child(1) { order: 3; }
.item:nth-child(2) { order: 1; }
.item:nth-child(3) { order: 2; }
    28. Як поєднувати Flexbox і Grid в одному layout?

    Це звична практика: використовуй Grid для глобального макету, а Flex для внутрішніх елементів.

    .wrapper {
  display: grid;
  grid-template-columns: 1fr 3fr;
  gap: 20px;
}
.sidebar {
  display: flex;
  flex-direction: column;
  gap: 10px;
}
    29. Як працює gap у Flexbox і Grid?

    gap додає відстань між дочірніми елементами без використання margin.

    Підтримується як у flex, так і в grid контейнерах.

    .container {
  display: flex;
  gap: 20px;
}
    30. У чому різниця між align-content і align-items?
    • align-items — вирівнює елементи всередині одного рядка (по поперечній осі).
    • align-content — вирівнює кілька рядків (якщо є перенос).
    .container {
  display: flex;
  flex-wrap: wrap;
  align-items: center;
  align-content: space-between;
}

    🧠 3. Responsive Design і одиниці вимірювання

    31. Що таке responsive design і як його реалізувати?

    Responsive design — це підхід, при якому сторінка адаптується до розміру екрана користувача.

    • Використання flex та grid для гнучких макетів.
    • Застосування media queries для зміни стилів.
    • Відносні одиниці (%, vw, em, rem).
    • Адаптивні зображення через srcset або picture.
    32. Як працюють %, em, rem, vw, vh, clamp()?
    • % — відносно батьківського елемента.
    • em — відносно розміру шрифту батьківського елемента.
    • rem — відносно шрифту елемента html.
    • vw / vh — 1% від ширини / висоти viewport.
    • clamp(min, preferred, max) — автоматично підбирає значення в заданих межах.
    font-size: clamp(1rem, 2vw, 2rem);
width: 80%;
height: 50vh;
    33. Як працюють media queries — різниця між min-width і max-width?
    • min-width — стилі застосовуються, коли екран ширший за вказане значення (mobile-first).
    • max-width — стилі застосовуються, коли екран вужчий за вказане значення (desktop-first).
    @media (min-width: 768px) {
  body { background: lightblue; }
}
@media (max-width: 767px) {
  body { background: pink; }
}
    34. Що таке container queries (новий стандарт CSS)?

    Container queries дозволяють змінювати стилі не від ширини viewport, а від ширини контейнера.

    .container {
  container-type: inline-size;
}
@container (min-width: 600px) {
  .card { flex-direction: row; }
}

    Це робить компоненти по-справжньому ізольованими та адаптивними.

    35. Як працює властивість aspect-ratio?

    aspect-ratio задає співвідношення сторін елемента.

    .box {
  width: 300px;
  aspect-ratio: 16 / 9;
  background: #ccc;
}

    Корисно для зображень, відео або блоків фіксованих пропорцій.

    36. Як робити адаптивні зображення (srcset, sizes)?

    Використовуй атрибути srcset і sizes для підвантаження потрібного розміру залежно від екрану.
    37. Що таке mobile-first підхід?

    Mobile-first — це принцип розробки, де стилі спочатку створюються для мобільних екранів, а потім масштабуються вгору через min-width media queries.

    /* базові стилі для мобільних */
body { font-size: 16px; }

/* більші екрани */
@media (min-width: 768px) {
  body { font-size: 18px; }
}
    38. Як протестувати адаптивність у DevTools?

    У Chrome DevTools є інструмент Device Toolbar (Ctrl+Shift+M), який дозволяє:

    • Переглядати сторінку на різних розмірах екранів.
    • Імітувати мобільні пристрої (iPhone, Galaxy тощо).
    • Перевіряти поведінку touch-ів, viewport, zoom тощо.
    39. Як зробити fluid-типографіку через clamp()?

    Використай clamp() для плавного масштабування шрифтів:

    h1 {
  font-size: clamp(1.5rem, 5vw, 3rem);
}

    Шрифт змінюється між 1.5rem і 3rem залежно від ширини viewport.

    40. Як визначити breakpoints і не вигадувати з голови?

    Breakpoints базують на контенті, а не на пристроях. Типові значення:

    • Mobile: 0–600px
    • Tablet: 601–992px
    • Laptop: 993–1200px
    • Desktop: 1201px+

    Але краще орієнтуйся на реальні “злами” дизайну — коли контент починає виглядати погано.

    🎨 4. Візуальні ефекти, transitions, animations

    41. У чому різниця між transition і animation?
    • transition — виконує плавну зміну між двома станами (початковим і кінцевим).
    • animation — може мати кілька кроків, циклів і складну часову лінію через @keyframes.
    .box {
  transition: transform 0.3s ease;
}
.box:hover {
  transform: scale(1.1);
}
    42. Як працюють @keyframes?

    @keyframes описують кроки анімації — від початку до кінця:

    @keyframes move {
  0% { transform: translateX(0); }
  100% { transform: translateX(100px); }
}
.box {
  animation: move 2s infinite alternate;
}
    43. Як зробити нескінченну анімацію або затримку між циклами?
    • animation-iteration-count: infinite; — нескінчне повторення.
    • animation-delay — затримка перед початком.
    • animation-direction: alternate; — чергування напрямку.
    .ball {
  animation: bounce 1s infinite alternate;
}
@keyframes bounce {
  from { transform: translateY(0); }
  to { transform: translateY(-50px); }
}
    44. Які властивості не можна анімувати?

    Не всі CSS-властивості підтримують анімацію. Не можна анімувати:

    • display
    • position
    • background-image (зміна URL)
    • visibility
    • z-index

    Порада: анімуй трансформації (transform, opacity) — вони не викликають перерахунок layout.

    45. Що таке transform (translate, scale, rotate, skew)?

    transform змінює форму або позицію елемента без зміни його розміру в DOM.

    .box {
  transform: translate(20px, 10px) rotate(15deg) scale(1.2);
}
    46. Як працює GPU-рендеринг (transform: translateZ(0))?

    Використання transform: translateZ(0) або will-change: transform створює новий “composite layer”, який обробляється GPU, а не CPU — це пришвидшує анімації.

    ⚠️ Не використовуй надмірно — це збільшує споживання пам’яті.

    47. Як оптимізувати анімації для кращої продуктивності?
    • Анімуй тільки opacity і transform.
    • Уникай top, left, width — вони викликають reflow.
    • Використовуй will-change тільки для активних елементів.
    • Зменшуй кількість одночасних анімацій.
    48. Що таке will-change і коли його варто використовувати?

    will-change повідомляє браузеру, які властивості змінюватимуться, щоб він міг оптимізувати рендер.

    .card:hover {
  will-change: transform, opacity;
}

    Використовуй обережно, лише для елементів, що часто змінюються (hover, drag, scroll).

    49. Як зробити fade-in, slide-in або spinner тільки CSS-ом?

    Приклади:

    Fade-in

    .fade {
  animation: fadeIn 1s ease-in;
}
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

    Slide-in

    .slide {
  animation: slideIn 0.8s ease-out;
}
@keyframes slideIn {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}

    Spinner

    .spinner {
  border: 4px solid #ccc;
  border-top: 4px solid #007BFF;
  border-radius: 50%;
  width: 40px;
  height: 40px;
  animation: spin 1s linear infinite;
}
@keyframes spin {
  to { transform: rotate(360deg); }
}
    50. Що таке prefers-reduced-motion і для чого воно потрібне?

    Це медіа-запит, який дозволяє враховувати користувацькі налаштування щодо зменшення анімацій — для людей із вестибулярними або когнітивними розладами.

    @media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0s !important;
    transition: none !important;
  }
}

    ⚙️ 5. SCSS / Препроцесори

    51. У чому різниця між SASS і SCSS синтаксисом?
    • SASS — старіший синтаксис без фігурних дужок і крапок з комою (на зразок YAML).
    • SCSS — сучасний синтаксис, максимально схожий на звичайний CSS.
    /* SCSS */
.button {
  color: white;
  background: blue;
}

/* SASS */
.button
  color: white
  background: blue
    52. Як працюють змінні, mixins, extends і placeholders у SCSS?
    • $variable — зберігає значення для повторного використання.
    • @mixin — шаблон, який можна підключати через @include.
    • @extend — наслідує стилі іншого селектора.
    • %placeholder — як @extend, але не створює власного селектора у CSS.
    $primary: #007BFF;

@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

%btn-base {
  padding: 10px 20px;
  border-radius: 5px;
}

.button {
  @include flex-center;
  @extend %btn-base;
  background: $primary;
}
    53. Що таке nesting і чому надмірна вкладеність — антипатерн?

    Nesting — це можливість вкладати селектори один у інший для зручності структури.

    .card {
  padding: 10px;

  h2 { font-size: 20px; }

  .btn {
    color: white;
  }
}

    Надмірний nesting (>3 рівнів) ускладнює підтримку коду й збільшує специфічність.

    54. Як використовуються partials (_variables.scss, _mixins.scss)?

    Partials — це окремі SCSS-файли, які не компілюються самостійно (починаються з “_”) і підключаються через @use або @import.

    /* _variables.scss */
$primary: #007BFF;

/* main.scss */
@use 'variables';

.button {
  background: variables.$primary;
}
    55. Як створити функцію у SCSS (@function)?

    Функції у SCSS дозволяють повертати обчислені значення.

    @function rem($px) {
  @return $px / 16 * 1rem;
}

h1 {
  font-size: rem(24);
}
    56. Що таке maps у SCSS?

    Map — це структура даних у форматі ключ–значення.

    $colors: (
  primary: #007BFF,
  success: #28a745,
  danger: #dc3545
);

.button {
  background: map-get($colors, primary);
}
    57. Як SCSS компілюється у звичайний CSS?

    Компіляція — це процес перетворення SCSS у чистий CSS через препроцесор (наприклад, Gulp, Webpack або CLI-команду):

    sass styles.scss styles.css --style compressed
    58. Як організувати структуру SCSS-проєкту?

    Зазвичай використовують модульну структуру:

    • base/ — типографіка, reset
    • layout/ — сітки, header, footer
    • components/ — кнопки, картки
    • utilities/ — змінні, міксіни
    • main.scss — точка входу
    59. Як підключати SCSS-змінні у JS через CSS variables?

    Передай значення SCSS-змінних у :root як CSS-перемінні:

    /* SCSS */
:root {
  --primary: #{$primary};
}

/* JS */
const color = getComputedStyle(document.documentElement)
  .getPropertyValue('--primary');
    60. Як реалізувати BEM у SCSS?

    SCSS полегшує написання BEM через вкладеність і амперсанд &:

    .block {
  &__element {
    color: black;
  }
  &--modifier {
    color: red;
  }
}

    🧱 6. Методології CSS

    61. Що таке BEM (Block Element Modifier)?

    BEM — методологія, що розділяє код на незалежні блоки, елементи та модифікатори.

    • Block — самостійна частина інтерфейсу (.card).
    • Element — складова частина блоку (.card__title).
    • Modifier — варіація блоку або елемента (.card--active).
    .card { ... }
.card__title { ... }
.card--active { ... }

    ✔️ Основна перевага: передбачувана структура, уникнення конфліктів стилів.

    62. Що таке SMACSS (Scalable and Modular Architecture for CSS)?

    SMACSS ділить CSS на категорії для кращої організації:

    • Base — базові стилі (html, body, a, p).
    • Layout — структура сторінки (header, footer, grid).
    • Module — незалежні блоки (кнопки, картки).
    • State — змінені стани (is-active, is-hidden).
    • Theme — кольорові схеми, теми.
    .header { ... } /* Layout */
.btn { ... } /* Module */
.is-active { ... } /* State */
    63. Що таке OOCSS (Object-Oriented CSS)?

    OOCSS — об’єктно-орієнтований підхід до стилізації, який відокремлює структуру від вигляду.

    • Structure — контейнер, розміри, відступи.
    • Skin — колір, фон, тіні.
    .media { display: flex; align-items: center; } /* Structure */
.avatar { border-radius: 50%; } /* Skin */
    64. Що таке Atomic CSS (або Functional CSS)?

    Atomic CSS — підхід, де один клас відповідає одній властивості.

    .text-center { text-align: center; }
.mt-4 { margin-top: 1rem; }
.bg-blue { background: blue; }

    ✔️ Швидка розробка, мінімальні конфлікти, схоже на підхід Tailwind.

    65. Що таке ITCSS (Inverted Triangle CSS)?

    ITCSS — методологія, що впорядковує стилі за рівнем специфічності, від загального до приватного:

    • 1️⃣ Settings — змінні, кольори, шрифти.
    • 2️⃣ Tools — міксіни, функції.
    • 3️⃣ Generic — reset, normalize.
    • 4️⃣ Elements — базові теги (h1, p, a).
    • 5️⃣ Objects — патерни без оформлення.
    • 6️⃣ Components — модулі UI.
    • 7️⃣ Utilities — допоміжні класи (text-center, hidden).
    66. Як поєднувати BEM і ITCSS у великому проєкті?

    Використовуй ITCSS для структури проєкту, а BEM — для іменування компонентів.

    /* Структура */
components/
  _buttons.scss
  _cards.scss

/* Іменування */
.card { ... }
.card__image { ... }
.card--featured { ... }
    67. Що таке CSS Modules і як вони працюють у React?

    CSS Modules — це спосіб ізолювання стилів для компонентів у React або Vue.

    • Імпортуєш CSS прямо у JS.
    • Класи автоматично унікалізуються (через хеш).
    /* button.module.css */
.btn { background: blue; color: white; }

/* Button.jsx */
import styles from './button.module.css';
<button className={styles.btn}>Click</button>
    68. Як працюють Styled Components і чим вони відрізняються від SCSS?

    Styled Components — це CSS у JS: стилі пишуться прямо у компоненті через template literals.

    import styled from "styled-components";

const Button = styled.button`
  background: blue;
  color: white;
  padding: 10px 20px;
`;

<Button>Click me</Button>

    ✔️ Повна ізоляція стилів, підтримка props та динамічних тем.

    69. Що таке Utility-first CSS?

    Utility-first — це підхід, де використовуються готові класи для конкретних властивостей, замість створення окремих компонентних класів.

    <button class="bg-blue-500 text-white px-4 py-2 rounded">
  Submit
</button>

    Tailwind CSS — найвідоміша реалізація цього підходу.

    70. Коли варто обрати методологію (BEM, ITCSS) замість фреймворку (Tailwind)?
    • ✔️ Методології — краще для великих проєктів із власним UI, коли потрібна гнучкість і контроль над CSS.
    • ⚡️ Фреймворки (Tailwind, Bootstrap) — швидкий старт, але складніше кастомізувати під великий масштаб.

    Ідеальний варіант — комбінувати: наприклад, Tailwind для утиліт + BEM для складних компонентів.

    💨 7. Tailwind CSS

    71. Як працює Tailwind CSS (utility-first підхід)?

    Tailwind CSS — це фреймворк із підходом utility-first, де ти використовуєш готові класи для стилізації без написання власного CSS.

    <button class="bg-blue-500 text-white font-semibold px-4 py-2 rounded hover:bg-blue-600">
  Натисни мене
</button>

    Tailwind дозволяє створювати складні дизайни, комбінуючи утилітарні класи без необхідності писати нові стилі.

    72. Що таке JIT-компіляція у Tailwind?

    JIT (Just-In-Time Compiler) генерує класи «на льоту», тільки ті, що реально використовуються у твоєму HTML/JSX.

    • ⚡️ Миттєве оновлення стилів у dev-режимі.
    • 📦 Зменшує розмір кінцевого CSS.
    • 🧩 Дозволяє використовувати динамічні значення (наприклад, bg-[color]).
    73. Як Tailwind прибирає невикористані стилі (purge / content)?

    Tailwind сканує файли проєкту й залишає лише ті класи, що реально використовуються.

    /* tailwind.config.js */
module.exports = {
  content: ["./src/**/*.{html,js,ts,jsx,tsx}"],
  theme: { extend: {} },
  plugins: [],
};

    Це суттєво зменшує фінальний розмір CSS у продакшені.

    74. Як працюють variants у Tailwind (hover:, focus:, md:)?

    Variants дозволяють змінювати стилі залежно від стану або breakpoint’у:

    <button class="bg-gray-300 hover:bg-gray-500 focus:ring-2 md:bg-blue-500">
  Кнопка
</button>

    Префікси: hover:, focus:, active:, md:, lg:, dark: тощо.

    75. Як реалізується theme customization у Tailwind?

    Теми або свої кольори додаються через розділ theme.extend у конфігурації:

    /* tailwind.config.js */
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: "#1E40AF",
      },
      fontFamily: {
        sans: ["Inter", "sans-serif"],
      },
    },
  },
};
    76. Як поєднувати Tailwind із BEM або React components?

    Tailwind можна поєднувати з BEM для складних компонентів або з React через props.

    function Button({ type }) {
  const base = "px-4 py-2 rounded text-white";
  const styles = type === "primary" ? "bg-blue-600" : "bg-gray-500";
  return <button class={`${base} ${styles}`}>Click</button>;
}

    Tailwind добре працює з компонентним підходом.

    77. Як писати кастомні класи у Tailwind через @apply?

    Використовуй @apply у власному CSS, щоб поєднати кілька утиліт у новий клас:

    /* styles.css */
.btn {
  @apply bg-blue-600 text-white font-semibold px-4 py-2 rounded hover:bg-blue-700;
}

    Це допомагає уникнути дублювання однакових класів у HTML.

    78. Як зробити dark-mode у Tailwind?

    Tailwind має вбудовану підтримку темної теми. Активується через darkMode у конфігурації:

    /* tailwind.config.js */
module.exports = {
  darkMode: "class", // або "media"
};

    Потім використовуєш префікс dark::

    <div class="bg-white dark:bg-gray-900 text-gray-900 dark:text-white">...
    79. Як додавати плагіни у Tailwind?

    Tailwind дозволяє підключати офіційні та кастомні плагіни через plugins:

    /* tailwind.config.js */
const forms = require("@tailwindcss/forms");
module.exports = {
  plugins: [forms],
};

    Популярні плагіни: @tailwindcss/forms, @tailwindcss/typography, @tailwindcss/aspect-ratio.

    80. Як оптимізувати Tailwind CSS у продакшені?
    • 🧹 Використовуй purge/content для видалення зайвих стилів.
    • ⚙️ Увімкни NODE_ENV=production — Tailwind згенерує мінімізований CSS.
    • 📦 Збери все у бандл через PostCSS або Vite.
    NODE_ENV=production npx tailwindcss -o build.css --minify

    ⚡️ 8. Performance & Advanced CSS

    81. Що таке Critical Rendering Path і як CSS впливає на нього?

    Critical Rendering Path (CRP) — це процес, який браузер проходить, щоб перетворити HTML, CSS і JS у пікселі на екрані.

    • CSS блокує рендеринг сторінки — поки не завантажиться, сторінка не відображається.
    • Тому важливо мінімізувати і оптимізувати CSS.

    Оптимізація:

    • Винось критичні стилі inline у <head>.
    • Використовуй media атрибут (media="print").
    • Використовуй preload для важливих CSS.
    82. Як мінімізувати render-blocking CSS?

    Щоб сторінка швидше відображалась, CSS не повинен блокувати побудову DOM.

    <link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="styles.css"></noscript>
    • Розбий CSS на критичний і не критичний.
    • Використовуй lazy-loading для великих стилів.
    • Використовуй PostCSS або tools (PurgeCSS, CriticalCSS).
    83. Що таке repaint і reflow?

    Reflow — це перерахунок розміщення елементів (layout). Repaint — це оновлення зовнішнього вигляду (color, background тощо).

    Reflow → Repaint = важка операція ⚠️

    Що викликає reflow:

    • Зміна розміру вікна.
    • Маніпуляції з DOM (appendChild, offsetWidth).
    • Зміна шрифтів або CSS layout.

    Оптимізація: мінімізуй зміни в DOM, використовуй transform замість top/left.

    84. Що таке layout thrashing?

    Це ситуація, коли JavaScript постійно чергує операції «читання» і «запис» DOM, змушуючи браузер щоразу робити reflow.

    // ❌ Погано
div.style.width = div.offsetWidth + 10 + 'px';

// ✅ Добре
const width = div.offsetWidth;
div.style.width = width + 10 + 'px';

    💡 Рішення — кешуй значення перед зміною стилів.

    85. Як DevTools допомагає аналізувати продуктивність CSS?

    У Chrome DevTools вкладка Performance показує:

    • Кількість reflow/repaint.
    • Час рендерингу кожного фрейму.
    • Вкладку Rendering → Paint flashing — підсвічує repaint-зони.

    Аналізуй, які стилі часто перераховуються, щоб уникнути надмірного перерендеру.

    86. Як оптимізувати CSS у продакшені?
    • 🧹 Використовуй PurgeCSS / Tailwind content для видалення зайвих стилів.
    • 📦 Мінімізуй CSS (gzip, cssnano).
    • 🚀 Винось критичний CSS inline, решту — async.
    • ⚙️ Розділяй стилі по сторінках або компонентно.
    npx postcss src/styles.css -o dist/styles.min.css --env production
    87. Що таке content-visibility: auto?

    Ця властивість дозволяє браузеру не рендерити елементи, поки вони не з’являться у viewport.

    .card {
  content-visibility: auto;
  contain-intrinsic-size: 300px;
}

    Результат — сторінка рендериться значно швидше (lazy-rendering).

    88. Як працює властивість contain?

    contain обмежує вплив елемента на решту сторінки, що зменшує кількість reflow.

    .component {
  contain: layout paint;
}

    Браузер розглядає цей елемент як “ізольований контейнер”.

    89. Як @layer допомагає керувати специфічністю?

    У CSS Layers можна визначати рівні стилів (base, components, utilities) і контролювати порядок накладання без !important.

    @layer base {
  h1 { color: black; }
}
@layer components {
  .title { color: blue; }
}
@layer utilities {
  .text-red { color: red; }
}

    Останній визначений шар має пріоритет, але все ще легко керований.

    90. Що таке CSS Houdini і як він розширює можливості CSS?

    CSS Houdini — це набір API, який дозволяє розробникам писати власні CSS-властивості та парсери.

    • Можна створювати кастомні властивості з поведінкою.
    • Підвищує гнучкість браузерного рендерингу.
    // Реєстрація нової властивості
CSS.registerProperty({
  name: "--rotation",
  syntax: "<angle>",
  inherits: false,
  initialValue: "0deg",
});

    Наприклад, тепер можна анімувати --rotation як звичайну властивість.

    🧠 10. Advanced / Trick CSS Questions

    91. Як працює stacking context (і які властивості його створюють)?

    Stacking context — це середовище, у якому елементи накладаються один на одного (z-index працює лише всередині нього).

    Створюється коли:

    • position: relative / absolute / fixed + z-index;
    • opacity < 1;
    • transform, filter, perspective;
    • isolation: isolate або mix-blend-mode.
    .parent {
  position: relative;
  z-index: 10;
}
.child {
  position: absolute;
  z-index: 5; /* тільки в межах parent */
}
    92. Як зробити трикутник у CSS без зображень?

    Трикутники створюються через властивість border із прозорими сторонами.

    .triangle {
  width: 0;
  height: 0;
  border-left: 50px solid transparent;
  border-right: 50px solid transparent;
  border-bottom: 100px solid red;
}

    Робить елемент нульового розміру з кольоровим одним бордером.

    93. Що таке clip-path і mask-image?
    • clip-path — обрізає елемент за заданою формою (полігон, коло).
    • mask-image — приховує частини елемента за допомогою прозорості.
    .circle {
  clip-path: circle(50% at 50% 50%);
}

.mask {
  mask-image: url(mask.svg);
  -webkit-mask-image: url(mask.svg);
}
    94. Як зробити скляний ефект (glassmorphism) лише CSS?

    Glassmorphism — це напівпрозорий фон із розмиттям і відблиском.

    .glass {
  background: rgba(255, 255, 255, 0.15);
  backdrop-filter: blur(10px);
  border: 1px solid rgba(255, 255, 255, 0.3);
}

    Використовує backdrop-filter, тому працює не у всіх браузерах (особливо IE).

    95. Що таке logical properties (margin-inline, padding-block)?

    Логічні властивості адаптують layout під напрямок письма (LTR / RTL / вертикальний текст).

    .box {
  padding-block: 20px;   /* top + bottom */
  margin-inline: 10px;   /* left + right */
}

    Дозволяє автоматично підтримувати мови з іншим напрямком тексту.

    96. Як зробити адаптивну grid без media queries (через minmax і auto-fit)?

    Grid може сам адаптувати кількість колонок без використання media queries.

    .container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}

    Кількість колонок автоматично змінюється відповідно до ширини контейнера.

    97. Як працює :has() — новий CSS селектор?

    :has() — перший селектор, що дозволяє вибирати батьківський елемент залежно від дочірнього.

    article:has(img) {
  border: 2px solid green;
}
input:has(+ label) {
  background: #f9f9f9;
}

    Підтримується у сучасних браузерах (Chrome 105+, Safari 15.4+).

    98. Що таке accent-color, color-scheme, scroll-timeline?
    • accent-color — дозволяє задати колір елементів форм (checkbox, range).
    • color-scheme — повідомляє браузеру про підтримку темної/світлої теми.
    • scroll-timeline — дає змогу створювати анімації, що залежать від скролу.
    input[type="checkbox"] {
  accent-color: #007bff;
}

:root {
  color-scheme: light dark;
}

@scroll-timeline scroll {
  scroll-source: body;
  orientation: block;
}
    99. Як створити власну spacing-систему через CSS Variables?

    Можна оголосити змінні для відступів, щоб зберегти консистентність дизайну.

    :root {
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 32px;
}
.card {
  padding: var(--space-md);
  margin-bottom: var(--space-lg);
}

    Такі змінні легко масштабувати під теми чи breakpoints.

    100. Як працює backdrop-filter і які його обмеження?

    backdrop-filter застосовує ефекти (blur, brightness, contrast) до області позаду елемента.

    .overlay {
  backdrop-filter: blur(8px) brightness(0.8);
  background: rgba(0, 0, 0, 0.3);
}

    Працює лише на елементах із напівпрозорим фоном і не підтримується в IE.

    101. Як створити кастомний scrollbar?

    Можна стилізувати скролбар через псевдоелементи ::-webkit-scrollbar.

    ::-webkit-scrollbar {
  width: 10px;
}
::-webkit-scrollbar-thumb {
  background: linear-gradient(#007bff, #00bcd4);
  border-radius: 5px;
}
::-webkit-scrollbar-track {
  background: #f1f1f1;
}
    102. Як зробити “skewed” або “angled” секцію без SVG?

    Можна використати transform: skewY() або псевдоелемент із clip-path.

    .section {
  background: #007bff;
  transform: skewY(-3deg);
  margin: 50px 0;
}
    103. Як CSS впливає на FCP та CLS?
    • FCP (First Contentful Paint): великі CSS-файли затримують перше відображення контенту → оптимізуй критичні стилі.
    • CLS (Cumulative Layout Shift): неправильні розміри елементів (зображень, шрифтів) викликають стрибки контенту.

    💡 Завжди задавай width і height для медіа, щоб уникнути CLS.
