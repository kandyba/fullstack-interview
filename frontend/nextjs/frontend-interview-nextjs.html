Запитання та відповіді для технічної співбесіди (Next.js)

1. Що таке Next.js і для чого він використовується?

Next.js — це React-фреймворк для створення веб-додатків з рендерингом на сервері (SSR), статичною генерацією (SSG) та можливістю створення API-ендпоїнтів.

Він спрощує SEO, покращує продуктивність та надає вбудований роутинг і оптимізацію.

2. Яка структура типової Next.js аплікації?
📦 my-app/
 ┣ 📂 pages/
 ┃ ┣ index.js
 ┃ ┣ about.js
 ┃ ┗ 📂 api/
 ┃   ┗ hello.js
 ┣ 📂 components/
 ┣ 📂 public/
 ┣ 📂 styles/
 ┣ next.config.js
 ┣ package.json
 ┗ README.md

/pages — автоматичний роутинг.
/api — серверні ендпоїнти.
/public — статичні файли.
/components — UI-компоненти.

3. У чому різниця між SSR, SSG, ISR та CSR?
  • SSR (Server-Side Rendering) — сторінка рендериться на сервері при кожному запиті (getServerSideProps).
  • SSG (Static Site Generation) — сторінка створюється під час білду (getStaticProps).
  • ISR (Incremental Static Regeneration) — статична сторінка оновлюється з певним інтервалом без повного білду.
  • CSR (Client-Side Rendering) — рендеринг виконується повністю у браузері (як у чистому React).
4. Що таке getStaticProps, getServerSideProps та getStaticPaths?
  • getStaticProps — викликається під час білду, використовується для SSG.
  • getServerSideProps — викликається при кожному запиті на сервер (SSR).
  • getStaticPaths — генерує список шляхів для динамічних сторінок при SSG.
export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();
  return { props: { posts } };
}
5. Як реалізується роутинг у Next.js?

Next.js має файловий роутинг — кожен файл у /pages автоматично стає маршрутом.

/pages/about.js → /about
/pages/blog/[id].js → /blog/1

Динамічні шляхи оголошуються через [param], а catch-all — через [...slug].

6. Що таке API Routes у Next.js?

API Routes дозволяють створювати серверні ендпоїнти без окремого бекенду.

// /pages/api/hello.js
export default function handler(req, res) {
  res.status(200).json({ message: 'Hello, API!' });
}
7. Як працює Image Optimization у Next.js?

Компонент <Image /> автоматично оптимізує зображення (lazy loading, responsive, формати WebP/AVIF).

import Image from 'next/image';

<Image src="/hero.jpg" alt="Hero" width={800} height={400} />
8. Що таке Middleware у Next.js?

Middleware — це функції, що виконуються перед обробкою запиту сторінкою або API. Використовуються для редиректів, аутентифікації, логування.

// middleware.js
import { NextResponse } from 'next/server';
export function middleware(req) {
  if (!req.cookies.token) return NextResponse.redirect('/login');
}
9. Що таке Layouts і як їх реалізувати?

Layout — це шаблон сторінки (header, footer, sidebar), який повторюється. Його можна створити через обгортку або власний компонент.

function Layout({ children }) {
  return (
    <>
      <Header />
      <main>{children}</main>
      <Footer />
    </>
  );
}
10. Як реалізувати редирект у Next.js?
export async function getServerSideProps() {
  return {
    redirect: {
      destination: '/login',
      permanent: false,
    },
  };
}
11. Як відбувається деплой Next.js-додатку?
  • Найпростіше — через Vercel (автоматичний деплой з GitHub).
  • Також можливий деплой через Docker або Node.js сервер (npm run build → npm start).
  • Для SSG можна експортувати статику через next export.
12. Як оптимізувати продуктивність у Next.js?
  • Використовувати <Image> для оптимізації зображень.
  • Кешувати дані (ISR, SWR, React Query).
  • Динамічно імпортувати компоненти: dynamic(() => import('./Chart')).
  • Використовувати CDN і prefetch лінки.
13. Яка різниця між Next.js App Router і Pages Router?
  • Pages Router — класичний підхід через /pages (до версії 13).
  • App Router — новий підхід (з Next.js 13) через /app, що базується на React Server Components.
  • App Router підтримує server actions, layouts, streaming.

Middle — Next.js

1. Що таке Dynamic Import у Next.js і навіщо він потрібен?

Dynamic import дозволяє динамічно завантажувати компоненти лише тоді, коли вони потрібні — це зменшує розмір початкового бандлу.

import dynamic from 'next/dynamic';
const Chart = dynamic(() => import('../components/Chart'), { ssr: false });
2. Що таке SWR і для чого його використовують?

SWR (stale-while-revalidate) — це React-хук від Vercel для кешування даних та автоматичного оновлення.

import useSWR from 'swr';
const { data, error } = useSWR('/api/user', fetcher);
3. Як у Next.js реалізується 404 сторінка?

Потрібно створити файл pages/404.js — він автоматично використовується для неіснуючих маршрутів.

4. Як передати глобальні стилі або Layout на всі сторінки?

Глобальні стилі додаються у _app.js або через app/layout.js (App Router).

import '../styles/global.css';
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />;
}
5. Як налаштувати environment variables у Next.js?

Через файли .env.local, .env.development, .env.production.

NEXT_PUBLIC_API_URL=https://api.example.com

Доступ у коді: process.env.NEXT_PUBLIC_API_URL.

6. Як реалізувати редирект або rewrite у next.config.js?
module.exports = {
  async redirects() {
    return [{ source: '/old', destination: '/new', permanent: true }];
  },
  async rewrites() {
    return [{ source: '/blog/:slug', destination: '/api/post/:slug' }];
  },
};
7. Як зробити кешування сторінок або даних у Next.js?

Для SSG можна додати параметр revalidate у getStaticProps:

export async function getStaticProps() {
  const data = await fetchData();
  return { props: { data }, revalidate: 10 };
}
8. Як організувати приватні сторінки (auth guard) у Next.js?

Можна реалізувати через Middleware або умовний рендеринг у getServerSideProps.

export async function getServerSideProps({ req }) {
  if (!req.cookies.token) return { redirect: { destination: '/login', permanent: false } };
  return { props: {} };
}
9. Як налаштувати глобальні метатеги для SEO?

Через компонент <Head> або в layout.js (App Router):

import Head from 'next/head';
<Head>
  <title>My App</title>
  <meta name="description" content="Next.js SEO Example" />
</Head>
10. Як у Next.js обробляти помилки?

Створити файл pages/_error.js або app/error.js (App Router). Також можна використовувати ErrorBoundary для локального відлову помилок.

Advanced — Next.js

1. Що таке React Server Components і як вони використовуються в Next.js 13+?

React Server Components (RSC) рендеряться на сервері без завантаження JavaScript у браузер, що зменшує розмір бандлу та підвищує швидкість.

2. Що таке Server Actions у Next.js?

Server Actions дозволяють виконувати серверний код без API Route. Використовуються для обробки форм або дій без створення окремого endpoint.

"use server";
export async function addPost(data) {
  await db.post.create({ data });
}
3. Як працює кешування у Next.js App Router?

Next.js автоматично кешує результат SSR/SSG запитів. Можна керувати поведінкою через fetch:

await fetch('/api/data', { cache: 'no-store' }); // без кешу
await fetch('/api/data', { next: { revalidate: 60 } }); // ISR
4. Що таке Route Groups?

Route Groups ((group)) дозволяють групувати маршрути без впливу на URL. Це зручно для layout’ів і структуризації великих проєктів.

/app/(marketing)/page.js → /
/app/(dashboard)/page.js → /
5. Що таке Edge Functions у Next.js?

Edge Functions виконуються ближче до користувача (на edge-серварах CDN) для зниження затримки. Використовуються для перевірки токенів, редиректів або легких запитів.

6. Як реалізується streaming у Next.js?

Streaming дозволяє відображати сторінку частинами, не чекаючи повного SSR-рендеру. Застосовується для швидшого показу контенту.

7. Як працює SEO-оптимізація у App Router?

У App Router метадані описуються через об’єкт metadata або функцію generateMetadata().

export const metadata = {
  title: "Blog",
  description: "Next.js metadata API"
};
8. Як у Next.js реалізується інтернаціоналізація (i18n)?

Через вбудовану опцію i18n у next.config.js або бібліотеки next-intl, react-intl.

9. Як порівняти Next.js з Nuxt і Remix?
  • Next.js — React, SSR + SSG, Vercel.
  • Nuxt.js — аналог для Vue.js.
  • Remix — ближчий до класичного fullstack з акцентом на forms і routes.
10. Як контролювати розмір бандлу і оптимізувати його?
  • Використовувати next/script з атрибутами lazyOnload.
  • Використовувати dynamic import і Tree-Shaking.
  • Аналізувати через next build --analyze.
  • Мінімізувати глобальні залежності.

Senior / Expert — Next.js

1. Як працює App Router порівняно з Pages Router на низькому рівні?

App Router базується на React Server Components (RSC) і працює через server + client boundary. Кожна сторінка тепер — асинхронний серверний компонент, який може рендерити частини UI незалежно. Pages Router — це client-side рендеринг з SSR/SSG підтримкою через старі API.

2. Як працює parallel routes і для чого вони потрібні?

Parallel routes дозволяють рендерити кілька незалежних сторінок одночасно в різних слотах layout-а. Використовується для панелей, модалок, dashboard layout’ів.

/app/@modal/(.)photo/[id]/page.js
/app/@feed/page.js
3. Як реалізується data fetching у App Router без getServerSideProps?

Усі серверні компоненти можуть виконувати async fetch безпосередньо всередині себе:

export default async function Page() {
  const data = await fetch('https://api.example.com/posts').then(r => r.json());
  return <Posts data={data} />;
}
4. Як використовуються React Cache і Suspense у Next.js?

Next.js використовує React Suspense для асинхронного завантаження компонентів і даних. Кешування даних відбувається через react.cache() або fetch з опцією next: { revalidate }.

5. Як працює Client Component vs Server Component?
  • Server Component рендериться на сервері, не містить JS у клієнті.
  • Client Component має директиву "use client" і дозволяє використовувати хуки, стани, ефекти.
"use client";
import { useState } from 'react';
export default function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}
6. Як оптимізувати SSR-рендеринг у великих Next.js додатках?
  • Використовувати streaming і Suspense.
  • Переходити на React Server Components для зменшення JS.
  • Виносити важкі запити у edge functions або ISR.
  • Використовувати кешування CDN і HTTP headers (Cache-Control).
7. Як організувати багатомовність з App Router (i18n routing)?

Через сегменти маршруту:

/app/[lang]/page.js → /en /ua /pl

Дані локалі можна передавати через middleware або context provider.

8. Як працюють Static Params у App Router?

Функція generateStaticParams() генерує шляхи для SSG на етапі білду.

export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts').then(r => r.json());
  return posts.map(post => ({ id: post.id }));
}
9. Як реалізувати custom server у Next.js?

Можна створити власний Node.js або Express сервер, який інтегрує Next.js middleware:

import next from 'next';
import express from 'express';
const app = next({ dev: true });
const handle = app.getRequestHandler();
app.prepare().then(() => {
  const server = express();
  server.get('/hello', (req, res) => res.send('Hello!'));
  server.all('*', (req, res) => handle(req, res));
  server.listen(3000);
});
10. Як організувати монорепозиторій з Next.js?

Next.js добре працює з Turborepo — інструментом для монорепозиторіїв від Vercel.

  • Кожен пакет (frontend, shared, api) має власний package.json.
  • Загальні компоненти зберігаються у /packages/ui.
  • Кешування збірок та lint’у через Turborepo pipeline.
{
  "pipeline": {
    "build": { "outputs": [".next/**"] },
    "lint": {}
  }
}
