Запитання та відповіді для технічної співбесіди (Бази даних)

Junior рівень

1. Що таке база даних?

База даних — це організована структура для зберігання, управління та отримання інформації. Вона дозволяє ефективно зберігати великі обсяги даних і виконувати над ними операції (вибірка, оновлення, видалення тощо).

Приклади: MySQL, PostgreSQL, MongoDB, SQLite, Redis.

2. У чому різниця між SQL і NoSQL базами даних?
  • SQL (реляційні) — дані зберігаються у вигляді таблиць із рядками та стовпцями. Підтримують мову запитів SQL. Приклад: MySQL, PostgreSQL.
  • NoSQL (нереляційні) — дані зберігаються у форматі документів, ключ-значення, графів чи колонок. Приклад: MongoDB, Redis, Cassandra.
SQL: SELECT * FROM users WHERE age > 18;
NoSQL (MongoDB): db.users.find({ age: { $gt: 18 } });

Основна різниця: SQL — структуровані дані з фіксованими зв’язками, NoSQL — гнучкі структури без жорсткої схеми.

3. Що таке CRUD?

CRUD — це базові операції з даними у будь-якій базі даних:

  • C — Create (створення запису)
  • R — Read (зчитування)
  • U — Update (оновлення)
  • D — Delete (видалення)
-- SQL приклад
INSERT INTO users (name) VALUES ('Serhii');
SELECT * FROM users;
UPDATE users SET name='Ivan' WHERE id=1;
DELETE FROM users WHERE id=1;
4. Що таке первинний (Primary Key) і зовнішній (Foreign Key) ключ?
  • Primary Key — унікальний ідентифікатор запису в таблиці (наприклад, id).
  • Foreign Key — посилання на первинний ключ іншої таблиці (зв’язок між таблицями).
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50)
);

CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id)
);
5. Які типи JOIN ви знаєте?
  • INNER JOIN — повертає тільки ті записи, які мають збіг у обох таблицях.
  • LEFT JOIN — усі записи з лівої таблиці + збіги з правої (якщо є).
  • RIGHT JOIN — навпаки, усі з правої таблиці + збіги з лівої.
  • FULL OUTER JOIN — усі записи з обох таблиць.
SELECT users.name, orders.id
FROM users
LEFT JOIN orders ON users.id = orders.user_id;
6. Що таке індекс у базі даних і навіщо він потрібен?

Індекс — це структура, яка прискорює пошук даних у таблиці, подібно до алфавітного покажчика у книзі.

Недолік: індекси сповільнюють операції вставки та оновлення, бо їх треба оновлювати.

CREATE INDEX idx_user_email ON users(email);
7. Що таке нормалізація бази даних?

Нормалізація — це процес організації таблиць і зв’язків для уникнення дублювання даних.

Основні форми:

  • 1NF — усі поля атомарні (одне значення в комірці).
  • 2NF — усі неключові атрибути залежать від первинного ключа.
  • 3NF — немає транзитивних залежностей між неключовими полями.
8. Що таке транзакція і які в неї властивості (ACID)?

Транзакція — це набір операцій, які виконуються як єдине ціле. Якщо одна з них не вдалася — усі скасовуються.

ACID:

  • Atomicity — усе або нічого.
  • Consistency — дані залишаються узгодженими.
  • Isolation — транзакції не впливають одна на одну.
  • Durability — після завершення транзакції зміни зберігаються назавжди.
9. Що таке MongoDB і чим вона відрізняється від SQL баз?

MongoDB — це документно-орієнтована NoSQL база даних. Дані зберігаються у форматі JSON-подібних документів замість таблиць.

  • Немає фіксованої схеми.
  • Легко масштабувати горизонтально (через шардінг).
  • Ідеально для гнучких структур даних — наприклад, користувацьких профілів або логів.
db.users.insertOne({ name: "Serhii", age: 36, skills: ["JS", "React"] });
10. Що таке ORM?

ORM (Object-Relational Mapping) — це технологія, що дозволяє працювати з базами даних через об’єкти, а не через SQL-запити вручну.

Приклади: Sequelize (Node.js), TypeORM, Prisma, Mongoose.

// Приклад із Sequelize
User.findAll({ where: { active: true } });

🟡 Middle рівень

11. У чому різниця між нормалізацією та денормалізацією?

Нормалізація — це поділ таблиць для усунення дублювання даних і забезпечення узгодженості.

Денормалізація — навпаки, об'єднує таблиці або дублює частину даних, щоб прискорити читання.

Нормалізація: таблиці users і addresses окремо.
Денормалізація: зберігаємо адресу прямо в users.
12. Для чого використовуються GROUP BY і HAVING?

GROUP BY групує результати запиту за певним полем, а HAVING застосовується для фільтрації вже згрупованих результатів.

SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 10;
13. У чому різниця між кластерним і некластерним індексом?
  • Кластерний — фізично сортує записи у таблиці (один на таблицю).
  • Некластерний — створює окрему структуру для швидкого пошуку (можна кілька).
CREATE CLUSTERED INDEX idx_users_id ON users(id);
CREATE NONCLUSTERED INDEX idx_users_email ON users(email);
14. Як працюють транзакції у MongoDB?

Починаючи з MongoDB 4.0, підтримуються мульти-документні транзакції. Вони працюють подібно до SQL, але виконуються у межах однієї реплікації або шард-кластера.

const session = client.startSession();
session.startTransaction();
try {
  await users.insertOne({ name: "Serhii" }, { session });
  await orders.insertOne({ user: "Serhii" }, { session });
  await session.commitTransaction();
} catch {
  await session.abortTransaction();
}
15. Як працюють індекси у MongoDB?

Індекси створюються по ключах документів і дозволяють швидко знаходити записи без сканування всієї колекції.

За замовчуванням створюється індекс по _id.

db.users.createIndex({ email: 1 }); // 1 - зростання, -1 - спадання
16. Що таке Aggregation Pipeline у MongoDB?

Aggregation Pipeline — це послідовність етапів обробки даних (подібно до SQL GROUP BY, JOIN, WHERE).

db.orders.aggregate([
  { $match: { status: "paid" } },
  { $group: { _id: "$userId", total: { $sum: "$amount" } } }
]);
17. Для чого використовуються LIMIT і OFFSET?

LIMIT обмежує кількість результатів, а OFFSET пропускає певну кількість рядків (для пагінації).

SELECT * FROM users LIMIT 10 OFFSET 20;
18. Що таке VIEW у SQL?

VIEW — це віртуальна таблиця, створена на основі запиту. Вона не зберігає дані, лише посилання на інші таблиці.

CREATE VIEW active_users AS
SELECT name, email FROM users WHERE active = true;

🔵 Middle+ / Senior рівень

19. Що таке EXPLAIN і навіщо його використовують?

EXPLAIN показує, як СУБД виконує запит — чи використовує індекси, скільки рядків сканує, у якому порядку об’єднує таблиці.

EXPLAIN ANALYZE SELECT * FROM users WHERE email='test@gmail.com';
20. Як оптимізувати SQL-запити?
  • Використовувати індекси для часто фільтрованих полів.
  • Уникати SELECT *.
  • Використовувати EXPLAIN для аналізу плану запиту.
  • Кешувати результати (наприклад, Redis).
21. Що таке Redis і як він використовується?

Redis — це in-memory база даних типу “ключ-значення”, використовується для кешування, черг і швидкого доступу до даних.

SET user:1 "Serhii"
GET user:1
22. Що таке реплікація у базах даних?

Реплікація — це копіювання даних з головного (master) сервера на підлеглі (replica) сервери.

Переваги: відмовостійкість, балансування навантаження, швидше читання.

23. Що таке шардінг?

Шардінг — це горизонтальний поділ бази даних на частини (шарди), щоб розподілити навантаження між кількома серверами.

Шард 1 — користувачі з id 1–1000
Шард 2 — користувачі з id 1001–2000
24. Що таке CAP theorem?

CAP теорема стверджує, що у розподіленій системі одночасно можна забезпечити лише 2 з 3 властивостей:

  • Consistency — усі вузли бачать однакові дані;
  • Availability — система завжди відповідає на запит;
  • Partition tolerance — система працює навіть при розриві з’єднання.

Приклади: MongoDB — AP, PostgreSQL — CA.

25. У чому різниця між ACID і BASE?
  • ACID — для реляційних БД, гарантує стабільність і узгодженість.
  • BASE — для NoSQL, забезпечує масштабованість і швидкість, але без повної консистентності.
ACID: PostgreSQL, MySQL
BASE: MongoDB, Cassandra
