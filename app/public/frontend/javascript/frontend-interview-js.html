Запитання та відповіді для технічної співбесіди (JavaScript — Основи)

Основи JavaScript

1. Які типи даних існують у JavaScript?

У JavaScript є 8 основних типів даних:

  • Number — числовий тип (включно з NaN та Infinity).
  • String — рядки тексту.
  • Boolean — логічні значення: true або false.
  • Object — колекція властивостей і значень.
  • null — спеціальне значення, що означає “нічого” або “порожньо”.
  • undefined — змінна оголошена, але їй не присвоєно значення.
  • Symbol — унікальний ідентифікатор, який не може бути повторений.
  • BigInt — використовується для представлення великих цілих чисел.
const bigInt = 1234567890123456789012345678901234567890n;
console.log(typeof bigInt); // bigint
2. У чому різниця між операторами == і ===?

== виконує порівняння з приведенням типів — перед порівнянням намагається перетворити значення до спільного типу.

=== виконує строге порівняння — без приведення типів. Значення вважаються рівними лише якщо збігаються і тип, і значення.

console.log(5 == "5");   // true  (рядок перетворюється на число)
console.log(5 === "5");  // false (різні типи)
console.log(null == undefined); // true
console.log(null === undefined); // false
3. У чому різниця між null і undefined?

Обидва представляють “порожнє” значення, але мають різне призначення:

  • undefined — означає, що змінна оголошена, але не має присвоєного значення. Це значення встановлюється автоматично.
  • null — спеціальне значення, яке ми присвоюємо самостійно, щоб позначити, що змінна порожня або не має значення.
let a;
console.log(a); // undefined (значення не присвоєно)

a = null;
console.log(a); // null (явно встановлено порожнє значення)

Висновок: undefined використовується, коли значення ще не визначено, а null — коли ми самі задаємо “порожнє” значення.

4. Якими способами можна оголосити змінну у JavaScript?

Оголосити змінну можна кількома способами:

  1. a = 5; — неявне оголошення (глобальна змінна, не рекомендується).
  2. var a = 5; — функціональна область видимості, має hoisting.
  3. let a = 5; — блочна область видимості, може змінюватися.
  4. const a = 5; — блочна область видимості, значення незмінне.

var вважається застарілим способом. let і const — сучасні способи оголошення змінних із блочною областю дії.

У разі const для об’єктів чи масивів можна змінювати їхній вміст, але не саму змінну:

const arr = [1, 2, 3];
arr.push(4); // ✅ можна
arr = [4, 5]; // ❌ TypeError
5. Яка різниця між ECMAScript та JavaScript?

ECMAScript — це стандарт мови, а JavaScript — його реалізація у браузерах. Інші реалізації: JScript, ActionScript.

6. Що таке динамічна типізація? Як її ще називають?

У JS тип змінної визначається під час виконання, а не оголошення. Це називають динамічною або слабкою типізацією.

let x = 10; x = "Привіт"; // дозволено
7. Які є варіанти додавання JS коду до сторінки?
  • Вбудований у тег: <script>alert("Hi")</script>
  • Зовнішній файл: <script src="app.js"></script>
  • Атрибут події: <button onclick="doSomething()">
8. Які є обмеження для коду JS, який запускається в браузері?
  • Не має прямого доступу до файлової системи.
  • Не може взаємодіяти з іншими доменами (через політику Same-Origin).
  • Працює в межах sandbox середовища.
9. Яка різниця між компільованою та інтерпретованою мовою? До якої відноситься JS?

Компільовані мови перетворюються у машинний код перед виконанням (C++, Java). Інтерпретовані виконуються построково (Python, JS). JS — інтерпретована, але сучасні рушії (V8) виконують JIT-компіляцію для прискорення.

Масиви

1. Яка різниця між методами splice() та slice()?
  • splice() — змінює оригінальний масив (видаляє, додає елементи).
  • slice() — створює копію частини масиву, не змінюючи його.
const arr = [1,2,3,4];
console.log(arr.slice(1,3)); // [2,3]
arr.splice(1,2); // видаляє 2 і 3
console.log(arr); // [1,4]
2. Яким чином можна створити копію всього масиву?

Є кілька способів створити копію масиву:

const arr = [1,2,3];
const copy1 = arr.slice();
const copy2 = [...arr];
const copy3 = Array.from(arr);

Усі три методи створюють новий масив, не змінюючи оригінал.

3. Яка різниця між методами filter() та find()?
  • filter() — повертає масив усіх елементів, що проходять умову.
  • find() — повертає перший елемент, що проходить умову, або undefined.
const arr = [1, 2, 3, 4];
arr.filter(x => x > 2); // [3, 4]
arr.find(x => x > 2);   // 3
4. Як працює метод reduce()?

reduce() проходить по всьому масиву, накопичуючи результат у “акумулятор”.

const arr = [1,2,3,4];
const sum = arr.reduce((acc, curr) => acc + curr, 0);
console.log(sum); // 10

Перший параметр — функція з акумулятором і поточним значенням, другий — початкове значення.

5. Яким чином можна визначити кількість негативних елементів у масиві?

Можна використати filter() і властивість length:

const arr = [1, -3, 5, -2, -8];
const count = arr.filter(x => x < 0).length;
console.log(count); // 3

Цикли (Loops)

1. У чому відмінність між циклами while та do-while?
  • while — перевіряє умову перед виконанням тіла циклу.
  • do...while — виконує тіло циклу хоча б один раз, навіть якщо умова хибна.
let i = 0;
do {
  console.log(i);
  i++;
} while (i < 0); // виконається 1 раз
2. Який оператор використовується для пропуску ітерації в циклі?

Для пропуску поточної ітерації використовується оператор continue.

for (let i = 0; i < 5; i++) {
  if (i === 2) continue;
  console.log(i); // 0, 1, 3, 4
}
3. Для чого потрібний оператор break?

Оператор break зупиняє виконання циклу достроково.

for (let i = 0; i < 5; i++) {
  if (i === 3) break;
  console.log(i); // 0, 1, 2
}
4. У чому різниця між for-in та for-of?
  • for...in — проходить по ключах (індексах) об’єкта або масиву.
  • for...of — проходить по значеннях елементів ітерабельного об’єкта.
const arr = ['a', 'b', 'c'];
for (let i in arr) console.log(i); // 0,1,2
for (let val of arr) console.log(val); // a,b,c
5. Який порядок виконання циклу for?

Цикл for має три частини:

  1. Ініціалізація — виконується один раз перед початком.
  2. Умова — перевіряється перед кожною ітерацією.
  3. Ітерація (оновлення) — виконується після кожного проходу.
for (let i = 0; i < 3; i++) {
  console.log(i);
}
// порядок: ініціалізація → перевірка → тіло → оновлення

Функції (Functions)

1. У чому різниця між function declaration і function expression?

🔹 Function Declaration

function sayHi() {
  console.log("Привіт!");
}
sayHi(); // Привіт!

Такі функції обробляються до виконання коду (hoisting). Вони доступні в усьому блоці, навіть до оголошення.

🔹 Function Expression

const sayHi = function() {
  console.log("Привіт!");
};
sayHi(); // Привіт!

Тут функція створюється під час виконання програми і присвоюється змінній. Вона не “піднімається” (не доступна до моменту ініціалізації).

Підсумок: Function Declaration піддається hoisting, Function Expression — ні.

2. Що таке стрілкові функції, і чим вони відрізняються від звичайних функцій?
  • Не мають власного this — беруть this з контексту, де були створені (лексичне this).
  • Не мають arguments — немає доступу до списку аргументів виклику.
  • Не можуть бути конструкторами — не можна використовувати з new.
  • Не мають super — не можуть використовувати ключове слово super (детальніше — у темі про наслідування класів).
  • Завжди анонімні — не мають власного імені, зручні для колбеків.
  • Мають коротший та компактніший синтаксис.
// Звичайна функція
function show() {
  console.log(this);
}

// Стрілкова функція
const showArrow = () => {
  console.log(this);
};

Висновок: Стрілкові функції — компактна альтернатива для простих задач, але не підходять там, де потрібно власне this, arguments, super, або виклик через new.

3. Що таке hoisting у контексті функцій?

Hoisting — це підняття оголошень функцій і змінних на початок області видимості. Function Declaration можна викликати до оголошення, Function Expression — ні.

5. Що таке колбек (callback) функція?

Це функція, яку передають як аргумент іншій функції для виконання після певної події або дії.

function greet(name, callback) {
  console.log("Hi " + name);
  callback();
}

function done() {
  console.log("Greeting complete");
}

greet("Serhii", done);
6. Що таке рекурсія?

Рекурсія — це коли функція викликає саму себе. Використовується для повторюваних завдань, наприклад обходу дерев або обчислення факторіалу.

function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}
console.log(factorial(5)); // 120
7. Що таке анонімна функція?

Функція без імені, зазвичай використовується у виразах або як callback.

setTimeout(function() {
  console.log("Через 2 секунди");
}, 2000);
8. Що таке IIFE (Immediately Invoked Function Expression)?

Це функція, яка виконується одразу після свого оголошення. Використовується для створення локальної області видимості.

(function() {
  console.log("IIFE виконано");
})(); // ✅
9. Що таке аргументи за замовчуванням у функції?

Це значення, які функція використовує, якщо під час виклику аргументи не були передані.

function greet(name = "Гість") {
  console.log("Привіт, " + name);
}
greet(); // Привіт, Гість
10. Що таке pure function?

Pure function — це функція, яка при однакових вхідних даних завжди повертає однаковий результат і не має побічних ефектів.

function sum(a, b) {
  return a + b; // не змінює зовнішні змінні
}

Об’єкти (Objects)

1. Що таке об’єкт у JavaScript?

Об’єкт — це структура даних, що зберігає пари ключ: значення. Ключі — це рядки або символи, значення — будь-який тип даних.

const user = {
  name: "Serhii",
  age: 36,
  active: true
};
console.log(user.name); // "Serhii"
2. Як додати або видалити властивість об’єкта?
const car = { brand: "BMW" };
car.color = "black"; // додати
delete car.brand;    // видалити
3. Як перевірити, чи існує властивість в об’єкті?

Є два основні способи:

  • Метод hasOwnProperty() — перевіряє, чи властивість належить об’єкту напряму (а не через прототип).
  • Оператор in — перевіряє наявність властивості в усьому ланцюжку прототипів.
const obj = { a: 5, b: "string" };

// Перший спосіб
console.log(obj.hasOwnProperty("a")); // true

// Другий спосіб
console.log("b" in obj); // true

Рекомендація: використовуй hasOwnProperty() для точнішої перевірки власних властивостей об’єкта.

4. У чому різниця між == і === при порівнянні об’єктів?

Об’єкти завжди порівнюються за посиланням, а не за вмістом.

const a = { name: "Serhii" };
const b = { name: "Serhii" };
console.log(a === b); // false (різні посилання)
const c = a;
console.log(a === c); // true
5. Як перебрати всі властивості об’єкта?

Можна використати for...in або методи Object.keys(), Object.values(), Object.entries().

const user = { name: "Serhii", age: 36 };

for (let key in user) {
  console.log(key, user[key]);
}

Object.entries(user).forEach(([key, val]) => {
  console.log(key, val);
});
6. Як клонувати об’єкт?

Найпростіше — через оператор spread або Object.assign().

const obj = { a: 1, b: 2 };
const copy1 = { ...obj };
const copy2 = Object.assign({}, obj);

Для глибокого клонування — structuredClone() або JSON.parse(JSON.stringify(obj)).

8. Як об’єднати два об’єкти?

Можна використовувати spread або Object.assign():

const obj1 = { a: 1 };
const obj2 = { b: 2 };
const merged = { ...obj1, ...obj2 }; // { a:1, b:2 }
10. Що таке прототип об’єкта?

Кожен об’єкт у JavaScript має прототип ([[Prototype]]), який використовується для успадкування властивостей та методів.

const animal = { eats: true };
const dog = Object.create(animal);
console.log(dog.eats); // true (успадковано)

Асинхронність (Promises, async/await, Event Loop)

1. Що таке асинхронність у JavaScript?

Асинхронність дозволяє виконувати код без блокування основного потоку. JS працює в одному потоці (single-threaded), але завдяки Event Loop може обробляти події, запити, таймери тощо.

console.log('Start');
setTimeout(() => console.log('Async'), 0);
console.log('End');
// Порядок: Start → End → Async
2. Що таке Event Loop?

Event Loop — це механізм у JavaScript, який керує виконанням асинхронного коду. JS — однопотокова мова, тому Event Loop дозволяє виконувати callback-и після завершення основного потоку (stack).

🔹 Принцип роботи:

  • Код у main thread виконується послідовно (синхронно).
  • Асинхронні завдання (таймери, проміси, API) потрапляють у чергу подій.
  • Після завершення основного коду, Event Loop перевіряє чергу та виконує завдання по черзі.
console.log("A");

setTimeout(() => console.log("B"), 0);

Promise.resolve().then(() => console.log("C"));

console.log("D");

// Вивід: A → D → C → B

Пояснення: Проміси (microtasks) виконуються раніше, ніж таймери (macrotasks).

3. Що таке Promise у JavaScript?

Promise — це спеціальний об’єкт, який використовується для роботи з асинхронним кодом. Він представляє значення, яке може бути отримане зараз, пізніше або ніколи.

Promise має три стани:

  • pending — очікування (обіцянка ще не виконана);
  • fulfilled — виконано успішно;
  • rejected — виконано з помилкою.

До промісу можна підключати обробники двох типів:

  • onFulfilled — викликається при успішному виконанні;
  • onRejected — викликається, якщо сталася помилка.

Загальна схема роботи Promise:

  1. Асинхронний код створює Promise і повертає його.
  2. Зовнішній код додає до Promise обробники через .then() або .catch().
  3. Коли асинхронна операція завершується, викликається відповідний обробник.
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("✅ Виконано успішно!");
    // або reject("❌ Помилка!");
  }, 1000);
});

promise
  .then(result => console.log(result))
  .catch(error => console.error(error));
4. Як обробити помилку в Promise?

За допомогою .catch() або блоку try...catch у async функції.

fetch('wrong-url')
  .then(res => res.json())
  .catch(err => console.error('Error:', err));
5. У чому різниця між async/await та then/catch?

async/await — це синтаксичний цукор над Promise, який робить асинхронний код більш читабельним і послідовним.

async function loadData() {
  try {
    const res = await fetch('https://api.example.com');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
6. Що робить Promise.all()?

Виконує кілька промісів паралельно і повертає результат, коли всі вони завершаться. Якщо один із них відхилиться — весь Promise.all() відхиляється.

Promise.all([
  fetch('/user'),
  fetch('/posts')
]).then(res => console.log('Done!'));
7. Яка різниця між Promise.all(), Promise.race() і Promise.allSettled()?
  • Promise.all() — чекає всі або помилку.
  • Promise.race() — повертає перший завершений (успіх чи помилка).
  • Promise.allSettled() — повертає результати всіх, незалежно від помилок.
8. Що таке microtasks і macrotasks у JS?
  • Microtasks — обробляються після поточного стека, перед макротасками (Promise, queueMicrotask).
  • Macrotasks — виконуються після мікротасок (setTimeout, setInterval, I/O).
setTimeout(() => console.log('Macro'));
Promise.resolve().then(() => console.log('Micro'));
console.log('Sync');
// Sync → Micro → Macro
9. Як зробити затримку (delay) за допомогою async/await?
function delay(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function run() {
  console.log('Початок');
  await delay(1000);
  console.log('Через 1 секунду');
}
run();

DOM та події (Events, Bubbling, Delegation)

1. Що таке DOM?

DOM (Document Object Model) — це об’єктне представлення HTML-документа, яке дозволяє JavaScript змінювати структуру, стилі й вміст сторінки.

document.body.style.background = 'lightblue';
document.getElementById('title').textContent = 'Hello DOM!';
2. Як знайти елемент у DOM?
  • getElementById() — за id
  • querySelector() — за CSS-селектором
  • querySelectorAll() — всі елементи, що підходять під селектор
const el = document.querySelector('.btn');
const all = document.querySelectorAll('p');
3. Як створити або видалити елемент у DOM?

Для створення використовують document.createElement(), для вставки — append(), prepend(), для видалення — remove().

const div = document.createElement('div');
div.textContent = 'Новий елемент';
document.body.append(div);

div.remove(); // видалити
4. Що таке подія (event) у JavaScript?

Подія — це реакція браузера на дію користувача (клік, наведення, натискання клавіші, прокрутка тощо).

document.querySelector('button').addEventListener('click', () => {
  alert('Кнопку натиснуто!');
});
5. Що таке event bubbling і capturing?
  • Capturing — подія йде згори вниз (від window → до target).
  • Bubbling — подія піднімається знизу вгору (від target → до window).
div.addEventListener('click', () => console.log('DIV'));
button.addEventListener('click', () => console.log('BUTTON'));
// Натискання на
6. Що таке делегування подій?

Делегування — це техніка, коли один обробник подій ставиться на спільного батька, і він реагує на події дочірніх елементів за допомогою event.target.

document.querySelector('ul').addEventListener('click', e => {
  if (e.target.tagName === 'LI') {
    console.log('Натиснуто пункт:', e.target.textContent);
  }
});

Перевага — менше слухачів подій, ефективніше управління DOM.

7. Як зупинити "спливання" події?

Метод event.stopPropagation() зупиняє подальше поширення події вгору по DOM.

button.addEventListener('click', e => {
  e.stopPropagation();
  console.log('Тільки кнопка!');
});
8. Для чого використовується preventDefault()?

Метод event.preventDefault() блокує стандартну поведінку елемента (наприклад, перехід за посиланням або перезавантаження форми).

document.querySelector('a').addEventListener('click', e => {
  e.preventDefault();
  console.log('Посилання заблоковано');
});
9. У чому різниця між target і currentTarget?
  • event.target — елемент, на якому подія фактично сталася.
  • event.currentTarget — елемент, на якому висить обробник.
div.addEventListener('click', e => {
  console.log('target:', e.target.tagName);
  console.log('currentTarget:', e.currentTarget.tagName);
});
10. Як дізнатися, яка клавіша була натиснута?

Подія клавіатури передає об’єкт event з властивістю key.

document.addEventListener('keydown', e => {
  console.log('Key pressed:', e.key);
});

Масиви та методи (map, filter, reduce, sort, forEach, find...)

1. Як створити масив у JavaScript?

Масив можна створити двома способами:

const arr1 = [1, 2, 3];
const arr2 = new Array(1, 2, 3);
2. Як перебрати елементи масиву?

Найпоширеніші способи перебору:

const nums = [1, 2, 3];

// for
for (let i = 0; i < nums.length; i++) console.log(nums[i]);

// for...of
for (let n of nums) console.log(n);

// forEach
nums.forEach(n => console.log(n));
3. У чому різниця між forEach() і map()?
  • forEach() — виконує дію для кожного елемента (нічого не повертає).
  • map() — створює новий масив із результатів функції.
const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2);
console.log(doubled); // [2, 4, 6]
4. Як відфільтрувати масив?

Метод filter() повертає новий масив, що містить лише ті елементи, які пройшли перевірку:

const nums = [1, 2, 3, 4, 5];
const even = nums.filter(n => n % 2 === 0);
console.log(even); // [2, 4]
5. Як знайти конкретний елемент у масиві?

Метод find() повертає перший елемент, який задовольняє умову.

const users = [{id: 1}, {id: 2}, {id: 3}];
const found = users.find(u => u.id === 2);
console.log(found); // {id: 2}
6. Як перевірити, чи масив містить певне значення?
const nums = [1, 2, 3];
console.log(nums.includes(2)); // true
console.log(nums.includes(5)); // false
7. Як об’єднати масиви?

Можна використати concat() або оператор spread:

const a = [1, 2];
const b = [3, 4];
const merged = [...a, ...b]; // [1, 2, 3, 4]
8. Як відсортувати масив?

Метод sort() змінює оригінальний масив. Для чисел треба передати функцію порівняння:

const nums = [10, 2, 5, 1];
nums.sort((a, b) => a - b);
console.log(nums); // [1, 2, 5, 10]
9. Як обчислити суму або інше агреговане значення?

Метод reduce() накопичує значення у єдиний результат.

const nums = [1, 2, 3, 4];
const sum = nums.reduce((acc, cur) => acc + cur, 0);
console.log(sum); // 10
10. Як перевірити, чи всі/деякі елементи відповідають умові?
  • every() — true, якщо всі елементи задовольняють умову.
  • some() — true, якщо хоч один елемент задовольняє умову.
const nums = [2, 4, 6];
console.log(nums.every(n => n % 2 === 0)); // true
console.log(nums.some(n => n > 5));        // true

Функції, Scope, Hoisting, Closure

1. Які є типи оголошення функцій у JavaScript?
  • Function Declaration — має hoisting.
  • Function Expression — створюється під час виконання.
  • Arrow Function — коротший синтаксис, не має власного this.
function sum(a, b) { return a + b; }       // Declaration
const mult = function(a, b) { return a*b;} // Expression
const sub = (a, b) => a - b;               // Arrow
2. Що таке scope (область видимості)?

Scope визначає, де змінна доступна у коді.

  • Global — доступна всюди.
  • Function — доступна лише в межах функції.
  • Block — для let і const, лише в межах блоку {}.
let x = 10;
function test() {
  let y = 20;
  console.log(x); // доступна
}
console.log(y); // помилка — змінна поза scope
3. Що таке hoisting?

Hoisting — це “підняття” оголошень функцій і змінних у верхню частину області видимості під час компіляції.

console.log(a); // undefined (а не помилка)
var a = 5;

sayHi(); // Працює
function sayHi() {
  console.log('Hello');
}

let і const теж “піднімаються”, але не ініціалізуються → ReferenceError.

5. Навіщо потрібні замикання?
  • Для збереження стану між викликами функцій.
  • Для створення приватних змінних (емуляція інкапсуляції).
  • Для реалізації callback або таймерів.
6. У чому різниця між var, let і const?
  • var — функціональний scope, має hoisting, дозволяє повторне оголошення.
  • let — блочний scope, без повторного оголошення.
  • const — блочний scope, значення не можна перевизначити.
var a = 1;
let b = 2;
const c = 3;
b = 5;   // ✅
c = 6;   // ❌ TypeError
7. Що таке функції вищого порядку (Higher-Order Functions)?

Функції, які приймають інші функції як аргументи або повертають функції.

function greet(name) {
  return () => console.log(`Hello, ${name}!`);
}

const hi = greet('Serhii');
hi(); // Hello, Serhii!
9. Що таке стрілочні функції та чим вони відрізняються?
  • Мають коротший синтаксис.
  • Не мають власного this, arguments, super.
  • Не можуть бути використані як конструктори.
const obj = {
  count: 0,
  inc: () => this.count++, // this не прив’язаний
  log: function() { this.count++; }
};
obj.inc(); // не працює
obj.log(); // працює
10. Що таке “callback hell” і як його уникнути?

“Callback hell” — це ситуація, коли виклики функцій вкладені один в одного, створюючи нечитабельний код.

Вирішується за допомогою Promises або async/await.

// Погано
doA(() => {
  doB(() => {
    doC(() => console.log('done'));
  });
});

// Добре
async function run() {
  await doA();
  await doB();
  await doC();
}
11. Що таке шаблонні літерали і для чого вони потрібні?

Шаблонні літерали — це рядки, оголошені у зворотних лапках (`), які дозволяють вставляти змінні або вирази без конкатенації, а також використовувати перенос рядків.

let b = 5;
const str = `Сума дорівнює ${b}`;
console.log(str); // "Сума дорівнює 5"

Переваги:

  • Можна вставляти змінні без + (через ${}).
  • Підтримується багаторядковість.
  • Зручніше для динамічного HTML або логів.
12. Що таке Set і Map у JavaScript?

Map — це структура даних для зберігання пар «ключ–значення». На відміну від об’єкта (Object), ключем може бути будь-який тип даних (рядок, число, об’єкт тощо).

const map = new Map();
map.set('name', 'Serhii');
map.set(1, 'number');
console.log(map.get('name')); // Serhii

Set — це колекція унікальних значень (без повторів). Кожне значення може з’являтися лише один раз.

const nums = new Set([1, 2, 2, 3]);
console.log(nums); // Set(3) {1, 2, 3}
nums.add(4);
nums.has(2); // true

Відмінність: Map зберігає пари ключ–значення, Set — лише унікальні значення.

13. Які існують способи створення об’єкта в JavaScript?

В JavaScript є кілька способів створення об’єктів:

🔹 1. За допомогою функції-конструктора

function User(name, surname) {
  this.name = name;
  this.surname = surname;
}

const user = new User('Ulbi', 'TV');
console.log(user); // User { name: 'Ulbi', surname: 'TV' }

🔹 2. За допомогою літеральної нотації

const myObject = {
  id: 1,
  name: "Sample"
};
console.log(myObject); // { id: 1, name: "Sample" }

🔹 3. За допомогою класу (ES6)

class User {
  constructor(name, surname) {
    this.name = name;
    this.surname = surname;
  }
}

const user = new User('Ulbi', 'TV');
console.log(user); // User { name: 'Ulbi', surname: 'TV' }

🔹 4. За допомогою Object.create()

const obj = Object.create({
  key: 'value'
});
console.log(obj.key); // value

Висновок: Найчастіше у сучасному коді використовують класи або літеральну нотацію, а Object.create() — коли потрібно задати конкретний прототип об’єкта.

14. Які значення вважаються falsy (хибними) у JavaScript?

Falsy — це значення, які при приведенні до логічного типу повертають false.

console.log(Boolean(false));      // false
console.log(Boolean(0));          // false
console.log(Boolean(""));         // false
console.log(Boolean(undefined));  // false
console.log(Boolean(null));       // false
console.log(Boolean(NaN));        // false
console.log(Boolean(0n));         // false (BigInt 0)

Список falsy-значень:

  • false
  • 0 або -0
  • Порожній рядок ""
  • null
  • undefined
  • NaN
  • 0n — BigInt нуль

Будь-яке інше значення (навіть порожній об’єкт чи масив) вважається truthy.

15. Як використовувати async/await для асинхронних запитів?

async / await — це синтаксичний цукор над Promise, що дозволяє писати асинхронний код у синхронному стилі.

  • async перед функцією означає, що вона завжди повертає Promise.
  • await змушує JavaScript чекати на виконання Promise перед продовженням коду.
async function getData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Помилка:", error);
  }
}

getData();

Примітка: await можна використовувати лише всередині функцій, позначених async.

16. Для чого потрібен оператор spread (...)?

Spread-оператор (три крапки) використовується для розгортання масивів або об’єктів. Він дозволяє швидко копіювати, комбінувати або передавати елементи без ручного перебору.

Використовується для:

  • Передачі аргументів у функцію: myFunc(...arr)
  • Копіювання або об’єднання масивів
  • Створення нових об’єктів на основі існуючих
// Масиви
const dateFields = [1970, 0, 1];
const newArray = [...dateFields, 2025];
console.log(newArray); // [1970, 0, 1, 2025]

// Об'єкти
const user = { name: "Serhii", age: 36 };
const extended = { ...user, city: "Kyiv" };
console.log(extended); // { name: "Serhii", age: 36, city: "Kyiv" }

Spread часто плутають із rest (...args), але різниця в тому, що spread — розгортає структуру, а rest — збирає кілька значень у масив.

17. Як уникнути посилальної залежності при копіюванні об’єкта?

Проблема: при простому присвоєнні або поверхневому копіюванні (наприклад, через = або {...obj}), новий об’єкт залишається пов’язаним із вихідним. Зміни у вкладених структурах впливатимуть на обидва.

🔹 Поверхневе копіювання

Якщо об’єкт не містить вкладених об’єктів, можна використати:

const obj = {
  key: 'value',
  field: 1
};

// Варіант 1
const copy = { ...obj };

// Варіант 2
const copy2 = Object.assign({}, obj);

🔹 Глибоке копіювання

Якщо об’єкт містить вкладені об’єкти:

const obj = {
  key: {
    field: 1
  }
};

Є кілька способів створити повну (глибоку) копію:

  1. Простий, але повільний спосіб: 
    const copy = JSON.parse(JSON.stringify(obj));

    ⚠️ Підходить лише для об’єктів без функцій і без прототипів.

  2. Реалізувати власну рекурсивну функцію копіювання (custom deepClone).
  3. Використати бібліотеку lodash — метод _.cloneDeep(obj).

Порада: завжди роби глибоке копіювання, якщо структура даних складна або має вкладені масиви/об’єкти.

18. Як змінити контекст функції у JavaScript?

Контекст — це значення this усередині функції. У JavaScript контекст можна змінити за допомогою методів bind(), call() або apply().

🔹 Метод bind()

Повертає нову функцію з «прив’язаним» контекстом, але не викликає її відразу.

function fn() {
  return this;
}

const obj = { name: "ULBI TV" };
const newFn = fn.bind(obj);

console.log(newFn()); // { name: 'ULBI TV' }

🔹 Методи call() і apply()

Викликають функцію з указаним контекстом одразу. Основна різниця між ними — у способі передачі аргументів:

  • call() — передає аргументи через кому;
  • apply() — приймає масив аргументів.
function greet(greeting, punctuation) {
  console.log(greeting + ', ' + this.name + punctuation);
}

const user = { name: 'Serhii' };

greet.call(user, 'Hello', '!');      // Hello, Serhii!
greet.apply(user, ['Hi', '!!!']);    // Hi, Serhii!!!

Висновок: - bind() — створює нову функцію з фіксованим this - call() та apply() — викликають функцію негайно

19. Що таке тернарний оператор у JavaScript?

Тернарний оператор — це коротка форма запису if...else. Його синтаксис використовує знак питання ? і двокрапку : та складається з трьох частин.

умова ? вираз_якщо_true : вираз_якщо_false

Приклад:

const age = 18;
const access = age >= 18 ? "Доступ дозволено" : "Доступ заборонено";
console.log(access); // "Доступ дозволено"

✅ Використовується для коротких умовних перевірок. ❌ Не рекомендується для складних логічних виразів — краще використовувати if...else.

20. Що таке деструктуризація у JavaScript?

Деструктуризація — це спеціальний синтаксис, який дозволяє «розпаковувати» значення з масивів або властивості об’єктів у окремі змінні.

🔹 Деструктуризація масиву

const arr = ["ULBI", "TV"];
const [firstName, surname] = arr;

console.log(firstName); // "ULBI"
console.log(surname);   // "TV"

🔹 Деструктуризація об’єкта

const options = {
  title: "Menu",
  width: 100,
  height: 200
};

const { title, width, height } = options;

console.log(title);  // "Menu"
console.log(width);  // 100
console.log(height); // 200

Переваги:

  • Зменшує кількість повторів коду;
  • Робить код читабельнішим;
  • Можна задавати значення за замовчуванням:
const user = { name: "Serhii" };
const { name, age = 36 } = user;
console.log(age); // 36
21. Які способи роботи з асинхронним кодом ви знаєте?

У JavaScript існує три основні способи обробки асинхронних подій:

🔹 1. Callback (функції зворотного виклику)

Класичний спосіб, коли функція викликається після завершення певної операції.

function loadScript(src, callback) {
  const script = document.createElement('script');
  script.src = src;

  script.onload = () => callback(script);

  document.head.append(script);
}

loadScript("app.js", () => console.log("Скрипт завантажено"));

⚠️ Недолік: при великій кількості вкладених викликів з’являється callback hell.

🔹 2. Promise

Обгортка над callback, яка дає можливість будувати послідовність асинхронних викликів.

new Promise((resolve, reject) => {
  setTimeout(() => resolve("data"), 500);
})
  .then(data => console.log(data))   // Виконано успішно
  .catch(err => console.error(err)); // Якщо сталася помилка

🔹 3. async / await

Сучасний синтаксис, який дозволяє писати асинхронний код у зручному «синхронному» стилі.

async function fetchTodos(url) {
  const resp = await fetch(url);
  const json = await resp.json();
  console.log(json);
}

fetchTodos("https://jsonplaceholder.typicode.com/todos/1");

Різниця: async/await — це просто «солодка» обгортка над Promise, яка робить код більш читабельним.

22. Для чого потрібні e.preventDefault() та e.stopPropagation()?

🔹 e.preventDefault()

Застосовується для запобігання стандартній поведінці браузера. Наприклад:

  • Клік по посиланню — не переходити за URL;
  • Натискання кнопки submit — не відправляти форму.
document.querySelector("a").addEventListener("click", function(e) {
  e.preventDefault();
  console.log("Перехід скасовано");
});

🔹 e.stopPropagation()

Зупиняє всплиття події (bubbling) по DOM-дереву. Подія не передається від дочірнього елемента до батьківського.

document.querySelector(".child").addEventListener("click", function(e) {
  e.stopPropagation();
  console.log("Клік лише по .child");
});

Всплиття подій — це коли подія піднімається від конкретного елемента до body і далі вгору по дереву.
За замовчуванням усі обробники спрацьовують «знизу догори».

Коротко:

  • preventDefault() — зупиняє дію браузера (але подія все одно «вспливає»).
  • stopPropagation() — зупиняє передачу події вгору по DOM.
23. Як відстежувати та обробляти помилки в JavaScript?

У JavaScript є спеціальна конструкція try...catch, яка дозволяє «ловити» помилки та обробляти їх без краху виконання програми.

🔹 Принцип роботи

  • Спочатку виконується код у блоці try.
  • Якщо помилок немає — блок catch пропускається.
  • Якщо у try виникає помилка — виконання зупиняється, і керування передається у catch(err).
try {
  // Пробуємо виконати код
  const data = JSON.parse("{ invalid JSON }");
  console.log(data);
} catch (err) {
  console.error("Помилка:", err.message);
} finally {
  console.log("Цей блок виконається завжди");
}

🔹 Пояснення:

  • try — код, який може викликати помилку.
  • catch(err) — обробляє помилку (змінна err містить об’єкт з інформацією).
  • finally — виконується завжди, незалежно від того, чи сталася помилка.

✅ Це основний спосіб безпечного виконання потенційно ризикованого коду, наприклад при роботі з API чи JSON.

24. Що таке DOM дерево?

DOM (Document Object Model) — це об’єктна модель HTML-документа, яка представляє сторінку у вигляді дерева елементів (вузлів).

Кожен HTML-тег є об’єктом. Вкладені теги — «діти» своїх батьківських тегів. Текст усередині тегів — також об’єкт.

HTML-документ
 └── html
      ├── head
      │     └── title → "Назва сторінки"
      └── body
            ├── h1 → "Привіт!"
            └── p → "Це приклад DOM дерева"

🔹 Як це працює

  • DOM створюється браузером під час завантаження сторінки.
  • JavaScript має доступ до DOM, щоб змінювати, додавати або видаляти елементи.
  • Кожен елемент DOM можна отримати за допомогою методів: getElementById(), querySelector(), getElementsByClassName() тощо.
const heading = document.querySelector("h1");
heading.textContent = "DOM змінено!";

Підсумок: DOM — це міст між HTML і JavaScript. Він дозволяє динамічно керувати вмістом сторінки.

25. Що таке прототипне наслідування?

Прототипне наслідування — це механізм JavaScript, при якому об’єкти можуть наслідувати властивості та методи інших об’єктів через [[Prototype]].

const person = {
  greet() {
    console.log("Привіт!");
  }
};

const user = Object.create(person);
user.name = "Serhii";

user.greet(); // "Привіт!"

Якщо у властивості об’єкта немає потрібного методу чи змінної — JS виконує пошук по ланцюжку прототипів (prototype chain).

Перевага: повторне використання коду. Недолік: складність при глибоких ланцюгах.

26. Як отримати властивість об’єкта в JavaScript?

Є два способи доступу до властивостей об’єкта:

🔹 1. Через крапку (dot notation) — статичний

const obj = { a: 5 };
console.log(obj.a); // 5

🔹 2. Через квадратні дужки (bracket notation) — динамічний

const key = "a";
console.log(obj[key]); // 5

Другий спосіб зручний, коли ім’я властивості зберігається у змінній або містить пробіли.

27. Що таке всплиття та погруження подій у JavaScript?

🔹 Всплиття (bubbling)

Коли відбувається подія, вона спочатку спрацьовує на елементі-цілі, а потім “вспливає” до його батьків і далі вгору по дереву DOM.


  

    
Клікни

Таким чином подія переходить від внутрішнього елемента до зовнішнього.

🔹 Погруження (capturing)

Це зворотна фаза — подія рухається згори вниз (від window до target). Використовується рідко, але може бути корисною.

Фази проходження події:

  1. Фаза погруження (capturing phase)
  2. Цільова фаза (target phase)
  3. Фаза всплиття (bubbling phase)
elem.addEventListener('click', handler, { capture: true });
// або старіший синтаксис:
elem.addEventListener('click', handler, true);

Коротко: Події мають три фази, і можна контролювати, на якій саме фазі реагуватиме обробник.

28. Що таке оператор optional chaining (?.) і для чого він потрібен?

Optional chaining (опціональний ланцюжок) — це оператор ?., який дозволяє безпечно звертатися до вкладених властивостей об’єкта, не створюючи помилку, якщо частина шляху відсутня.

🔹 Приклад:

const user = { name: "Serhii", address: { city: "Kyiv" } };

console.log(user.address.city);  // Kyiv
console.log(user.address.street); // undefined

// Безпечний доступ:
console.log(user?.address?.street); // undefined (без помилки)

🔸 Якщо будь-яка частина виразу перед ?. дорівнює null або undefined — обчислення зупиняється, і результатом буде undefined.

Перевага: запобігає помилці TypeError: Cannot read property ... of undefined.

29. Що таке Shadow DOM?

Shadow DOM — це частина специфікації Web Components, яка дозволяє створювати ізольовану область DOM усередині елемента.

Вона приховує внутрішню структуру компонента від зовнішніх стилів та скриптів.

🔹 Приклад:

const shadowHost = document.querySelector('#box');
const shadow = shadowHost.attachShadow({ mode: 'open' });

shadow.innerHTML = `
  <style>p { color: red; }</style>
  <p>Привіт із Shadow DOM!</p>
`;

Shadow DOM створює власне дерево елементів, ізольоване від основного DOM, — це допомагає уникати конфліктів стилів і забезпечує інкапсуляцію.

30. Для чого потрібні поліфіли?

Поліфіл (polyfill) — це фрагмент коду або бібліотека, яка додає підтримку сучасних функцій у старих браузерах, де вони ще не реалізовані.

🔹 Приклад:

// Наприклад, додавання методу includes() для старих браузерів:
if (!Array.prototype.includes) {
  Array.prototype.includes = function(value) {
    return this.indexOf(value) !== -1;
  };
}

✅ Поліфіли дозволяють забезпечити кросбраузерність і зворотну сумісність.

31. Що таке функції-конструктори?

Функції-конструктори — це спеціальні функції, які використовуються для створення об’єктів.

Вони викликаються з ключовим словом new і автоматично створюють новий об’єкт.

🔹 Правила створення конструктора:

  1. Ім’я конструктора повинно починатися з великої літери.
  2. Викликається за допомогою new.
function User(name) {
  this.name = name;
  this.isAdmin = false;
}

const user1 = new User("Serhii");

console.log(user1.name); // Serhii
console.log(user1.isAdmin); // false

Під капотом:

  1. Створюється порожній об’єкт {}.
  2. this посилається на цей об’єкт.
  3. Об’єкт автоматично повертається з функції.
32. Як отримати список ключів і значень об’єкта в JavaScript?

Для отримання списку ключів або значень об’єкта використовуються вбудовані методи:

const obj = {
  field: 'value',
  key: 'value 2'
};

const keys = Object.keys(obj);
const values = Object.values(obj);

console.log(keys);   // ['field', 'key']
console.log(values); // ['value', 'value 2']

Також можна отримати пари ключ–значення за допомогою Object.entries():

console.log(Object.entries(obj));
// [['field', 'value'], ['key', 'value 2']]
33. Як зробити наслідування класу в ES6?

У ES6 наслідування реалізується за допомогою ключового слова extends. Дочірній клас успадковує всі властивості та методи батьківського класу.

class User {
  constructor(name) {
    this.name = name;
  }

  sayHi() {
    console.log(`Привіт, ${this.name}!`);
  }
}

class Admin extends User {
  constructor(name, rating) {
    super(name); // викликає конструктор батьківського класу
    this.rating = rating;
  }

  banUser() {
    console.log(`${this.name} заблокував користувача.`);
  }
}

const admin = new Admin("Serhii", 5);
admin.sayHi();   // Привіт, Serhii!
admin.banUser(); // Serhii заблокував користувача.

super() викликає конструктор батьківського класу. Усі класи у JS базуються на механізмі прототипного наслідування.

34. Що таке мікро- і макрозадачі в JavaScript?

JavaScript виконує асинхронний код у двох чергах:

🔹 Макрозадачі (macrotasks)

Це основні події, що потрапляють у чергу виконання після основного коду:

  • setTimeout, setInterval
  • setImmediate (у Node.js)
  • Події DOM, I/O операції

🔹 Мікрозадачі (microtasks)

Виконуються після поточного коду, але до переходу до макрозадач.

  • Promise.then(), catch(), finally()
  • queueMicrotask()
  • process.nextTick() (у Node.js)
console.log("1");

setTimeout(() => console.log("2"), 0);

Promise.resolve().then(() => console.log("3"));

console.log("4");

// Вивід: 1 → 4 → 3 → 2

🧠 Висновок: Event Loop завжди виконує всі microtasks перед переходом до наступної macrotask.

35. Що таке генератори в JavaScript і як вони працюють?

Генератори — це спеціальні функції, які можуть повертати кілька значень послідовно замість одного.

Вони позначаються зірочкою * після ключового слова function, а значення повертаються через yield.

function* generateSequence() {
  yield 1;
  yield 2;
  yield 3;
}

const generator = generateSequence();

console.log(generator.next().value); // 1
console.log(generator.next().value); // 2
console.log(generator.next().value); // 3

next() — повертає об’єкт із двома властивостями:

  • value — поточне значення;
  • done — булеве значення, чи завершено генерацію.

✅ Генератори зручні для створення ітераторів, потокової обробки даних або відкладеного виконання.

38. Які способи збереження даних у браузері ви знаєте?

У браузері існує кілька основних підходів до збереження даних:

🔹 1. LocalStorage / SessionStorage

Дозволяють зберігати пари ключ/значення у браузері.

  • LocalStorage — дані зберігаються навіть після перезавантаження сторінки або закриття браузера.
  • SessionStorage — дані зберігаються лише протягом поточної сесії (доки вкладка відкрита).

Основні методи:

localStorage.setItem("key", "value");     // зберегти
localStorage.getItem("key");              // отримати
localStorage.removeItem("key");           // видалити
localStorage.clear();                     // очистити

🔸 Обидва сховища зберігають дані у вигляді рядків, тому складні об’єкти треба перетворювати через JSON.stringify() і JSON.parse().

🔹 2. Cookies (куки)

Це короткі рядки даних, які зберігаються у браузері й передаються разом із запитом до сервера через заголовок Cookie.

  • Зазвичай використовуються для авторизації, налаштувань користувача або аналітики.
  • Можуть мати обмежений строк життя та розмір (~4 KB).
  • Куки додаються до кожного запиту на той самий домен.

🔹 3. IndexedDB

IndexedDB — вбудована база даних у браузері. Вона потужніша за LocalStorage і дозволяє:

  • зберігати великі обсяги структурованих даних;
  • використовувати індекси та запити;
  • зберігати об’єкти будь-якого типу;
  • працювати офлайн (наприклад, у Service Workers).

IndexedDB підходить для офлайн-додатків або складних клієнтських SPA-застосунків.

41. Як сервер може заборонити читання cookies із браузера?

Для цього використовується спеціальний прапорець HttpOnly.

Коли сервер надсилає куку з параметром HttpOnly, JavaScript не може отримати доступ до неї через document.cookie.

🔹 Приклад заголовка:

Set-Cookie: sessionId=abc123; HttpOnly; Secure; Path=/;
  • Захищає куки від викрадення через XSS-атаки.
  • JavaScript не може їх прочитати або змінити.
  • Такі куки використовуються для зберігання токенів сесії або автентифікації.

✅ Це найкраща практика безпеки при роботі з авторизацією на стороні клієнта.

42. Що таке регулярні вирази та для чого вони потрібні?

Регулярні вирази (RegExp) — спеціальний синтаксис для пошуку, перевірки або заміни текстових шаблонів у рядках.

// Приклад: перевірка email-адреси
const regex = /^[\w.-]+@[\w.-]+\.\w+$/;
console.log(regex.test("user@mail.com")); // true
console.log(regex.test("invalid@"));      // false

📘 Регулярки часто використовуються для валідації форм, пошуку тексту або заміни фрагментів у рядках через replace().

const text = "Hello 2025!";
console.log(text.replace(/\d+/g, "2026")); // Hello 2026!

✅ Основні прапорці: g — глобальний пошук, i — нечутливість до регістру, m — багаторядковий пошук.

43. Чим відрізняються WeakSet і WeakMap від Map та Set?

WeakMap і WeakSet — це спеціальні структури даних, які мають слабкі посилання на об’єкти.

Основна відмінність від Map полягає в тому, що ключі WeakMap можуть бути лише об’єктами, а не примітивами.

let weakMap = new WeakMap();

let obj = {};
weakMap.set(obj, "ok");

console.log(weakMap.get(obj)); // "ok"

Коли об’єкт більше не має посилань у коді, JavaScript автоматично видаляє його з пам’яті, а разом із ним — і запис у WeakMap або WeakSet.

  • WeakMap не підтримує ітерацію (for..of не працює).
  • WeakMap не має властивості size.
  • WeakSet працює аналогічно, але зберігає лише об’єкти без ключів.
  • WeakMap не підтримує ітерацію та методи keys(), values(), entries(), тому немає способу отримати всі ключі або значення від нього.
  WeakMap має лише такі методи:

  weakMap.set(key, value)
  weakMap.get(key)
  weakMap.delete(key)
  weakMap.has(key)

✅ Використовується для зберігання даних, що “прив’язані” до об’єктів (наприклад, кеш).

Jсновну роботу WeakMap/WeakSet – бути «додатковим» сховищем даних для об’єктів, які зберігаються/управляються в іншому місці.

44. Чому два об’єкти з однаковими властивостями при порівнянні дають false?

В JavaScript об’єкти порівнюються не за вмістом, а за посиланням у пам’яті.

const a = { key: 5 };
const b = { key: 5 };

console.log(a === b); // false

Навіть якщо об’єкти виглядають однаково, вони зберігаються у різних місцях пам’яті, тому посилання різні.

✅ Для перевірки рівності вмісту потрібно порівнювати властивості вручну або використовувати JSON.stringify(a) === JSON.stringify(b).

45. Чому примітиви (Number, String тощо) мають методи, наприклад 'some'.toUpperCase()?

Хоча примітиви не є об’єктами, JavaScript тимчасово обгортає їх у об’єкт-обгортку відповідного типу: String, Number, Boolean, Symbol.

Коли ви викликаєте метод, наприклад 'abc'.toUpperCase(), двигун JS створює тимчасовий об’єкт new String('abc'), викликає метод і одразу знищує цей об’єкт.

let str = "hello";
console.log(str.toUpperCase()); // "HELLO"

✅ Саме завдяки цьому ми можемо використовувати методи навіть на примітивах.

46. Як перевірити, з якого класу створено об’єкт?

Оператор instanceof дозволяє перевірити, чи належить об’єкт певному класу (з урахуванням наслідування).

class Rabbit {}
let rabbit = new Rabbit();

console.log(rabbit instanceof Rabbit); // true

✅ Використовується для типізації та перевірки екземплярів у складних структурах.

47. Як вивести в консоль час перебування користувача на сайті (кожні 5 секунд)?

Можна скористатись методом setInterval(), який виконує функцію через певний інтервал часу.

let time = 0;

setInterval(() => {
  time += 5;
  console.log(`Користувач на сайті ${time} секунд`);
}, 5000);

✅ Інтервали зручно використовувати для оновлення UI, таймерів або періодичного опитування API.

48. Як зупинити інтервал у JavaScript?

Для зупинки інтервалу використовується функція clearInterval().

let time = 0;

const interval = setInterval(() => {
  time += 5;
  console.log(`Пройшло ${time} секунд`);

  if (time === 10) {
    clearInterval(interval);
    console.log("Інтервал зупинено");
  }
}, 5000);

✅ clearInterval() приймає ідентифікатор, який повертає setInterval().

49. Що таке чиста функція?

Чиста функція — це функція, яка завжди повертає однаковий результат при однакових вхідних даних і не має побічних ефектів.

✅ Властивості чистих функцій:

  • Повертає той самий результат при тих самих аргументах.
  • Не змінює зовнішні змінні або об’єкти (немає побічних ефектів).
// Чиста функція
function sum(a, b) {
  return a + b;
}

// Нечиста функція
let count = 0;
function addToCount(value) {
  count += value; // змінює зовнішню змінну
}

Чисті функції легко тестувати та використовувати в функціональному програмуванні.

51. Навіщо потрібні проміси, якщо можна працювати з callback-функціями?

До появи Promise асинхронний код часто будувався на вкладених callback-функціях:

getData(function(data) {
  console.log("Отримали дані:", data);
  getUserInfo(data.id, function(info) {
    console.log("Інформація користувача:", info);
  });
});

Такий підхід називають callback hell (пекло колбеків), бо вкладеність робить код важким для читання та підтримки.

Рішення — використання Promise:

getData()
  .then(data => getUserInfo(data.id))
  .then(info => console.log("Інформація користувача:", info))
  .catch(error => console.error("Помилка:", error));

✅ Переваги Promise:

  • Легше читати послідовний код.
  • Можна обробляти помилки через .catch().
  • Можна комбінувати кілька запитів через Promise.all().
52. Що буде, якщо додати скрипт перед тегом <body>?

У сучасних сайтах скрипти зазвичай «важчі» за HTML, тому їх обробка займає більше часу.

Коли браузер завантажує HTML і доходить до тега <script>, він зупиняє побудову DOM, щоб спочатку виконати цей скрипт.

Якщо скрипт підключений у <head>, браузер виконає його перед рендерингом сторінки. Це означає, що DOM ще не створено, і скрипт не зможе звернутися до елементів нижче за деревом.

🚫 Проблеми при розміщенні скриптів у верхній частині сторінки:

  • Скрипт може спробувати звернутися до елементів, яких ще немає в DOM.
  • Великі скрипти блокують побудову сторінки — користувач бачить порожню сторінку, поки JS не виконається.

✅ Тому рекомендовано розміщувати скрипти перед закриваючим тегом </body> або додавати атрибути defer / async.

<script src="main.js" defer></script>
53. Як перевірити, чи є об’єкт масивом?

Для цього використовується спеціальний метод Array.isArray(), який повертає true або false.

console.log(Array.isArray([1, 2, 3]));   // true
console.log(Array.isArray({ a: 1 }));    // false
console.log(Array.isArray("hello"));     // false

✅ Цей метод надійніший за instanceof Array, оскільки коректно працює навіть між різними контекстами (iframe, window тощо).

54. Що таке рекурсія? Як уникнути нескінченного виклику?

Рекурсія — це спосіб, коли функція викликає саму себе безпосередньо або через інші функції.

Будь-яка рекурсивна функція складається з двох частин:

  • Базовий випадок — умова зупинки, яка завершує виклики.
  • Рекурсивний випадок — виклик самої себе з новими параметрами.
// Приклад рекурсії: факторіал
function factorial(n) {
  if (n === 1) return 1;      // базовий випадок
  return n * factorial(n - 1); // рекурсивний виклик
}

console.log(factorial(5)); // 120

✅ Щоб уникнути нескінченного циклу рекурсії, завжди додавай базову умову зупинки — без неї функція викликатиме себе безкінечно.

55. Що таке «strict mode» в JavaScript і як його включити?

Strict mode — це особливий режим виконання JavaScript, який вводить деякі обмеження, щоб уникнути помилок, що виникають через неявне використання глобальних змінних та інших небезпечних практик. Для активації достатньо додати директиву "use strict"; на початку скрипта.

"use strict";
x = 10; // Помилка: x не оголошено
56. Що таке асинхронний виклик функцій з використанням setTimeout()?

setTimeout() дозволяє відкласти виконання функції на певний проміжок часу (в мілісекундах).

setTimeout(() => {
    console.log("Це виведеться через 3 секунди");

57. Як працює метод setInterval() в JavaScript?

Метод setInterval() виконує задану функцію через певні інтервали часу, повторюючи це доти, поки не буде викликано clearInterval().

let count = 0;
let intervalId = setInterval(() => {
    count++;
    console.log(count);
    if (count >= 5) {
        clearInterval(intervalId);
    }
}, 1000);
58. Як працює метод clearTimeout() в JavaScript?

Метод clearTimeout() зупиняє виконання функції, яка була запланована через setTimeout().

let timeoutId = setTimeout(() => {
    console.log("Це не буде виведено!");
}, 5000);

clearTimeout(timeoutId);
59. Що таке проксі (Proxy) в JavaScript?

Proxy в JavaScript дозволяє створювати об'єкти, що перехоплюють операції з іншими об'єктами, такі як доступ до властивостей, методи і т.д.

let person = {
    name: "Олексій",
    age: 30
};
let handler = {
    get: function(target, prop) {
        if (prop in target) {
            return target[prop];
        } else {
            return `Властивість ${prop} не існує`;
        }
    }
};
let proxy = new Proxy(person, handler);
console.log(proxy.name); // Олексій
console.log(proxy.city); // Властивість city не існує
60. Як працює метод Object.freeze() в JavaScript?

Метод Object.freeze() заморожує об'єкт, запобігаючи будь-яким змінам його властивостей. Властивості об'єкта не можуть бути змінені, додані або видалені.

let person = { name: "Олексій" };
Object.freeze(person);
person.name = "Іван"; // Не буде змінено
console.log(person.name); // Олексій
61. Як працює "Lazy Loading" в JavaScript?

Lazy Loading — це техніка, яка дозволяє відкладати завантаження ресурсів або модулів до того моменту, коли вони стануть необхідними. Це дозволяє покращити продуктивність додатку.

const modulePromise = import('./someModule.js'); // Завантажує модуль лише при необхідності
modulePromise.then(module => {
    module.doSomething();
});
62. Що таке "Web Workers" і коли їх слід використовувати?

Web Workers — це API, яке дозволяє виконувати JavaScript в окремому потоці, паралельно з основним потоком, що дозволяє виконувати важкі обчислення, не блокуючи інтерфейс користувача.

const worker = new Worker('worker.js');
worker.postMessage('Start task');

worker.onmessage = function(event) {
    console.log('Message from worker: ', event.data);
};
63. Що таке "Curry" в JavaScript і навіщо він потрібен?

Curry — це техніка перетворення функції з кількома аргументами на послідовність функцій, кожна з яких приймає один аргумент. Це допомагає створювати частково застосовані функції.

function multiply(a) {
    return function(b) {
        return a * b;
    };
}

const multiplyBy2 = multiply(2);
console.log(multiplyBy2(5)); // 10
64. Що таке "Singleton" в JavaScript і як його реалізувати?

Singleton — це шаблон проектування, що гарантує існування тільки одного екземпляра певного об'єкта протягом всього життєвого циклу додатка.

const Singleton = (function() {
    let instance;

    function createInstance() {
        return { name: 'Singleton' };
    }

    return {
        getInstance: function() {
            if (!instance) {
                instance = createInstance();
            }
            return instance;
        }
    };
})();

const instance1 = Singleton.getInstance();
const instance2 = Singleton.getInstance();
console.log(instance1 === instance2); // true
65. Що таке "Memoization" в JavaScript?

Memoization — це оптимізація, яка запам'ятовує вже обчислені значення функції для того, щоб уникнути їх повторного обчислення при однакових аргументах.

const memoize = (fn) => {
    const cache = {};
    return (n) => {
        if (cache[n]) {
            return cache[n];
        }
        const result = fn(n);
        cache[n] = result;
        return result;
    };
};

const factorial = memoize((n) => n <= 1 ? 1 : n * factorial(n - 1));
console.log(factorial(5)); // 120
