Що таке RxJS і навіщо він потрібен?

RxJS (Reactive Extensions for JavaScript) - це бібліотека для реактивного програмування з використанням Observables.

Основні переваги:

  • Управління асинхронними операціями
  • Композиція складних потоків даних
  • Обробка подій
  • Скасування операцій
Що таке Observable?

Observable - це потік даних, на який можна підписатися і отримувати значення з плином часу.

const observable = new Observable(subscriber => {
  subscriber.next(1);
  subscriber.next(2);
  setTimeout(() => {
    subscriber.next(3);
    subscriber.complete();
  }, 1000);
});

observable.subscribe({
  next: value => console.log(value),
  complete: () => console.log('Done')
});
Різниця між Observable і Promise?

Observable:

  • Множинні значення
  • Можна скасувати (unsubscribe)
  • Lazy - виконується тільки при підписці
  • Багато операторів для трансформації

Promise:

  • Одне значення
  • Не можна скасувати
  • Eager - виконується одразу
  • Обмежені методи (.then, .catch)
Що таке Subject в RxJS?

Subject - це спеціальний тип Observable, який дозволяє multicast значення багатьом Observer'ам.

Типи Subject:

  • Subject - базовий multicasting
  • BehaviorSubject - зберігає поточне значення
  • ReplaySubject - replay N останніх значень
  • AsyncSubject - емітить тільки останнє значення
const subject = new BehaviorSubject(0);
subject.next(1);
subject.subscribe(val => console.log(val)); // 1
subject.next(2); // 2
Популярні оператори RxJS?

Transformation: map, switchMap, mergeMap, concatMap

Filtering: filter, debounceTime, distinctUntilChanged, take

Combination: combineLatest, merge, forkJoin, zip

Error Handling: catchError, retry, retryWhen

input$.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(term => this.search(term))
).subscribe(results => console.log(results));
