Що таке Redux?

Redux - це бібліотека для управління станом додатку. Вона реалізує Flux архітектуру з передбачуваними змінами стану.

Три принципи Redux:

  • Single source of truth - один Store для всього стану
  • State is read-only - стан може бути змінений тільки через actions
  • Changes are made with pure functions - reducers - чисті функції
Основні концепції Redux?

Store: Зберігає стан додатку

Actions: Об'єкти, що описують що сталося

Reducers: Функції, що визначають як стан змінюється

Dispatch: Метод для відправки actions

// Action
const increment = { type: 'INCREMENT' };

// Reducer
function counter(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    default:
      return state;
  }
}

// Store
const store = createStore(counter);
store.dispatch(increment);
Що таке Action Creators?

Action Creators - це функції, які створюють actions.

// Action Creator
function addTodo(text) {
  return {
    type: 'ADD_TODO',
    payload: {
      id: Date.now(),
      text,
      completed: false
    }
  };
}

// Використання
store.dispatch(addTodo('Learn Redux'));
Що таке Middleware в Redux?

Middleware - це проміжний шар між dispatch action і reducer, який дозволяє розширити функціональність Redux.

Популярні middleware:

  • redux-thunk - для асинхронних actions
  • redux-saga - складні side effects
  • redux-logger - логування actions
// Redux Thunk приклад
function fetchUsers() {
  return async (dispatch) => {
    dispatch({ type: 'FETCH_USERS_START' });
    try {
      const users = await api.getUsers();
      dispatch({ type: 'FETCH_USERS_SUCCESS', payload: users });
    } catch (error) {
      dispatch({ type: 'FETCH_USERS_ERROR', error });
    }
  };
}
Як працює combineReducers?

combineReducers - утиліта для об'єднання декількох reducers в один root reducer.

import { combineReducers } from 'redux';

const rootReducer = combineReducers({
  users: usersReducer,
  posts: postsReducer,
  comments: commentsReducer
});

// Результуючий state
{
  users: { ... },
  posts: { ... },
  comments: { ... }
}
