Запитання для інтерв'ю по Git

1. Що таке Git?

Git — це розподілена система керування версіями, що дозволяє відстежувати зміни в коді під час розробки програмного забезпечення. Git дозволяє кільком розробникам працювати над одним проектом одночасно, зберігаючи історію змін в коді.

2. Яка різниця між Git та GitHub?

Git — це система керування версіями, яка використовується для відстежування змін у коді. GitHub — це платформа для хостингу Git-репозиторіїв, яка надає інструменти для співпраці над кодом та управління проектами.

3. Що таке репозиторій у Git?

Репозиторій (repo) — це місце зберігання файлів вашого проекту та їх історії змін. У Git репозиторій містить всі коміти, гілки, теги та інші об'єкти, пов'язані з вашим проектом.

4. Для чого використовуються гілки (branches) у Git?

Гілки дозволяють працювати над різними версіями проекту без впливу на основну гілку. Вони використовуються для розробки нових функцій, виправлення помилок чи експериментів без зміни основної кодової бази.

5. Що таке коміт (commit) у Git?

Коміт — це знімок змін, які були внесені в проект. Він включає в себе повідомлення, яке описує, що саме було змінено, і дозволяє зберегти стан проекту на певний момент часу.

6. Що таке pull request у Git?

Pull request — це запит на злиття змін з однієї гілки в іншу. Зазвичай використовується в GitHub або GitLab для огляду та обговорення змін перед їх злиттям в основну кодову базу.

7. У чому різниця між rebase та merge у Git?

Rebase перезаписує історію, переміщуючи або об'єднуючи послідовність комітів на нову базову точку. Це дозволяє створити лінійну історію, але може змінити попередні дані комітів.

Merge об'єднує історії двох гілок, створюючи новий коміт без зміни попередніх даних. Merge зберігає історію про те, як зміни були об'єднані та які були злиті гілки.

8. Що таке файл .gitignore?

Файл .gitignore вказує Git, які файли чи каталоги потрібно ігнорувати в проекті. Це дозволяє виключити з відстежування непотрібні або чутливі файли, наприклад, файли конфігурацій або результати компіляції.

9. Яка різниця між git pull та git fetch?

git fetch завантажує зміни з віддаленого репозиторію в ваш локальний репозиторій, але не зливає їх у робочу директорію. Це дозволяє вам подивитися доступні зміни перед тим, як їх змержити.

git pull поєднує команду git fetch та git merge, тобто завантажує зміни з віддаленого репозиторію та автоматично зливає їх у вашу робочу директорію.

10. Що робить команда 'git status'?

Команда git status показує поточний стан вашої робочої директорії та індексу. Вона показує, які файли були змінені, додані для коміту чи не відстежуються Git.

11. Що таке git clone?

Команда git clone створює копію віддаленого репозиторію на вашому комп'ютері. Вона завантажує всю історію комітів та файли з віддаленого репозиторію та створює локальну копію.

12. Як відмінити зміни, зроблені в робочій директорії (без коміту)?

Щоб скасувати зміни в робочій директорії, можна використати команду git checkout -- , що відновить файл до останнього коміту. Якщо ви хочете скасувати всі зміни в директорії, використовуйте git restore або git restore .

13. Що таке git stash?

Команда git stash дозволяє тимчасово зберегти незавершені зміни, щоб потім відновити їх. Це корисно, коли потрібно переключитися на іншу гілку без коміту незавершених змін.

14. Як злисти дві гілки в Git?

Для злиття двох гілок використовують команду git merge . Це зливає зміни з вказаної гілки в поточну гілку. Якщо є конфлікти, їх потрібно буде вирішити вручну.

15. Як переглянути історію комітів в Git?

Щоб переглянути історію комітів, використовуйте команду git log. Це виведе список комітів у зворотньому хронологічному порядку, включаючи ідентифікатори комітів, авторів і повідомлення комітів.

16. Як скасувати останній коміт в Git?

Для скасування останнього коміту використовуйте команду git reset --soft HEAD~1 для збереження змін у робочій директорії або git reset --hard HEAD~1 для скасування всіх змін і самого коміту.

17. Як відновити видалений коміт в Git?

Якщо ви випадково видалили коміт, можна використати команду git reflog для перегляду всіх комітів і знайти ідентифікатор коміту. Потім використовуйте git reset --hard , щоб повернутися до потрібного коміту.

18. Що таке git fetch?

Команда git fetch завантажує зміни з віддаленого репозиторію, але не зливає їх у вашу робочу директорію. Це дозволяє вам переглянути зміни перед тим, як їх інтегрувати в свою локальну гілку.

19. Як видалити локальну гілку в Git?

Для видалення локальної гілки використовуйте команду git branch -d . Якщо гілка ще не була змержена, для примусового видалення використовуйте git branch -D .

20. Що таке git remote?

Git remote — це посилання на віддалений репозиторій, з яким ви працюєте. Ви можете додавати віддалені репозиторії за допомогою команди git remote add , а також переглядати їх через команду git remote -v.

21. Що таке git rebase?

Команда git rebase дозволяє переносити або об'єднувати коміти з однієї гілки в іншу. Вона перезаписує історію комітів і дозволяє отримати більш лінійну історію змін.

22. Що таке git merge і як він відрізняється від git rebase?

Команда git merge об'єднує зміни з однієї гілки в іншу, зберігаючи історію комітів з обох гілок. У той час як git rebase переносить зміни, перезаписуючи історію, створюючи лінійну послідовність комітів.

23. Як переглянути зміни, що були внесені у файл після останнього коміту?

Для перегляду змін у файлі після останнього коміту використовуйте команду git diff . Це покаже різницю між поточним станом файлу і останнім комітом.

24. Як додати зміни в git stage?

Для додавання змін у stage використовуйте команду git add , де — це файл або директорія, яку ви хочете додати. Для додавання всіх змін використовуйте git add .

25. Як вивести список всіх гілок у репозиторії?

Для перегляду списку всіх гілок в репозиторії використовуйте команду git branch. Для перегляду віддалених гілок додайте -r: git branch -r.

26. Як змінити повідомлення останнього коміту в Git?

Щоб змінити повідомлення останнього коміту, використовуйте команду git commit --amend. Це дозволить змінити повідомлення, не змінюючи вміст коміту.

27. Як синхронізувати локальний репозиторій з віддаленим?

Для синхронізації локального репозиторію з віддаленим використовуйте команди git fetch для завантаження змін з віддаленого репозиторію і git pull для злиття цих змін з вашою поточною гілкою.

28. Як працює git reset і які є його варіанти?

Команда git reset скидає поточний коміт або зміни. Існують три основних варіанти: --soft (скидає коміт, але зберігає зміни в stage), --mixed (скидає коміт і зміни зі stage), --hard (повністю скидає коміт і зміни, включаючи робочу директорію).

29. Як додати віддалений репозиторій до вашого проекту?

Для додавання віддаленого репозиторію використовуйте команду git remote add , де — це ім'я для вашого віддаленого репозиторію, а — це його адреса.

30. Як відновити видалений файл у Git?

Для відновлення видаленого файлу використовуйте команду git checkout -- . Це дозволить повернути файл з конкретного коміту, навіть якщо він був видалений в поточній гілці.
