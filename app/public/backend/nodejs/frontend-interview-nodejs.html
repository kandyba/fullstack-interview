Запитання та відповіді: Node.js

1. Що таке Node.js?
Node.js — це середовище виконання JavaScript, яке працює поза браузером. Воно побудоване на рушії V8 і дозволяє створювати серверні програми. 
Основні особливості Node.js:
- Асинхронна подієва модель.
- Однопотокова архітектура.
- Підтримка модулів CommonJS.
2. У чому різниця між require та import у Node.js?
  • require — це частина стандарту CommonJS, використовується у старіших версіях Node.js.
  • import — це частина стандарту ES6, потребує використання ECMAScript модулів (ESM).
// Використання require
const fs = require('fs');

// Використання import
import fs from 'fs';
3. Що таке пакетний менеджер npm?
npm (Node Package Manager) — це менеджер пакетів для Node.js, який дозволяє встановлювати, оновлювати та управляти залежностями проєкту. 
Основні команди npm:
- npm init: ініціалізація нового проєкту.
- npm install package_name: встановлення пакету.
- npm update: оновлення залежностей.
4. Що таке модулі в Node.js?
Модулі — це незалежні блоки коду, які можна імпортувати та використовувати в інших файлах. У Node.js існують вбудовані модулі, як-от fs, http, та створені користувачем модулі. 
// Вбудований модуль
const fs = require('fs');

// Користувацький модуль
const myModule = require('./myModule');
5. Як працює подієва модель Node.js?
Node.js використовує асинхронну подієву модель, що дозволяє обробляти багато запитів без блокування головного потоку. 
const http = require('http');

const server = http.createServer((req, res) => {
  res.end('Привіт, Node.js!');
});

server.listen(3000, () => {
  console.log('Сервер запущено на порту 3000');
});
6. Що таке буфери в Node.js?
Буфери — це тимчасове сховище для двійкових даних. Вони корисні при роботі з потоками, такими як файли або мережеві запити. 
const buf = Buffer.from('Привіт');
console.log(buf.toString()); // "Привіт"
7. Що таке потоки в Node.js?
Потоки — це інтерфейси для роботи з даними, які передаються частинами. Вони можуть бути Readable, Writable, Duplex або Transform. 
const fs = require('fs');

const readStream = fs.createReadStream('./input.txt');
const writeStream = fs.createWriteStream('./output.txt');

readStream.pipe(writeStream);
8. Як обробляти помилки в Node.js?
Для обробки помилок використовують try...catch, події error або обробники для асинхронних функцій. 
try {
  const data = fs.readFileSync('./file.txt');
  console.log(data);
} catch (error) {
  console.error('Помилка:', error.message);
}
9. Як працює пакет express?
Express — це веб-фреймворк для Node.js, який спрощує створення серверних додатків. 
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Привіт, Express!');
});

app.listen(3000, () => {
  console.log('Сервер працює на порту 3000');
});
10. Як виконати HTTP-запити у Node.js?
HTTP-запити у Node.js можна виконувати за допомогою вбудованого модуля http, бібліотеки axios або node-fetch. 
const https = require('https');

https.get('https://jsonplaceholder.typicode.com/todos/1', (res) => {
  let data = '';

  res.on('data', (chunk) => {
    data += chunk;
  });

  res.on('end', () => {
    console.log(JSON.parse(data));
  });
});
11. Що таке Middleware у Node.js?
Middleware — це функції, які виконуються між отриманням запиту і відправкою відповіді. У Express middleware використовується для обробки запитів, виконання логіки або зміни відповіді. 
const express = require('express');
const app = express();

const logger = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(logger);

app.get('/', (req, res) => {
  res.send('Привіт, Middleware!');
});

app.listen(3000, () => {
  console.log('Сервер працює на порту 3000');
});
12. Що таке подієвий цикл у Node.js?
Подієвий цикл — це механізм, який Node.js використовує для обробки асинхронних операцій. Він дозволяє неблокуючий ввід/вивід і обробляє події через чергу колбеків. 
console.log('Початок');

setTimeout(() => {
  console.log('Таймаут');
}, 0);

console.log('Кінець');
13. Як працювати з файлами у Node.js?
Для роботи з файлами використовується вбудований модуль fs. Він підтримує як синхронні, так і асинхронні методи. 
const fs = require('fs');

// Читання файлу
fs.readFile('./file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// Запис у файл
fs.writeFile('./output.txt', 'Привіт, файл!', (err) => {
  if (err) throw err;
  console.log('Файл записано');
});
14. Як створити HTTP-сервер у Node.js?
HTTP-сервер створюється за допомогою модуля http. Він обробляє запити та відповіді. 
const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Привіт, сервер!');
});

server.listen(3000, () => {
  console.log('Сервер працює на порту 3000');
});
15. Що таке кластеризація у Node.js?
Кластеризація дозволяє використовувати всі ядра процесора для запуску кількох екземплярів Node.js, що підвищує продуктивність. 
const cluster = require('cluster');
const http = require('http');
const os = require('os');

if (cluster.isMaster) {
  const cpus = os.cpus().length;
  console.log(`Запускаємо ${cpus} воркерів`);

  for (let i = 0; i < cpus; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker) => {
    console.log(`Воркер ${worker.process.pid} завершив роботу`);
  });
} else {
  http.createServer((req, res) => {
    res.end('Привіт, кластер!');
  }).listen(3000);
}

// main.js

Node.js має кілька глобальних змінних, наприклад __dirname, __filename, process, які доступні у всіх модулях. 
console.log('Каталог:', __dirname);

console.log('Процес:', process.version);
18. Що таке події у Node.js?
Події у Node.js дозволяють обробляти асинхронні операції через EventEmitter. 
const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('greet', () => {
  console.log('Привіт, події!');
});

emitter.emit('greet');
19. Як працювати з базами даних у Node.js?
Node.js підтримує підключення до різних баз даних, наприклад MongoDB, PostgreSQL, MySQL. Для цього використовуються бібліотеки, як-от mongoose для MongoDB або pg для PostgreSQL. 
// Підключення до MongoDB

mongoose.connect('mongodb://localhost/test', { useNewUrlParser: true, useUnifiedTopology: true })
  .then(() => console.log('Підключено до MongoDB'))
  .catch(err => console.error('Помилка підключення:', err));

// Генерація помилки

28. Як підключити HTTPS у Node.js?

const options = {

33. Що таке middleware у Node.js?
Middleware — це функції, які виконуються між отриманням запиту та його обробкою. Вони використовуються для логування, автентифікації та інших завдань. 
const express = require('express');
const app = express();

app.use((req, res, next) => {
  console.log('Middleware виконується');

app.get('/', (req, res) => {

34. Що таке кластеризація у Node.js?
Кластеризація дозволяє запускати кілька процесів Node.js для використання всіх ядер процесора. 
const cluster = require('cluster');
const http = require('http');
const numCPUs = require('os').cpus().length;

if (cluster.isPrimary) {
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }
} else {
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end('Привіт, кластер!');
  }).listen(3000);
}

38. Як використовувати проміси в Node.js?

39. Як працювати з async/await у Node.js?
async/await дозволяє писати асинхронний код у синхронному стилі. 
const fs = require('fs').promises;

async function readFile() {
  try {
    const data = await fs.readFile('file.txt', 'utf8');
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

readFile();
