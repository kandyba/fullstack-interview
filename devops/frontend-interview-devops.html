Запитання та відповіді для технічної співбесіди (DevOps для Fullstack)

1. Що таке DevOps?

DevOps — це підхід до розробки, який поєднує Development (розробку) і Operations (експлуатацію) з метою автоматизації процесів доставки, деплою та підтримки застосунків.

Мета — швидше постачати якісний продукт, зменшити кількість помилок і прискорити оновлення.

2. Що таке CI/CD?

CI/CD (Continuous Integration / Continuous Deployment) — це набір практик, які забезпечують безперервну інтеграцію коду та його автоматичне розгортання.

  • CI — автоматична перевірка та збірка після кожного пушу коду.
  • CD — автоматичне деплоювання коду на сервер або хмару.
3. Які інструменти CI/CD ти використовував?

Найчастіше — GitHub Actions, GitLab CI/CD або Jenkins.

Для кожного пушу можна автоматично запускати тести, збірку та деплой.

name: Build and Deploy
on: push
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install && npm run build
4. Що таке Docker і навіщо його використовують?

Docker — це система контейнеризації, що дозволяє запускати застосунки в ізольованому середовищі.

Контейнери працюють однаково на будь-якому сервері — “працює в мене” = “працює всюди”.

5. Що таке Dockerfile?

Dockerfile — це інструкція для створення Docker-образу.

FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["npm", "start"]
6. Для чого потрібен Docker Compose?

Він дозволяє запускати кілька контейнерів одночасно (наприклад, frontend, backend і db).

services:
  web:
    build: .
    ports: ["3000:3000"]
  db:
    image: postgres
    environment:
      POSTGRES_PASSWORD: root
7. Чим контейнер відрізняється від віртуальної машини?
  • Віртуальна машина має власну ОС.
  • Контейнер використовує ядро хоста, тому легший і швидше запускається.
8. Що таке Nginx і як його зазвичай використовують?

Nginx — це HTTP-сервер, який часто працює як reverse proxy, балансувальник навантаження або статичний сервер.

server {
  listen 80;
  server_name myapp.com;
  location / {
    proxy_pass http://localhost:3000;
  }
}
9. Що таке змінні оточення (environment variables)?

Це змінні, що зберігають конфігураційні дані (паролі, API-ключі, URL бази тощо).

PORT=3000
DATABASE_URL=postgres://root:1234@localhost:5432/mydb
10. Як зазвичай використовують .env файли?

Файл .env містить конфігураційні змінні, які підвантажуються у проєкт через бібліотеку, наприклад dotenv.

require('dotenv').config();
console.log(process.env.PORT);
11. Що таке SSH і як ним користуватись?

SSH (Secure Shell) — це протокол для безпечного підключення до віддаленого сервера.

ssh user@192.168.0.10
scp file.txt user@host:/home/user/
12. Як можна запустити Node.js застосунок на сервері?
  • PM2 — зручний процес-менеджер (pm2 start app.js).
  • systemd — сервіс ОС для автозапуску після перезавантаження.
13. Які інструменти моніторингу ти знаєш?

Для базового моніторингу: htop, top, journalctl, docker stats.

Для продакшну: Prometheus, Grafana, Sentry.

14. Які етапи зазвичай має CI/CD pipeline?
  • Lint + тести
  • Збірка (build)
  • Деплой (deploy)
  • Моніторинг і сповіщення
15. Як можна зробити резервну копію бази даних?
# PostgreSQL
pg_dump mydb > backup.sql

# MongoDB
mongodump --db=mydb --out=/backup
16. Які хмарні сервіси ти знаєш?
  • AWS (EC2, S3, RDS)
  • Google Cloud Platform (GCP)
  • Azure
  • Vercel, Netlify — для деплою фронтенду
17. Що таке Reverse Proxy?

Це сервер, який приймає запити від користувача і перенаправляє їх на внутрішній застосунок.

Зазвичай використовується для SSL, кешу, балансування або приховування внутрішньої архітектури.

18. Як отримати безкоштовний SSL-сертифікат?

За допомогою Let's Encrypt і утиліти certbot.

sudo certbot --nginx -d mysite.com
19. Як переглянути логи процесу на сервері?
journalctl -u nginx -f
docker logs my_container
pm2 logs
20. У чому різниця між Continuous Delivery та Continuous Deployment?

Continuous Delivery — автоматична збірка і тестування, але деплой — вручну.

Continuous Deployment — деплой відбувається автоматично після успішних тестів.

21. Які інструменти для збірки застосунку ти використовував?

Залежно від стеку: Vite, Webpack, Gulp або Parcel. На бекенді — npm scripts або Docker build.

22. Що таке Load Balancer?

Load Balancer — це система, яка розподіляє навантаження між кількома серверами. Це допомагає уникнути перевантаження одного вузла й підвищує доступність сервісу.

Приклад: Nginx, HAProxy, або балансувальники в AWS / GCP.

23. Як кешування допомагає в DevOps?

Кешування зменшує навантаження на сервер і базу даних. Зазвичай використовується Redis або Cloudflare Cache для збереження частих запитів чи сесій користувачів.

24. Що таке log rotation?

Це процес автоматичного архівування старих логів і створення нових файлів журналів, щоб запобігти переповненню диску. У Linux зазвичай використовується утиліта logrotate.

25. Як оновлюються пакети безпеки на сервері?

Регулярно виконуються команди:

sudo apt update
sudo apt upgrade

На продакшн-серверах часто використовується unattended-upgrades або ручне оновлення після тестування.

26. Що таке firewall і для чого він потрібен?

Firewall (брандмауер) — це система, що контролює мережевий трафік і захищає сервер від несанкціонованого доступу.

sudo ufw enable
sudo ufw allow 22/tcp
sudo ufw allow 80,443/tcp
27. Що таке rollback у контексті деплою?

Rollback — це повернення до попередньої стабільної версії застосунку, якщо після оновлення виникла помилка.

Зазвичай реалізується через:

  • збереження попередніх Docker-образів,
  • Git-теги версій,
  • автоматичні backup-скрипти.
28. Як переглянути логи контейнера Docker?
docker logs my_container
docker logs -f my_container   # у реальному часі

Також можна перенаправити логи у файл або систему моніторингу (наприклад, Loki чи ELK stack).

29. Що таке Docker network і навіщо її створюють?

Docker network — це віртуальна мережа, у якій контейнери можуть спілкуватися між собою по імені сервісу.

docker network create mynet
docker run --network=mynet backend
docker run --network=mynet frontend
30. Як зберігаються секрети у CI/CD системах?

Усі конфіденційні дані (паролі, ключі API) зберігаються у зашифрованому вигляді в CI/CD-сховищі секретів. Наприклад:

  • GitHub Secrets — через Settings → Secrets
  • GitLab Variables — Settings → CI/CD → Variables

У workflow їх можна викликати як ${{ secrets.MY_KEY }}.
